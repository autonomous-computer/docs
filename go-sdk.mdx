## Example

```go
client := sdkgo.NewClient(os.Getenv("SECAPI_API_KEY"))
client.BaseURL = "https://api.secapi.ai"

calendar, err := client.MarketCalendar(map[string]string{
  "market": "XNYS",
  "duration": "5",
})
```

## Live example

Run the repo-tested agent workflow when you want the smallest production-backed
integration check: it resolves AAPL, fetches the latest 10-K, and extracts Item
1A in compact mode using `SECAPI_API_KEY` from the environment.

```bash
export SECAPI_API_KEY="secapi_live_..."
go run ./examples/agent_workflow
```

This same workflow ships in the JavaScript, Python, Go, and Rust SDKs, so you
can port it across stacks without changing the request shape.

Dashboard/account workflows can use bearer-token auth with `sdkgo.NewBearerTokenClient(os.Getenv("SECAPI_BEARER_TOKEN"))`.

Requests include `Accept: application/json` and a `User-Agent` like `secapi-go/1.0.1` so support and server logs can identify SDK traffic.

## Typed request params

The Go SDK keeps the raw `map[string]string` methods for exact REST control,
and adds typed params for common read paths when you want editor completion:

```go
filing, err := client.LatestFiling(sdkgo.LatestFilingParams{
  Ticker: "AAPL",
  Form: "10-K",
  View: sdkgo.ResponseViewAgent,
}.Params())

history, err := client.FactorHistory("VALUE", sdkgo.FactorHistoryParams{
  Range: "1y",
  ResponseMode: "compact",
  Include: "trust,series",
}.Params())
```

Typed params are available for `LatestFiling`, `LatestSection`,
`SearchSections`, `AllStatements`, and `FactorHistory`. Add `Extra` for newer or
less common query parameters without changing the wire contract.

## Typed agent responses

Use `*Agent` helpers when you want the compact agent response shape plus typed
Go fields:

```go
filing, err := client.LatestFilingAgent(sdkgo.LatestFilingParams{
  Ticker: "AAPL",
  Form: "10-K",
}.Params())
value := func(s *string) string {
  if s == nil {
    return ""
  }
  return *s
}
fmt.Println(filing.AccessionNumber, value(filing.FilingURL))

sections, err := client.SearchSectionsAgent(sdkgo.SectionSearchParams{
  Ticker: "AAPL",
  Form: "10-K",
  Query: "risk factors",
  Limit: "3",
}.Params())
for _, section := range sections.Data {
  fmt.Println(value(section.SectionKey), value(section.SourceURL))
}
```

Typed agent helpers are available for `ResolveEntityAgent`,
`LatestFilingAgent`, `SearchSectionsAgent`, and `StatementAgent`. They force
`view=agent` on supported routes before decoding into exported structs.
Nullable API fields decode as pointers so callers can distinguish missing data
from empty strings or zeros.

## Dilution intelligence

```go
// Top 10 highest-dilution-risk issuers
ratings, err := client.DilutionRatings(map[string]string{
  "overall_risk": "high",
  "limit":        "10",
})

// Single-ticker dilution rollup
score, err := client.DilutionScore(map[string]string{"ticker": "BBBB"})

// ATM offerings only (boolean filter on the events list)
atms, err := client.DilutionEvents(map[string]string{
  "ticker": "BBBB",
  "is_atm": "true",
})

// Coverage + freshness rollup (no agent shape — already compact)
coverage, err := client.DilutionCoverage(map[string]string{"ticker": "BBBB"})
```

The full surface — `DilutionEvents`, `DilutionEventDetail`, `DilutionWarrants`, `DilutionConvertibles`, `DilutionRofr`, `DilutionLockups`, `DilutionCashPosition`, `DilutionCorporateActions`, `DilutionNasdaqCompliance`, `DilutionRatings`, `DilutionReverseSplits`, `DilutionScore`, `DilutionShareFloatHistory`, `DilutionCoverage` — mirrors the REST routes 1:1. Pass `"view": "agent"` in any list call (except `DilutionCoverage`) for the compact agent-mode shape.
