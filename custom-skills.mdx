---
title: "Custom skills"
description: "Write portable SEC API workflow instructions that produce source-backed results"
---

A skill is an instruction file for an agent environment. SEC API does not define a universal skill installer or scheduler: use the file layout and installation mechanism documented by the agent product that will run it. Keep the SEC API portion portable by describing the workflow, its evidence requirements, and its stop conditions.

## Start with one bounded job

This example is deliberately plain Markdown. Adapt its frontmatter and location to the target agent environment.

```markdown
---
name: filing-risk-review
description: Retrieve and summarize a single issuer's latest 10-K risk factors with filing citations.
---

# Filing risk review

1. Resolve the issuer with `GET /v1/entities/resolve`.
2. Retrieve the latest 10-K with `GET /v1/filings/latest`.
3. Retrieve Item 1A with `GET /v1/filings/latest/sections/item_1a`.
4. Return the company name, accession number, form, filing date, filing URL, requestId, and a bounded summary of Item 1A.
5. Mark missing evidence as unavailable. Do not infer facts outside the retrieved source.

Use `x-api-key` from the runtime environment. Do not place credentials in this file.
```

## Design for evidence and change

Use exact REST paths only after checking the [API reference](/api-reference), or instruct an MCP client to discover tools with `tools.search` and `tools.describe`. A skill should name the source fields it must retain, especially `accessionNumber`, `filingDate`, `filingUrl`, and `requestId` for filing work.

Keep the scope narrow: one issuer, a filing family, a date range, and a result limit where the route supports one. A current latest filing can change. Do not encode a historical accession number, a fixed rate, or an assumed tool catalog as a success condition.

## Operate safely

Keep `SECAPI_API_KEY` in the agent runtime's secret store. Before repeated work, inspect billing and limits, quote the step when appropriate, and stop on authentication, validation, billing, or quota failures. Account changes and scheduling belong to the product or orchestration system that owns those capabilities; this documentation does not prescribe a client-specific command for either.

Published workflow skills are available in [secapi-ai/secapi-skills](https://github.com/secapi-ai/secapi-skills). Review the skill's instructions and your agent environment's installation documentation before enabling one in a production workflow.
