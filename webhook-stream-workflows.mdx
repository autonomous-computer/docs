---
title: "Webhook and stream workflows"
description: "Configure SEC API delivery, verify signed events, and inspect delivery history from your application"
---

Use webhooks when SEC API should push an event to your application. Use stream polling when your application needs cursor-controlled retrieval. Configure delivery in the signed-in dashboard; webhook and stream control-plane changes require a human-authenticated organization session.

## Create a webhook endpoint

Open the [SEC API dashboard](https://secapi.ai/app/webhooks) to manage webhook endpoints, delivery subscriptions, signing secrets, test deliveries, and replay settings. Do not use an API key or an autonomous agent for these changes: the API returns `human_auth_required` so a person can review the destination and subscription.

For a filing monitor, register a public HTTPS endpoint, subscribe it to the event you need, save the signing secret, then create the monitor that should deliver to it.

In the dashboard, enter the HTTPS destination and select the event types. Save the signing secret where your receiver can verify raw request bodies without exposing it to clients.

## Verify every webhook

Verify the exact raw request body against the `x-secapi-signature` header before parsing JSON. Then check the event type, store the delivery identifier, and preserve the source URL and request ID with your downstream work.

Treat a webhook as a notification to retrieve and evaluate evidence. Do not treat an event payload alone as an investment conclusion.

## Inspect delivery history

Inspect delivery attempts, event names, secret rotation, test delivery, replay, and subscription changes in the signed-in dashboard. Preserve the delivery identifier and source filing references when you investigate a failure.

## Choose polling when you need checkpoints

Stream polling gives your application control over cursors and replay. Store the cursor only after you have processed an event idempotently, and use the returned cursor rather than an event timestamp as the next checkpoint.

Read [Stream polling](/stream-polling) for the endpoint and CLI flow, or [Build a filing monitor](/tutorials/build-filing-monitor) for a complete monitor example.
