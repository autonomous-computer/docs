Coverage belongs to a request, not to the API in the abstract. Begin with the filing and representation you need, then check whether the response supports the use you intend.

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings?ticker=AAPL&form=10-K&limit=25"
```

## Documented historical boundary

Filing and section workflows document a nine-year searchable floor. Completeness checks cover required filing forms for the two most recently completed years. The current year has an availability floor, not a completed-year completeness guarantee.

Older filings can be returned, but issuer, form, document format, and materialization path determine whether a section, normalized fact, or derived result is present. Treat inventory targets or a successful query for one issuer as planning evidence, not a general coverage promise.

## Confirm the representation you need

| Requirement | Start with | Confirm in the response |
| --- | --- | --- |
| Source filing | filing or latest-filing route | issuer, form, filing date, accession number, filing URL |
| Filing text or section | render or section route | the chosen filing and section contain the requested material |
| Normalized value | facts or statements route | period, units, provenance, and disclosed fields |
| Event or ownership result | relevant event or ownership route | date range, form family, and returned limitation fields |

A source document is not a guarantee that every fact is normalized or comparable across issuers. A response for one year does not prove availability for another. A fast success does not establish timing fitness.

## Make unavailable evidence visible

Store the filing accession number and URL with results. Preserve returned periods, provenance, capabilities, warnings, and degradation fields. Do not silently substitute a different issuer, filing, period, or data type when the requested evidence is unavailable.

Market-calendar results have an independent scope. Read returned `coverage`, `confidence`, and `statusNote`: `configured_holidays` indicates configured holiday coverage, while `weekend_only` and `tentative` do not establish a complete local-holiday calendar.

SEC API does not claim equal data density across EDGAR, a universal real-time lag, or normalization for every exhibit and filing family.
