description: "Prove one authenticated, source-linked MCP filing workflow before enabling an agent loop."
---

Use hosted MCP when a team wants a client to discover and invoke SEC API tools inside an agent session. It is a transport evaluation, not evidence that a model summary is correct. Use REST when your application owns the workflow directly or needs the documented HTTP response surface.

## The buyer job

Connect one supported MCP client, authenticate it without exposing a secret, and retrieve a latest filing plus a named section with source identity intact. Public discovery at `GET https://api.secapi.ai/mcp` describes the deployed server. It does not authenticate a client or authorize tool use.

## Smallest proof of concept

Keep the key in the client process environment and configure it as `x-api-key`. Then have the client use authenticated MCP tool discovery before selecting tools; the inventory is the contract for that deployed session.

```bash
export SECAPI_API_KEY="secapi_..."
curl --fail-with-body -sS https://api.secapi.ai/mcp
```

Ask the configured client to resolve `AAPL`, retrieve its latest `10-K`, then retrieve `Item 1A`. Require it to return the tool names, accession number, filing URL, source text, and returned request metadata. Cross-check the underlying retrieval path:

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/latest?ticker=AAPL&form=10-K"
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/latest/sections/item_1a?ticker=AAPL&form=10-K"
```

`POST /mcp` is authenticated JSON-RPC. Do not describe discovery as anonymous MCP access: only the `GET /mcp` metadata request is public.

## What a passing result contains

Keep a secret-redacted discovery/configuration record, client name, selected tool names, tool arguments, accession number, filing URL, section identity and text, plus `requestId`, `traceparent`, and returned provenance/freshness when supplied. The REST cross-check should identify the same source record; it need not make a claim about identical model prose.

## Source and production limits

- A client may not implement every deployed transport behavior. Treat the current tool inventory and a real call as the compatibility test.
- A fluent answer without filing identity is not a passing research result.
- The returned section is filing evidence, not investment, legal, or accounting advice.
- `401`, `403`, JSON-RPC validation, billing, quota, and tool failures are distinct states. Preserve their identifiers; do not retry a broad agent task blindly.
- Estimate a bounded tool sequence with [Pricing and limits](/evaluate/pricing-and-limits) before enabling repeated or autonomous runs.

## Next decision

Use MCP in production only when the client can retain source identifiers and keep the API key in its own secret facility. Otherwise, use REST behind your server while the client boundary is repaired.
