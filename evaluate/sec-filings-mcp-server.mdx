title: "Evaluate SEC Filings MCP Server"
description: "Test hosted MCP discovery, environment-backed authentication, and source-linked filing retrieval."
---

Evaluate MCP as an agent transport, not as a substitute for evidence. The concrete job is to connect one client to the deployed server, request a latest filing and a named section, and confirm the client returns the same SEC source identity a REST workflow would preserve.

## Connection boundary

Read the deployed discovery metadata at `https://api.secapi.ai/mcp` before configuring a client. Supply the API key through an environment variable as the `x-api-key` header. Do not put the key in source control, browser-visible configuration, or a prompt transcript.

The hosted MCP surface and REST are related but not interchangeable in every client. Verify the deployed discovery result and the actual tool inventory before assuming a client can make a particular call.

## Proof of concept

1. Configure the MCP client from the deployed discovery metadata with an environment-backed `x-api-key`.
2. Ask it to find one issuer's latest 10-K and retrieve Item 1A.
3. Require the result to include its tool calls, accession number, filing URL, source text, and request metadata.
4. Cross-check the expected retrieval behavior with REST:

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/latest?ticker=AAPL&form=10-K"

curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/latest/sections/item_1a?ticker=AAPL&form=10-K"
```

## Expected evidence

Capture the discovery response or configuration version, client and tool names, request/result transcript with secrets removed, accession number, filing URL, `requestId`, `traceparent`, and returned source/freshness metadata. The test passes only when the client makes source review possible; a fluent summary without its filing identity does not pass.

## Failure modes

- Stale client configuration: re-read deployed discovery metadata instead of reusing an old local config.
- Authentication failure: confirm that `x-api-key` is injected from the environment and never exposed in the client-visible URL.
- Missing tool or unsupported behavior: use the deployed tool inventory and a REST cross-check to define the actual boundary.
- Tool failure or uncited result: retain the error, then retrieve the filing or section directly before downstream use.

MCP agent loops can compound usage. Run the same workload through [Pricing and limits](/evaluate/pricing-and-limits) before enabling autonomy.
