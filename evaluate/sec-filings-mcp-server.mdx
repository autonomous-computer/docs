## What to test

- Public discovery metadata at `GET https://api.secapi.ai/mcp`
- Auth through the `x-api-key` header
- A single issuer workflow across entity resolution, filing retrieval, section extraction, statements, and ownership
- Error handling with `requestId` and `traceparent`
- Agent budget checks through billing and limits endpoints

## Proof-of-concept prompt

<Prompt>
Evaluate the SEC API hosted MCP server for this agent workspace. Confirm the install metadata, register the streamable HTTP MCP server with the x-api-key header, resolve AAPL, fetch the latest 10-K, extract Item 1A, pull annual statements, and return a compact report with every tool call, requestId, source accession number, freshness metadata, and any billing or quota warning.
</Prompt>

## Evaluation path

<Steps>
  <Step title="Fetch live install metadata">
    Do not rely on old local notes. Read the deployed MCP metadata before configuring a client.
  </Step>
  <Step title="Run one evidence-preserving workflow">
    A useful MCP smoke test resolves an issuer, retrieves a filing, extracts a section, and returns citations.
  </Step>
  <Step title="Add client-specific guardrails">
    Store the API key in environment variables, not in checked-in MCP config. Require the agent to preserve request metadata.
  </Step>
</Steps>
