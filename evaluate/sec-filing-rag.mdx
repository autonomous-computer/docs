Evaluate filing RAG as a retrieval and verification problem, not a question of
whether a model can summarize a long document. A useful test returns a passage
the reader can locate in the filing, shows what the retrieval system could not
establish, and keeps the final interpretation separate from the quoted source.

## A 30-minute evaluation

<Steps>
  <Step title="Choose one issuer and one question">
    Pick a question that can be checked in a recent filing, such as a new risk
    factor or a change in management's discussion of demand.
  </Step>
  <Step title="Retrieve a cited passage">
    Use semantic or section search with an issuer and form constraint. Preserve
    `accession`, `section_key`, `source_url`, and any available offsets or
    highlighted snippet.
  </Step>
  <Step title="Test a retrieval failure">
    Try a query with an ambiguous term. If the response reports a degraded
    citation or vector-search state, keep the filing identity and inspect the
    source instead of manufacturing a quote.
  </Step>
</Steps>

## Requests to run

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/search/semantic?q=supply-chain%20risk&ticker=TSLA&form=10-K&mode=hybrid&limit=5&view=agent"

curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/filings/latest/sections/item_1a?ticker=TSLA&form=10-K&mode=compact&view=agent"
```

For cost observability, inspect the response headers. `SECAPI-Estimated-Cost`
and token-count headers are headers, not JSON fields. Request exact token
counting with `SECAPI-Compute-Headers: token-count` when the route supports it.

## Prompt for an analyst or agent

<Prompt>
For TSLA, retrieve cited passages about supply-chain risk from recent 10-K
filings. Return accession, section key, source URL, highlighted snippet, and

interpretation, and say what still requires opening the filing.
</Prompt>
