---
title: "Evaluate SEC Filing RAG"
description: "Test cited filing retrieval before permitting generated answers."
---

Use this guide for an application that must answer a filing question with inspectable source sections. The evaluation is about retrieval and attribution first. It does not establish that ranked results are complete, material, or sufficient for a legal, accounting, or investment conclusion.

## The buyer job

For one real question, retrieve candidate sections, open the section selected for an answer, and keep retrieval evidence separate from the application's interpretation. Use direct EDGAR when a reviewer must browse the original filing without a retrieval layer.

## Smallest proof of concept

Run semantic retrieval for a conceptual question. Replace the placeholders in the section request with returned values, then repeat the test with full-text search for an exact phrase. The two methods are not expected to rank the same way.

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/search/semantic?q=regulatory%20risk&ticker=TSLA&form=10-K&mode=hybrid&limit=5"

curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/ACCESSION/sections/SECTION_KEY?ticker=TSLA&form=10-K"

curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/search/fulltext?q=regulatory%20risk&ticker=TSLA&form=10-K&limit=5"
```

## What a passing result contains

For each generated assertion, retain the retrieved language, accession number, section key, filing URL, form, filing date, available character range, `requestId`, and returned provenance/freshness. If the response supplies `_citation_degraded`, preserve it with the result. A user must be able to distinguish a source quotation from your application's interpretation.

## Source and production limits

- Search narrows a corpus; it does not prove completeness, absence, or relevance to a decision.
- A snippet without a fetched source section is not enough to quote or substantiate a claim.
- A semantic or full-text dependency can be unavailable independently. Surface that condition as failed retrieval, not as an absence of disclosure.
- Citation spans and snippets can be unavailable or degraded; require a retrieval fallback in the answer policy.
- Bound the corpus, result count, and retries, then quote the planned loop through [Pricing and limits](/evaluate/pricing-and-limits).

## Next decision

Permit answer generation only when every material filing assertion has a fetched section and stable source identity. Keep uncited retrieval results in a research queue rather than presenting them as answers.
