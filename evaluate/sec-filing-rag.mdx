Evaluate retrieval before generation. The concrete job is to retrieve candidate filing sections for one research question, fetch the selected source section, and verify that a downstream answer can distinguish source text from interpretation.

## Endpoints and boundaries

- `GET /v1/search/semantic` finds section matches for an open-ended query. It supports `keyword`, `semantic`, and `hybrid` modes; hybrid is the default.
- `GET /v1/filings/{accession_number}/sections/{section_key}` retrieves the named source section for inspection or citation.
- `GET /v1/search/fulltext` is the separate lexical retrieval path when exact terms matter.

Search ranking narrows a corpus; it does not prove completeness, materiality, or a legal conclusion. Citation spans and snippets can be unavailable. Check `_citation_degraded` when present and preserve accession plus section identity.

## Proof of concept

```bash
# Retrieve candidate filing sections for a real research question.
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/search/semantic?q=regulatory%20risk&ticker=TSLA&form=10-K&mode=hybrid&limit=5"

# Fetch the cited source before quoting or summarizing it.
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/ACCESSION/sections/SECTION_KEY?ticker=TSLA&form=10-K"
```

Replace `ACCESSION` and `SECTION_KEY` with values returned by the first request. Repeat with `/v1/search/fulltext` using an exact phrase, then compare the evidence rather than expecting both retrieval methods to rank identically.

## Expected evidence

Keep the retrieved text with accession, section key, source URL, form, filing date, available character range, `requestId`, and returned provenance/freshness metadata. Require generated output to separate:

1. The cited filing language.
2. The filing and section identity.
3. The application's interpretation.

## Failure modes

- Invalid or missing `q`, filters, or semantic mode is a request problem, not an empty search result.
- An unavailable full-text backend or vector dependency can return `503`; surface it as unavailable retrieval rather than an invented absence of disclosure.
- A highlighted snippet without a retrievable source section is insufficient evidence for a quote.
- Repeated semantic retrieval has its own meter class. Check [Pricing and limits](/evaluate/pricing-and-limits) before an agent loop or corpus-wide evaluation.
