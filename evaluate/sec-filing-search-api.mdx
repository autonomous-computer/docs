---
title: "Evaluate SEC Filing Search API"
description: "Test a reviewable path from issuer filters to a specific SEC filing and source section."
---

Use this evaluation when a developer or research buyer needs to decide whether SEC API can supply the filing-discovery layer for a product. The filing endpoint returns a constrained document manifest. It does not determine what a filing says, whether a disclosure is material, or whether the returned list establishes EDGAR coverage.

## Reader job: find the right document before reading it

Start with an issuer, form family, and time boundary your users actually need. Select a filing from the manifest, then follow its accession number into section retrieval. This gives a reviewer a repeatable route from an application finding back to a filed document.

## Run a three-request proof

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings?ticker=AAPL&forms=10-K,10-Q&date_from=2024-01-01&date_to=2025-12-31&limit=10&sort=filing_date_desc"

curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/search/fulltext?q=material%20weakness&ticker=AAPL&form=10-K&limit=10"

curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/ACCESSION/sections/SECTION_KEY?ticker=AAPL&form=10-K&mode=full"
```

The manifest query answers which filings qualify. The full-text query is optional when the question is an exact phrase; use `GET /v1/search/semantic` instead for related language. Replace the final request placeholders only with fields from a returned filing or section result.

## Capability boundary

`GET /v1/filings` accepts issuer identifiers, `form` or `forms`, filing date range, filing or fiscal-year selectors, text `q`, accession number, `limit`, and `cursor`. Its response carries filing records plus `hasMore` and `nextCursor`. A cursor belongs to its original filter set; do not reuse it after changing the query.

For a selected filing, retain issuer identity, form, filing date, accession number, filing URL, request filters, and `requestId`. For a content finding, also retain the section key, returned source text, and available provenance or freshness fields. The filing manifest alone is not a passage-level citation.

## What to test before rollout

- Verify that the identifiers and form family you use resolve to the intended issuer and documents.
- Test the form-only latest path separately if your product uses it; a current daily-index response can be terminal and report `hasMore: false`.
- Treat empty results, invalid filters, authorization failures, and unavailable search dependencies as different application states.
- Use direct EDGAR when your team intends to own collection and document parsing. Use the accession handoff when you want SEC API to supply a documented filing and section workflow.

## Decision point

Adopt the discovery path when your interface can show which filters selected a filing and can retain that filing’s identity beside downstream claims. Before extending the test to a watchlist or historical corpus, price the actual request volume and cadence.
