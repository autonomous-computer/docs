Use this guide when a research or engineering team needs to locate a filing, retrieve the relevant source text, and show a reviewer exactly what it relied on. It is not a claim that search covers all EDGAR history or that a search result answers a disclosure question.

## The buyer job

Build one reviewable path from an issuer and form to a filing section. Raw EDGAR is appropriate when your team wants to operate the document collection and parsing layer itself. This API is a fit when the application needs a filtered filing manifest and an accession-number handoff to content retrieval.

## Smallest proof of concept

Use one issuer and one question your users actually ask. The first request returns filing identity; the second is a literal text search; the final request retrieves the selected section. `ACCESSION` and `SECTION_KEY` must come from the preceding results.

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings?ticker=AAPL&form=10-K&sort=filing_date_desc&limit=1"

curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/search/fulltext?q=material%20weakness&ticker=AAPL&form=10-K&limit=10"

curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/ACCESSION/sections/SECTION_KEY?ticker=AAPL&form=10-K"
```

Use `/v1/search/semantic` separately when the test is conceptual rather than an exact phrase. Do not combine pagination cursors after changing filters.

## What a passing result contains

Save the issuer identity, form, filing date, accession number, filing URL, selected section key, returned source text, `requestId`, and any returned `provenance` or `freshness`. A reviewer should be able to open the filing, reproduce the filters, and see why that section was selected.

## Source and production limits

- A missing or empty result is a retrieval outcome, not evidence that the issuer made no disclosure.
- Full-text and semantic ranking serve different jobs; neither is a completeness or materiality determination.
- A snippet is not a citation until the application retrieves and retains the underlying section.
- Treat validation errors and unavailable search dependencies differently from empty results. Keep the request ID when escalating either case.
- Before moving from one issuer to a watchlist or corpus, quote the actual request path and cadence with [Pricing and limits](/evaluate/pricing-and-limits).

## Next decision

Adopt the path only when your product can retain the filing and section identity beside every displayed finding. Otherwise, use direct EDGAR retrieval or keep the workflow as assisted search rather than evidence-backed output.
