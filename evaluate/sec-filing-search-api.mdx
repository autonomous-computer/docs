Evaluate one issuer and one research question. A useful proof of concept proves that your application can find a filing, preserve its SEC source, and retrieve the exact section it will show a user.

## Run a source-linked proof

<Steps>
  <Step title="Choose one issuer and one question">
    Start with a known issuer, filing form, and question that needs a source-backed answer.
  </Step>
  <Step title="Find the filing record">
    Retrieve the filing manifest and retain its accession number before constructing a retrieval URL.
  </Step>
  <Step title="Compare literal and related-language search">
    Run full-text and hybrid semantic search, then inspect the result type and cited section before treating either as evidence.
  </Step>
  <Step title="Verify the cited source">
    Open the returned filing section and keep its identity and request metadata with any conclusion.
  </Step>
</Steps>

```bash
# Find a recent annual report.
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings?ticker=AAPL&form=10-K&sort=filing_date_desc&limit=1"

# Search filing content for a question.
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/search/fulltext?q=material%20weakness&ticker=AAPL&form=10-K&limit=10"

# Search related language and inspect its cited section.
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/search/semantic?q=artificial%20intelligence%20regulation&ticker=AAPL&form=10-K&mode=hybrid&limit=10"
```

For each result, retain accession number, form, filing date, source URL, endpoint, `requestId`, and route-specific provenance or freshness metadata.

## What success looks like

- The integration scopes issuer and form explicitly, then follows the returned accession rather than guessing a filing URL.
- It distinguishes full-text's filings-and-sections response from semantic and section search result lists.
- It verifies an excerpt in the source filing before presenting a conclusion.
- It handles `400`, `429`, and transient server errors as distinct operational states.

<Prompt>
