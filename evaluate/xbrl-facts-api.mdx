## What to test

- Normalized all-statements retrieval with `GET /v1/statements/all`
- Tag-level facts with `GET /v1/facts`
- Period alignment across annual and quarterly filings
- Provenance fields that connect values back to filings
- Token and cost headers for repeated model-refresh jobs

## Proof-of-concept prompt

<Prompt>
Evaluate SEC API XBRL facts for a financial-model refresh. Resolve MSFT, fetch the latest annual all-statements bundle, pull Revenues facts for the last four annual periods, and return each value with unit, period end, accession number, source filing URL, requestId, freshness, and any caveat about normalization.
</Prompt>

## Evaluation path

<Steps>
  <Step title="Compare statements and raw facts">
    Pull both normalized statements and a tag-level fact series for one issuer. Confirm the reconciliation trail is good enough for your analyst or agent.
  </Step>
  <Step title="Exercise edge cases">
    Test a company with segment disclosures, amended filings, or extension tags before judging the integration complete.
  </Step>
  <Step title="Budget the refresh loop">
    Estimate cost for the issuer universe and cadence you actually need. Agents should inspect billing and limits before wide fanout.
  </Step>
</Steps>
