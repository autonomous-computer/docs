title: "Evaluate XBRL Facts API"
description: "Run a source-aware financial-model refresh proof using normalized statements and SEC XBRL facts."
---

Use this guide when a model or research table needs normalized statements, tag-level facts, and filing context for reconciliation. SEC API supplies statement bundles and facts with response metadata. It does not replace issuer-specific accounting judgment, taxonomy review, or a restatement policy.

## Test one model input end to end

Refresh one issuer's annual inputs and reconcile a statement line to a fact series. Start with a period your reviewer already understands. If your team needs to own taxonomy parsing and normalization, raw SEC XBRL may be the better fit.

## Retrieve the statement and fact record

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/statements/all?ticker=MSFT&period=annual&limit=1"

curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/facts?ticker=MSFT&tag=Revenues&form=10-K&limit=4"
```

For a selected row, compare form, fiscal labels, `periodEnd`, `filedAt`, unit, and the requested and resolved tag before treating values as reconcilable. Values are raw units; any display conversion belongs in an explicit model layer.

## Preserve the reconciliation record

For every model input, store the requested tag, `resolvedTag`, value, unit, form, fiscal period, period end, filing date, and available provenance or filing URL. Store `accessionNumber` when present, plus enclosing `requestId`, `traceparent`, and freshness fields when supplied. An absent accession can be valid; do not create an identifier your response did not return.

## When to use this layer

Use SEC API when a reusable normalized layer and an evidence record are the practical need. Use a direct SEC-data pipeline when you need complete control of taxonomy handling or issuer-specific transformations. In either model, do not assume a missing tag proves that a company did not disclose the metric, or that annual and quarterly values are interchangeable.

## Test the exceptions before rollout

- Reconcile units, form, period basis, filing choice, and restatement treatment before diagnosing a mismatch.
- Inspect issuer extensions and the resolved concept rather than assuming common tags mean the same thing across issuers.
- Keep amendments and restatements in an explicit selection policy.
- Cache historical observations and bound refresh concurrency for the issuer cadence you actually operate.

## Next step

Adopt the layer for a model only after a reviewer can reconcile one displayed value to its fact series and document the exception policy for covered issuers.
