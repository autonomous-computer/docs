Use this evaluation when you are deciding whether SEC API can support a real
financial-model refresh, research agent, or company-data product. The test is
not “did I receive JSON?” It is whether the response gives you a usable value,
a period you understand, and a path back to the filed source.

## A 30-minute evaluation

<Steps>
  <Step title="Choose one issuer and one question">
    Start with an annual revenue, margin, or cash-flow question for an issuer
    your team already understands. Record the filing you expect to reconcile.
  </Step>
  <Step title="Compare a statement and a fact series">
    Request the annual all-statements bundle, then request the underlying tag
    series. Compare the period, unit, and citation fields before comparing the
    values.
  </Step>
  <Step title="Test a boundary">
    Try a quarterly period, a foreign private issuer, or a company with a
    known segment disclosure. Capture what requires filing judgment rather
    than treating an empty or partial response as a product verdict.
  </Step>
</Steps>

## Requests to run

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/statements/all?ticker=MSFT&period=annual&limit=1"

curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/facts?ticker=MSFT&tag=Revenues&period=annual&limit=4"
```

For a company briefing, add a scoped enrichment request rather than assuming
every enrichment is mandatory:

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/companies/overview?ticker=MSFT&include=segments,dilution"
```

## What good evidence looks like

- The fact response tells you the requested tag, the resolved tag, and the
  alias strategy.
- The values carry a period and unit appropriate to the question.
- You retain accession number and filing URL from provenance where supplied.
- You know whether the response shape is compact or the default metadata-rich
  view.
- A batch or optional company enrichment is checked for a partial or degraded
  result before it enters a model or agent answer.

## Prompt for an analyst or agent

<Prompt>
For MSFT, retrieve the latest annual all-statements bundle and a four-period
annual Revenues fact series. Compare the requested and resolved fact tag,
then return the values with period, unit, accession number, filing URL, and
request ID. Identify anything that needs the source filing or an accounting
judgment before it is used in a model.
</Prompt>
