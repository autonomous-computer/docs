Use this guide to build a queue of filed insider-ownership records for human or policy review. The output is disclosure evidence, not a trading signal, legality assessment, statement of motive, 10b5-1-plan status, or current-ownership determination.

## The buyer job

For one issuer, retrieve a bounded Form 4 queue, preserve each row's filing identity, and inspect the source filing before applying a label or action. Direct filing retrieval is appropriate when you only need an original form; this route is appropriate when you need filterable normalized records first.

## Smallest proof of concept

Start with one issuer and one form. Use `transaction_code` only to preserve the SEC-reported code, not to infer a discretionary trade. Continue pagination only with `nextCursor` while `hasMore` is true.

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/insiders?ticker=AAPL&forms=4&transaction_code=P&limit=10"
```

The route also accepts issuer `cik`, owner, security-title, and filing-date filters. `date_from` and `date_to` apply to filing date, not transaction date.

## What a passing result contains

For each queue row, retain issuer, reporting owner, form, filing date, transaction date, transaction code, security title, shares, price/value when returned, accession number, filing URL, `requestId`, and provenance/freshness. The test passes when a reviewer can open the cited filing and see the transaction and its context.

## Source and production limits

- `P` and `S` are reported Form 4 transaction codes. Options, gifts, tax withholding, planned sales, and other context can change the interpretation.
- Filing-derived data does not establish motive, legality, investment outcome, or a complete current holdings picture.
- An empty result is not proof that an issuer or owner had no relevant activity.
- Cursor pagination must use the returned cursor; do not synthesize offsets.
- Inspect footnotes and the cited filing before classifying a row, then quote the intended watchlist cadence through [Pricing and limits](/evaluate/pricing-and-limits).

## Next decision

Deploy this as a review queue only when each downstream label links back to the filing and your policy explicitly separates SEC codes from inferred behavior.
