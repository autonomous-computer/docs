## What to test

- Recent Form 4 transactions for one issuer
- Purchase versus sale filtering
- Insider identity, title, transaction code, share count, price, and post-transaction ownership
- Filing citation fields and amendment behavior
- Monitoring cadence for two-business-day reporting windows

## Proof-of-concept prompt

<Prompt>
Evaluate SEC API insider transaction data for a portfolio monitor. For NVDA and JPM, fetch recent insider transactions, separate open-market purchases from sales and option-related transactions, and return a concise table with insider name, title, transaction date, shares, price, remaining ownership, accession number, filing URL, requestId, and caveats.
</Prompt>

## Evaluation path

<Steps>
  <Step title="Start with one issuer and one filter">
    Test purchase-only and all-transaction views so your workflow does not mistake routine compensation activity for open-market buying.
  </Step>
  <Step title="Preserve the filing trail">
    Require accession number and filing URL in the downstream artifact. Insider signals without source context are too easy to misuse.
  </Step>
  <Step title="Budget recurring monitors">
    Check [Pricing and limits](/evaluate/pricing-and-limits) before running broad watchlists.
  </Step>
</Steps>
