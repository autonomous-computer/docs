## What to test

- Manager holdings retrieval through ownership endpoints
- Quarter-over-quarter comparison behavior
- Identifier normalization across manager CIK, issuer CUSIP, ticker, and CIK
- Filing-date versus report-date handling
- Alert or polling cadence for quarterly monitor jobs

## Proof-of-concept prompt

<Prompt>
Evaluate SEC API 13F coverage for a manager-monitor workflow. Pull Berkshire Hathaway's latest reported holdings, compare the latest two available quarters if supported by the endpoint, and return added, reduced, increased, and removed positions with manager CIK, issuer identifiers, report period, filed date, accession number, requestId, and source URL.
</Prompt>

## Evaluation path

<Steps>
  <Step title="Pick a real manager">
    Use a manager you already track, not a generic demo account. Confirm holdings, identifiers, report period, and filing date match your expectations.
  </Step>
  <Step title="Check comparison semantics">
    Verify how new, closed, increased, and reduced positions are represented before relying on downstream signals.
  </Step>
  <Step title="Connect to an agent workflow">
    Hand the result to a monitor agent and require the final answer to preserve source filings and request metadata.
  </Step>
</Steps>
