Use this checklist to test whether the 13F API fits a manager-monitor or research workflow. The useful evaluation result is not a broad coverage claim: it is a selected manager report with source identifiers plus a separate latest-two filing-date row diff that your application can handle.

## Run two contract checks

Retrieve a manager's selected report:

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/owners/13f?cik=0001067983&limit=20"
```

Compare the latest two parsable reports:

```bash
curl -X POST \
  -H "content-type: application/json" \
  -H "x-api-key: $SECAPI_API_KEY" \
  -d '{"cik":"0001067983","limit":50}' \
  "https://api.secapi.ai/v1/owners/13f/compare"
```

For `GET /v1/owners/13f`, confirm that your application keeps the manager identity, report date, filing date, accession number, filing URL, provenance, and request ID. For `POST /v1/owners/13f/compare`, confirm the two filing dates, returned row statuses and deltas, provenance, and request ID.

The comparison response is a latest-two filing-date row diff. It does not directly provide an accession number or report date for each compared report. Use `GET /v1/owners/13f/filings` to identify available filings, then `GET /v1/owners/13f` with `reportDate` or `filingDate` to retrieve a selected report and its identifiers.

## What to verify

<Steps>
  <Step title="Select a known manager">
    Verify one selected report's identifiers against its source filing.
  </Step>
  <Step title="Choose the report before comparing">
    List filings with `GET /v1/owners/13f/filings` before testing a specific report date.
  </Step>
  <Step title="Keep comparison scope explicit">
    Confirm that the comparison returns the latest two parsable reports' filing dates and row diff, not a user-selected pair or report identifiers.
  </Step>
  <Step title="Test the operational boundary">
    Label shares and value as reported 13F fields, then test an invalid CIK so an error cannot become a no-change signal.
  </Step>
</Steps>

## Evaluation prompt

<Prompt>
Use `GET /v1/owners/13f/filings` and `GET /v1/owners/13f` to retrieve a selected manager report with its report date, filing date, accession number, and filing URL. Use `POST /v1/owners/13f/compare` only for the latest-two filing-date row diff: return its filing dates, row status, deltas, provenance, and request ID. State that the comparison is a delayed, limited disclosure and does not prove the manager's current or complete portfolio.
</Prompt>

## Limits to keep after evaluation

Form 13F is periodic and has a defined reporting scope. It does not expose every short, derivative, cash balance, liability, non-reportable asset, or trade after the report date. A comparison row is a useful lead for filing review, not an explanation of manager intent.
