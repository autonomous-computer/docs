---
title: "Connect SEC API to an MCP client"
description: "Add the hosted SEC API MCP server to your IDE, agent framework, or assistant and verify one source-backed filing workflow."
---

SEC API's hosted MCP server uses streamable HTTP. Give your client the endpoint and an API key, then verify that it can resolve an issuer and retrieve a filing before you rely on it for an agent workflow.

```text
URL:    https://api.secapi.ai/mcp
Header: x-api-key: $SECAPI_API_KEY
```

Pick your client below. Each install page includes a verification prompt so you can confirm the server is connected before an agent starts making broader requests.

## IDE-native MCP clients

- [Claude Desktop](/mcp-install#claude-desktop) — config in `claude_desktop_config.json`
- [Claude Code](/mcp-install#claude-code) — single `claude mcp add` command
- [Cursor](/mcp-install#cursor) — config in `.cursor/mcp.json`
- [Windsurf](/clients/windsurf) — config in `~/.codeium/windsurf/mcp_config.json`
- [Continue](/clients/continue) — config in `~/.continue/config.yaml`

## Agent frameworks

- [LangChain](/clients/langchain) — Python and JavaScript via `langchain-mcp-adapters`
- [LlamaIndex](/clients/llamaindex) — Python via `llama-index-tools-mcp`
- [Vercel AI SDK](/clients/vercel-ai-sdk) — TypeScript via `@ai-sdk/mcp`

## Hosted assistants

- [Custom GPT](/clients/custom-gpt) — ChatGPT Apps (Developer Mode) plus a classic OpenAPI Action fallback for free-tier users

## Don't see your client?

Any client that speaks MCP `2025-06-18` over streamable HTTP can use `https://api.secapi.ai/mcp` with the `x-api-key` header from `SECAPI_API_KEY`. The full reference configuration is in [MCP Server Installation](/mcp-install).

Create an account at secapi.ai/signup to get an API key, or use your existing SEC API key with these clients.
