Webhook delivery is an at-least-once delivery surface for customer events. Each delivery is signed, logged, and replayable when the original payload is still retained. We do not promise global ordering; customers should de-dupe with the stable event id.

## Delivery contract

| Metric | Target |
|---|---|
| **Contract** | At-least-once delivery |
| **Ordering** | Best effort within one endpoint or subscription |
| **Replay** | Manual replay for retained delivery attempts |
| **Billing** | Successful customer-visible webhook and stream delivery units use `delivery_events` |
| **Payload signing** | HMAC-SHA256 on every delivery |

## Retry Policy

SEC API records every delivery attempt and exposes manual replay. Automatic retry and endpoint auto-pause are rollout-gated operational features; docs will list exact windows and attempt counts only after the retry worker is enabled in production.

## Signed Payloads

Every webhook delivery includes `x-secapi-signature: t=<unix_seconds>,v1=<hex_hmac>` and `x-secapi-signature-timestamp`. Verify the signature against the exact raw request body before parsing JSON:

```typescript
import { createHmac, timingSafeEqual } from "crypto"

function verifyWebhookSignature(rawBody: Buffer, signatureHeader: string, secret: string): boolean {
  const parts = new Map(signatureHeader.split(",").map((part) => part.split("=", 2) as [string, string]))
  const timestamp = parts.get("t")
  const provided = parts.get("v1")
  if (!timestamp || !provided) return false

  const expected = createHmac("sha256", secret)
    .update(`${timestamp}.${rawBody.toString("utf8")}`)
    .digest("hex")
  return provided.length === expected.length
    && timingSafeEqual(Buffer.from(provided), Buffer.from(expected))
}
```

## Event Types

Use the event catalog endpoint rather than copying a static list:

```bash
curl "https://api.secapi.ai/v1/event_types?status=public_emitting" \
  -H "x-api-key: $SECAPI_API_KEY"
```

Only `public_emitting` entries are public product surface. Reserved event types, including filing and dilution producers, are excluded from public launch claims until their producers are enabled.

## Delivery Audit

Every delivery attempt is logged and queryable:

```bash
# List recent deliveries
curl "https://api.secapi.ai/v1/webhook_endpoints/{id}/deliveries?limit=10" \
  -H "x-api-key: $SECAPI_API_KEY"

# Replay a failed delivery
curl -X POST "https://api.secapi.ai/v1/webhook_endpoints/{id}/deliveries/{delivery_id}/replay" \
  -H "x-api-key: $SECAPI_API_KEY"
```

## Monitoring

Track webhook health via the observability endpoint, using an API key that has the `admin:operator` scope:

```bash
curl "https://api.secapi.ai/v1/observability" \
  -H "x-api-key: $SECAPI_API_KEY"
```

Response includes delivery success rate, average latency, and recent failures.

## Secret Rotation

Rotate your webhook signing secret without downtime:

```bash
curl -X POST "https://api.secapi.ai/v1/webhook_endpoints/{id}/rotate_secret" \
  -H "x-api-key: $SECAPI_API_KEY"
```

The previous secret is invalidated immediately and the new secret is returned once in the response. Update the receiving service before the next delivery arrives.
