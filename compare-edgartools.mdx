Use this page to choose where a filing workflow runs. The decision is whether your team owns local retrieval and parsing in Python, or calls a managed HTTP contract from the application that needs the data.

## Start with one job

Choose a known issuer, form, and question. For example, retrieve AAPL's latest 10-K risk factors and retain the filing that supplied the text.

| Decision criterion | Choose the path that proves it |
| --- | --- |
| Your application is a local Python research workflow and must control its own source retrieval and processing | Evaluate your local workflow against the SEC source record. |
| A service, worker, or non-Python application needs a documented HTTP interface | Evaluate SEC API's filing search and retrieval routes. |
| The result must be traceable to a filing | Keep the accession number, form, filing date, and filing URL with the result on either path. |
| A downstream system needs a specific rendered filing or exhibit | Test that exact representation before adopting either path. |

## Evaluate the managed path

First establish the filing identity, then retrieve only the section your job needs:

```bash
curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings?ticker=AAPL&form=10-K&limit=5"

curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/latest/sections/item_1a?ticker=AAPL&form=10-K"
```

Accept the managed route only when its filing identity, section availability, and response shape meet your application's requirements. Preserve `requestId` and any returned warning or provenance fields with the result.

## Keep the ownership boundary explicit

SEC API provides documented filing search and retrieval routes. It does not make every filing representation, exhibit, or historical record interchangeable. A successful latest-filing request does not prove that a particular source document or arbitrary filing URL is available in the form your workflow requires.

If your team owns the local workflow, it also owns its retrieval, caching, parsing, and operational controls. If it adopts SEC API, it should still retain the source filing identity and keep a direct-source fallback for unqualified filing families.

## Next action

Run the two requests above alongside your current workflow for one issuer and form. Compare the accession number, filing date, section text, missing-data behavior, and evidence record before changing storage or prompts.
