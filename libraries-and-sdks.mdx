## Verify the integration

Use the same small workflow in every runtime. It checks authentication, issuer resolution, filing retrieval, and the evidence fields you will need later.

1. Export `SECAPI_API_KEY` in the service or shell that makes the request.
2. Resolve `AAPL` and retain the returned CIK.
3. Retrieve the latest `10-K` for that issuer.
4. Retrieve Item `1A`, then retain `accessionNumber`, `filingDate`, `filingUrl`, and `requestId` with the extracted result.

The latest filing changes over time. Assert against the response shape and the presence of source identifiers, not against one historical accession number.

## Use the right reliability boundary

SDKs handle documented transient failures for safe reads. Your application should still set its own request deadlines, preserve request IDs, and avoid layering unbounded retries around the client. Only retry a write when your application can safely repeat it.

Read [SDK reliability](/sdk-reliability) for the JavaScript and Python retry policy, or [API conventions](/api-conventions) for authentication, versioning, and response metadata.
