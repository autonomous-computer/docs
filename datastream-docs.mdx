# Datastream Docs

## What this surface is for

- solve investor workflows, not just endpoint puzzles
- copy a prompt into your coding agent and get moving
- understand the best path through the API without reading every contract first

## Start with a workflow

### Issuer research

What you will do:

- resolve the issuer
- pull the latest filing
- extract the sections that matter
- fetch the all-statements bundle

```md
Build an issuer research workflow with Omni Datastream. Resolve AAPL, pull the latest 10-K, extract Item 1A and Item 7, fetch the all-statements bundle, and return a markdown brief with provenance and freshness metadata attached to each section.
```

### Offering monitor

What you will do:

- search recent S-1 and 424B4 filings
- pull the newest accession numbers
- return the issuer, filing family, and prospectus URL

```md
Use Omni Datastream to watch new offerings activity. Search recent S-1 and 424B4 filings, list the newest issuers with accession numbers and filing URLs, and sort the output by filing date in descending order.
```

### Ownership monitor

What you will do:

- inspect institutional investor holdings
- inspect ticker-centric holder graphs
- identify the most important changes

```md
Use Omni Datastream to compare a manager portfolio with the latest holder graph for AAPL. Highlight the biggest positions, the biggest quarter-over-quarter changes, and the source accessions behind each result.
```

### Market utility workflow

What you will do:

- confirm whether a market is open
- fetch a documented index roster
- read the OMNI Volatility Score

```md
Use Omni Datastream to check whether XNYS is open today, fetch the Nasdaq-100 constituent roster, confirm that NVDA is in the index, and return the OMNI Volatility Score for NVDA with the factor breakdown and freshness metadata.
```
