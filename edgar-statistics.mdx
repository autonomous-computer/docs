title: "EDGAR coverage and statistics"
description: "Understand the difference between the SEC EDGAR corpus, the filing records returned by SEC API, and route-specific derived coverage"
---

SEC API uses SEC EDGAR filings as a primary source for filing-oriented results. That does not make every EDGAR record equally searchable, renderable, or available as structured data. This page describes how to establish coverage for a request; it is not a live inventory counter or a service-level commitment.

## Start with the filing record

Use the filings surface to establish the issuer, form, filing date, accession number, and filing URL for the document you intend to use.

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings?ticker=AAPL&form=10-K&limit=5"
```

The returned record is evidence for that filing. A search result does not by itself establish that every exhibit, section, XBRL fact, or derived analysis is available.

## Coverage is route-specific

| Need | Evidence to retain | Boundary to test |
| --- | --- | --- |
| Find a filing | accession number, filing date, filing URL | issuer, form, and requested period |
| Read filing content | selected filing and section identifiers | whether that filing contains the requested content |
| Compare disclosed values | period, unit, `periodBasis`, and provenance when returned | issuer taxonomy and disclosure differences |
| Use a derived result | source fields plus capability, warning, and degradation fields | route-specific availability and methodology |

Generic filing search can identify many EDGAR filing types. Dedicated routes cover only the filing families and representations documented in their individual references. Do not infer a dedicated extraction from the existence of a filing manifest.

## Historical results

The filing and section workflows document a nine-year searchable floor. Completeness checks apply to required filing forms for the two most recently completed years; the current year is an availability floor rather than a completed-year completeness guarantee. Older records may be returned, but formats, extraction density, and derived views vary by issuer and filing.

When history matters, query the requested issuer and period, then retain the returned accession number and source URL. For a material conclusion, verify the underlying filing rather than treating a no-result response as proof that no filing exists.

## What SEC API does not claim here

This documentation does not publish a fixed filing count, universal extraction rate, all-form normalization guarantee, or platform-wide ingestion-latency target. SEC publication, a filing record appearing in an API response, and a derived representation becoming available are separate events.
