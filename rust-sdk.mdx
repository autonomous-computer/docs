<Info>

</Info>

## Typed request builders

Use typed builders for high-frequency paths when you want editor completion and
canonical query names:

```rust
use sec_api_sdk_rust::{
    LatestFilingRequest, LatestSectionRequest, ResolveEntityRequest, ResponseView,
    SecApiClient, SemanticSearchMode, SemanticSearchRequest,
};

let client = SecApiClient::new(std::env::var("SECAPI_API_KEY").ok());

let entity = client
    .resolve_entity_with(&ResolveEntityRequest::new().ticker("AAPL").view(ResponseView::Agent))
    .await?;

let filing = client
    .latest_filing_with(&LatestFilingRequest::new().ticker("AAPL").form("10-K"))
    .await?;

let section = client
    .latest_section_with(&LatestSectionRequest::new("item_1a").ticker("AAPL").form("10-K").mode("compact"))
    .await?;

let semantic = client
    .semantic_search_with(
        &SemanticSearchRequest::new("risk factors")
            .ticker("AAPL")
            .form("10-K")
            .mode(SemanticSearchMode::Hybrid)
            .limit("3")
            .view(ResponseView::Agent),
    )
    .await?;
```

Builders are available for entity resolution, latest filings, latest sections,
and semantic search. Add `extra(key, value)` for newer or less common query
parameters without changing the wire contract.

## HTTP configuration

The Rust SDK uses a 30-second request timeout by default and retries safe read requests (`GET`) on transient network errors, `408`, `429`, `502`, `503`, and `504` responses. Mutating requests are not retried automatically. `SecApiClient::new(None)` reads API-key and bearer-token environment variables, and dashboard/account workflows can set bearer auth explicitly with `with_bearer_token(...)`. Requests include `Accept: application/json` and a `User-Agent` like `secapi-rust/1.0.1` so support and server logs can identify SDK traffic. Tighten the timeout budget for latency-sensitive services with `with_timeout(...)`:

```rust
use sec_api_sdk_rust::SecApiClient;
use std::time::Duration;

let client = SecApiClient::new(std::env::var("SECAPI_API_KEY").ok())
    .with_timeout(Duration::from_secs(10));
```

Customize retry ownership with `with_retry_config(...)`, or disable SDK retries when your service already wraps the client:

```rust
use sec_api_sdk_rust::{RetryConfig, SecApiClient};
use std::time::Duration;

let client = SecApiClient::new(std::env::var("SECAPI_API_KEY").ok())
    .with_retry_config(RetryConfig {
        max_retries: 2,
        initial_backoff: Duration::from_millis(200),
        max_backoff: Duration::from_secs(2),
    });

let no_sdk_retries = SecApiClient::new(std::env::var("SECAPI_API_KEY").ok())
    .without_retries();
```

For proxies, custom TLS, or shared connection pools, pass a configured `reqwest::Client`:

```rust
use sec_api_sdk_rust::SecApiClient;

let http = reqwest::Client::builder()
    .timeout(std::time::Duration::from_secs(15))
    .build()
    .unwrap();

let client = SecApiClient::new(std::env::var("SECAPI_API_KEY").ok())
    .with_http_client(http);
```

Your application needs a compatible direct `reqwest` dependency when it builds the custom client. Chained setters are last-write-wins: calling `with_timeout(...)` after `with_http_client(...)` replaces the custom client, including proxy, TLS, and connection-pool settings. Set timeout on your `reqwest::Client` builder when using `with_http_client(...)`.

## Dilution intelligence

```rust
// Top 10 highest-dilution-risk issuers
let ratings = client.dilution_ratings(&[("overall_risk", "high"), ("limit", "10")]).await?;

// Single-ticker dilution rollup
let score = client.dilution_score(&[("ticker", "BBBB")]).await?;

// ATM offerings only (boolean filter on the events list)
let atms = client.dilution_events(&[("ticker", "BBBB"), ("is_atm", "true")]).await?;

// Coverage + freshness rollup (no agent shape — already compact)
let coverage = client.dilution_coverage(&[("ticker", "BBBB")]).await?;
```

The full surface — `dilution_events`, `dilution_event_detail`, `dilution_warrants`, `dilution_convertibles`, `dilution_rofr`, `dilution_lockups`, `dilution_cash_position`, `dilution_corporate_actions`, `dilution_nasdaq_compliance`, `dilution_ratings`, `dilution_reverse_splits`, `dilution_score`, `dilution_share_float_history`, `dilution_coverage` — mirrors the REST routes 1:1. Pass `("view", "agent")` in any list call (except `dilution_coverage`) for the compact agent-mode shape.
