# Changelog

## Discipline

- every externally visible ship gets a dated entry on the same day it lands in the release branch
- entries should describe the operator-visible outcome, not only the internal ticket title
- any compatibility change should link to the relevant migration guide and mention the pinned `omni-version` expectation
- benchmark, freshness, and coverage claims should only be copied from tracked source-of-truth artifacts
