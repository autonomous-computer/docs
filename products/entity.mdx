Resolve an issuer, manager, fund, or insider identifier to a canonical entity record before you join it to filings, ownership, or market-facing workflows.

## What you can do

- Resolve `ticker` or its `symbol` alias, CIK, FIGI, composite FIGI, share-class FIGI, ISIN, CUSIP, or a name query.
- Request `view=agent` for primary identifiers and match metadata, or `view=compact` for core identity fields.
- Search canonical entities by identifier, ticker, CIK, or name with `q` and optionally narrow by `entity_type`.
- Search the EDGAR entity records separately when that is the dataset your application needs.

## Endpoints

| Method | Path | What it returns |
| --- | --- | --- |
| `GET` | [`/v1/entities/resolve`](/api-reference/entities/get-v1-entities-resolve) | One resolved entity with match information. |
| `GET` | [`/v1/entities`](/api-reference/entities) | A paginated canonical entity search. |
| `GET` | [`/v1/entities/edgar`](/api-reference/entities) | A paginated EDGAR entity search. |

## Example

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/entities/resolve?ticker=AAPL&view=agent"
```

The agent view includes `primaryIdentifiers`, `matchConfidence`, and `matchBasis`. Store the returned CIK with a downstream filing query when exact issuer identity matters.

## Boundaries to keep in view

- At least one supported identifier is required. An empty request is a request error, not an entity search.
- A name, ticker, or identifier match is resolution metadata, not proof of a legal or economic relationship.
- Tickers can be ambiguous or change over time. Use CIK or an accession number for filing-specific work.
- Entity searches use offset cursors. Keep the same query and filters while paging.
