---
title: "Intelligence workflows"
description: "Call company, security, portfolio, watchlist, and research workloads through SEC API intelligence endpoints."
---

Intelligence endpoints return composed, traceable outputs for a defined research job, from a company or security briefing to a portfolio or filing-footnote investigation.

## What you can do

- Request company and security bundles.
- Submit portfolio, watchlist, country-report, macro-exposure, or footnote investigation requests with their documented JSON bodies.
- Submit an intelligence query that returns a completed result immediately or a job that can be polled.
- Preserve traces, source references, freshness metadata, and `requestId` from the response when they are present.

## Endpoints

| Method | Path | What it returns |
| --- | --- | --- |
| `GET` | [`/v1/intelligence/company`](/api-reference/intelligence) | Company context across supported data planes. |
| `GET` | [`/v1/intelligence/security`](/api-reference/intelligence) | A security bundle with snapshot, catalysts, filings, and factor context. |
| `POST` | [`/v1/intelligence/query`](/api-reference/intelligence) | A completed intelligence result or queued job. |
| `GET` | [`/v1/intelligence/query/{jobId}`](/api-reference/intelligence) | A completed result or the current job state. |
| `POST` | [`/v1/intelligence/portfolio`](/api-reference/intelligence) | Portfolio intelligence output. |
| `POST` | [`/v1/intelligence/watchlist`](/api-reference/intelligence) | A watchlist briefing. |
| `POST` | [`/v1/intelligence/country-report`](/api-reference/intelligence/post-v1-intelligence-country-report) | A country intelligence bundle with macro history and forward view. |
| `POST` | [`/v1/intelligence/macro-exposure`](/api-reference/intelligence/post-v1-intelligence-macro-exposure) | Revenue-weighted macro sensitivities for a portfolio. |
| `POST` | [`/v1/intelligence/footnotes/query`](/api-reference/intelligence) | A filing-footnote investigation result or collection. |

## Example

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/intelligence/company?ticker=AAPL"
```

For `POST /v1/intelligence/query`, a `200` is a completed result and a `202` is a queued job. Poll the returned job ID until the request reaches a final result.

## Boundaries to keep in view

- A bundle is a composed research output, not a substitute for reviewing its supporting filings, traces, or source references.
- A queued job can remain in progress, fail, or be absent. Handle `202`, `502`, and `404` explicitly rather than treating them as a completed answer.
- Output coverage depends on the supporting data planes and request body. Do not infer an unavailable signal from an omitted field.
- Use the endpoint-specific API reference for POST request-body fields; this page intentionally does not duplicate their schemas.
