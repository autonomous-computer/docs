---
title: "Ownership disclosures"
description: "Query SEC-derived 13F, 13D/13G, Forms 3/4/5, compensation, and board disclosures through focused REST endpoints."
---

Use ownership endpoints to move from an issuer or manager identifier to the SEC disclosure family that supports a holdings, insider, beneficial-ownership, compensation, or board review.

## What you can do

- Retrieve the latest normalized 13F report for a manager CIK or a quarter-specific institutional ownership extract.
- View institutional holdings by manager or holders by ticker, and compare the latest two 13F reports for a manager.
- Search beneficial-ownership reports across the 13D and 13G form families.
- Search normalized Forms 3, 4, and 5 records with issuer, owner, security-title, transaction-code, and date filters.
- Retrieve compensation records derived from the latest DEF 14A and board records.

## Endpoints

| Method | Path | What it returns |
| --- | --- | --- |
| `GET` | [`/v1/owners/13f`](/api-reference/owners) | The latest normalized 13F report for a manager CIK. |
| `GET` | [`/v1/owners/13f/filings`](/api-reference/owners) | Recent 13F filings and quarter-end report dates for a manager CIK. |
| `GET` | [`/v1/owners/institutional/ticker`](/api-reference/owners) | A ticker-centric institutional holder graph. |
| `GET` | [`/v1/owners/institutional/investor`](/api-reference/owners/get-v1-owners-institutional-investor) | A manager's ranked holdings and quarter-over-quarter changes. |
| `GET` | [`/v1/owners/institutional/extract`](/api-reference/owners/get-v1-owners-institutional-extract) | A quarter-specific institutional ownership extract for a manager. |
| `POST` | [`/v1/owners/13f/compare`](/api-reference/owners) | A comparison of the latest two 13F reports for a manager. |
| `GET` | [`/v1/owners/13d-13g`](/api-reference/owners) | Beneficial-ownership reports in the 13D/13G families. |
| `GET` | [`/v1/insiders`](/api-reference/insiders) | Normalized Forms 3, 4, and 5 records. |
| `GET` | [`/v1/compensation`](/api-reference/compensation) | Compensation records derived from the latest DEF 14A. |
| `GET` | [`/v1/board`](/api-reference/board) | Board records. |

## Example

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/insiders?ticker=AAPL&forms=4&transaction_code=P&limit=25"
```

Keep the form, filing date, filer or owner identity, accession, source URL, and response `requestId` with any downstream alert or report.

## Boundaries to keep in view

- 13F reports are manager disclosures for their reporting period; they are not a real-time ownership ledger.
- Insider records are derived from Forms 3, 4, and 5. Transaction interpretation requires the underlying filing and its context.
- Beneficial ownership is limited to the 13D and 13G families, while compensation is derived from the latest DEF 14A.
- A missing row is not proof of no position, transaction, beneficial owner, compensation arrangement, or board role.
