---
title: "Filing Intelligence"
description: "Use filing events, diffs, transcripts, and guidance as source-linked inputs to a filing research workflow."
---

Filing Intelligence provides derived views over selected filing material. Use those views to find and prioritize source filings, then inspect the SEC document before relying on the result.

## Compare two reporting periods

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/diff?ticker=AAPL&form=10-K&sections=item_1a_risk_factors,item_7_mdna"
```

`ticker` and `form` are required. Without `from` and `to`, the route compares the latest filing with the prior filing of that form. Set `from` and `to` to an accession number or ISO date for an explicit pair. Set `include_hunks=false` when section-level changes are sufficient.

## Query the persisted stores

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/events?forms=8-K,6-K&tickers=AAPL&limit=25"

curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/transcripts?tickers=AAPL&limit=10"

curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/guidance?tickers=AAPL&direction=raised&limit=10"
```

These endpoints support their documented filters and cursor pagination. Keep each result's filing identity and any returned source URL. Guidance results identify the source by `accessionNumber` but do not return a source URL, so use [filing retrieval](/products/filing-search) to fetch the filing record when a derived item affects a decision.

## Limits to keep visible

- A diff covers the selected pair; it is not a complete disclosure history.
- Transcript and guidance output depends on covered filing material. Absence is not evidence that an issuer held no call or gave no guidance.
- Derived events, summaries, and scores are retrieval aids, not legal, accounting, or investment conclusions.
