title: "SEC enforcement actions"
description: "Find official SEC enforcement releases and preserve the source record for screening and diligence"
---

Screen official SEC litigation releases, administrative proceedings, and Accounting and Auditing Enforcement Releases (AAERs) to build a diligence or monitoring queue. The route answers a narrow question: which published actions match this issuer, respondent, conduct category, date range, or reported penalty range?

## Start with a bounded screening query

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/events/enforcement?source_type=litigation_release&violation_type=fraud&date_from=2025-01-01&limit=25"
```

The response is a paginated list of action records. Record the release number, release URL, publication date, source family, `requestId`, and any returned trace, provenance, or freshness fields with the review item. Read the release before drawing a conclusion about facts, respondents, remedies, or legal status.

## Route and inputs

| Method | Path | Useful inputs | Output |
| --- | --- | --- | --- |
| `GET` | [`/v1/events/enforcement`](/api-reference/events/get-v1-events-enforcement) | `query`, `source_type`, `violation_type`, respondent text, ticker, CIK, date range, penalty range, `limit`, `cursor` | Official-source action records and pagination metadata. |

Use `source_type` for one source family and issuer or respondent filters for a known subject. Reuse the returned cursor only while the complete filter set stays unchanged.

## Coverage and evidence boundaries

- Coverage is limited to the three documented SEC source families. No result is not evidence that the SEC has never acted against a person or issuer.
- `publishedAt` is the publication date from the source record; it is not necessarily the date of the alleged conduct, order, settlement, or payment.
- `violation_type` is a normalized classifier, not a legal conclusion. `other` can represent supported actions outside the named categories.
- A reported penalty is extracted data. A missing value does not mean a zero-dollar penalty.

## Next action

Open the returned release, capture the source record with the action ID, and resolve ambiguous issuer or respondent names before linking the result to a filing or ownership workflow.
