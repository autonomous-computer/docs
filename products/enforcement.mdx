title: "SEC enforcement actions"
description: "Search official SEC enforcement releases, proceedings, and AAERs with source-linked filters for research and screening workflows."
---

Use the enforcement endpoint to find official SEC litigation releases, administrative proceedings, and Accounting and Auditing Enforcement Releases (AAERs), then carry the returned release URL into review.

## What you can do

- Search release titles, excerpts, release numbers, and document URLs with `query`.
- Restrict results to an official source family with `source_type`.
- Filter by a normalized `violation_type`, respondent text, issuer ticker or CIK, publication date, or reported penalty range.
- Page through a stable filter set with the returned `nextCursor`.

## Endpoint

| Method | Path | What it returns |
| --- | --- | --- |
| `GET` | [`/v1/events/enforcement`](/api-reference/platform/get-v1-events-enforcement) | A paginated list of official SEC enforcement actions and their source references. |

## Example

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/events/enforcement?source_type=litigation_release&violation_type=fraud&date_from=2025-01-01&limit=25"
```

`publishedAt` is the SEC source publication date. Preserve the release URL, trace reference, provenance, freshness metadata, and `requestId` when they are returned so a reviewer can return to the underlying action.

## Boundaries to keep in view

- `violation_type` is a normalized classifier. It is not a legal conclusion; `other` covers supported actions outside the named categories.
- Extracted penalties are derived fields. A missing penalty is not a zero penalty.
- Results are limited to the three documented SEC source families. A missing result is not evidence that the SEC has never acted against a person or issuer.
- Continue pagination only with the exact `nextCursor` returned for unchanged filters.
