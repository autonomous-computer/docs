---
title: "Filing content search"
description: "Search SEC filing manifests and section text with full-text or semantic retrieval, then verify each result against its cited source."
---

Use content search when you need language inside a filing or section. It narrows the corpus; the filing accession and cited section remain the source you should retrieve before quoting or deciding.

## What you can do

- Run full-text search across filing manifests and section content with a required `q` query.
- Scope retrieval by ticker, CIK, form, accession number, filing date, fiscal year, or filing year.
- Run semantic search in `keyword`, `semantic`, or `hybrid` mode; hybrid is the default.
- Use returned accession, section key, source URL, and available character spans to retrieve the cited section.

## Endpoints

| Method | Path | What it returns |
| --- | --- | --- |
| `GET` | [`/v1/search/fulltext`](/api-reference/search/get-v1-search-fulltext) | Filing and section matches for a text query. |
| `GET` | [`/v1/search/semantic`](/api-reference/search/get-v1-search-semantic) | Section matches with citation fields for semantic retrieval. |
| `GET` | [`/v1/sections/search`](/api-reference/filings) | Filing-section and snippet matches with filing-scoped filters. |
| `GET` | [`/v1/filings/{accession_number}/sections/{section_key}`](/api-reference/filings) | The source section to inspect or cite. |

## Example

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/search/semantic?q=supply%20chain%20risk&ticker=AAPL&form=10-K&mode=hybrid&limit=10"
```

Semantic rows can include `accession`, `section_key`, `source_url`, `char_start`, `char_end`, and `highlighted_snippet`. Preserve those identifiers and `requestId`, then fetch the selected section before presenting text as a quote.

## Boundaries to keep in view

- `q` is required on both routes. Invalid semantic modes are request errors.
- Citation spans and highlighted snippets can be unavailable. Check `_citation_degraded` when it is present and retain the accession and section identity.
- Full-text search requires its configured search backend; semantic search also requires vector-search dependencies. An unavailable backend can return `503` rather than an invented match.
- Search ranking is retrieval assistance, not a statement that the result is complete, material, or legally dispositive.
