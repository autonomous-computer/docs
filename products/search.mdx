---
title: "Filing Content Search"
description: "Search SEC filing content with full-text, semantic, or section retrieval and keep the filing citation with every result."
---

Use full-text search when exact words matter. Use semantic search when related wording matters. Use section search when you want a cited passage from extracted filing sections. Search finds evidence; it does not establish a conclusion.

## Full-text search

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/search/fulltext?q=material%20weakness&ticker=AAPL&form=10-K&limit=10"
```

`q` is required. Full-text search returns separate `filings` and `sections` lists. Scope it with `ticker` or `cik`, `form`, `accession_number`, `date_from`, `date_to`, `filing_year`, or `fy`.

## Semantic search

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/search/semantic?q=supply%20chain%20risk&ticker=AAPL&form=10-K&mode=hybrid&limit=10"
```

`mode` is `keyword`, `semantic`, or `hybrid`; the default is `hybrid`. Semantic result rows carry filing identity and citation fields including `accession`, `section_key`, `source_url`, `ticker`, `char_start`, `char_end`, and `highlighted_snippet`.

The character fields can be `null` when no usable span can be produced. When `_citation_degraded` is present, keep the source identity and retrieve the filing section instead of manufacturing an excerpt.

## Read the source section

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/0000320193-24-000123/sections/item_1a?ticker=AAPL&form=10-K"
```

`view=agent` on semantic and section search reduces response detail but preserves the citation fields. It removes semantic `score` and `retrievalMode`; use the default response when you need those retrieval details.

## Production notes

- An invalid `mode` returns `400`. Full-text and semantic search require `q`.
- Full-text and vector search have separate runtime dependencies. An unavailable backend returns an error; it is not an empty result set.
- Preserve `requestId`, `traceparent`, and source metadata where supplied.
