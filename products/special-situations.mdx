Special Situations is a paid, authenticated API for following filing-derived M&A, tender offers, going-private transactions, spin-offs, activist campaigns, restructurings, and bankruptcies. It keeps the current situation record, the filing trail behind it, and historical digest snapshots distinct so applications can choose the right evidence for the question at hand.

## Discover live and historical situations

- List situations by type, subtype, status, ticker, sector, market-cap bucket, country, announcement date, or update time.
- Filter with `forms` to expand a supported EDGAR form to the situation types it can open or advance, or query one form directly.
- Read a reverse-chronological JSON feed or authenticated RSS feed of situation events.
- Use the calendar for disclosed record dates, votes, expiries, and expected closes; use stats and performance for aggregate counts and closed-situation outcome cohorts.
- List weekly digest issues or retrieve one by number or slug. Issues are immutable publication-time snapshots, not a substitute for current situation detail.
- Request `enrich=false` for a minimal projection when scanning a wide result set.

## Review source-backed detail

Retrieve a situation for its current structured detail and filing timeline, or use the paginated filings route when the timeline is the only required object. The compact summary carries the latest timeline event with rendered Markdown and terms.

For a source-cited research handoff, use the export route to produce a Markdown brief with filing links and accession numbers. The underwriting pack combines canonical detail, the filing timeline, and Copy-for-LLM Markdown in one deterministic, source-cited response. These are research aids: retain the cited filing identities and inspect the original SEC filing for a decision-critical fact.

## Monitor a bounded set

The hosted MCP tool `situations.watch` can create a durable situation monitor for exact situation IDs or filters such as type, status, ticker, sector, and subtype. It supports direct email or an explicitly acknowledged organization-level webhook endpoint or stream subscription subscribed to `monitor.match`. Configure and test that organization destination before creating a webhook-backed monitor; the monitor does not set a separate per-monitor destination. An exact watch includes filing provenance in its `monitor.match` events.

The tool does not imply a separate public watchlist API or an alias for one. Use the documented situation IDs and filters, retain delivery IDs and filing provenance, and make webhook receivers idempotent.

## Authentication and billing

Send an organization API key in the `x-api-key` header. Access is subject to the organization's plan, credits, and applicable request or delivery meter. Check [`/v1/limits`](/auth-and-pricing) and [`/v1/billing/rates`](/auth-and-pricing) before a batch or agent loop, and record returned billing and correlation headers instead of embedding pricing or limits in an integration. `401`, `402`, and `403` require credential, billing, or entitlement resolution; `429` should be retried only with bounded backoff.

## Example

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/situations?types=merger&statuses=pending&limit=25"
```

Follow a returned situation ID with its timeline, export, or underwriting endpoint as appropriate. Retain the filing identities and source references from the response, and use the original filing when a fact will support a decision.

## Boundaries to keep in view

- The authenticated API is separate from public embed routes. The paid RSS feed is also distinct from the capped anonymous embed RSS surface.
- Situations are filing-derived. They do not confirm that a transaction will close or that a disclosed date, term, or status remains current.
- `enrich=false` changes the returned projection, not the billed request class. It is useful for payload size, not quota savings.
- `limit` is 1 through 100. Keep filters unchanged while advancing the returned cursor; a missing situation or timeline is not proof that no related event exists.
- The public Special Situations product is limited to SEC filing-derived data. It does not include broader market, fundamentals, estimates, valuation, or symbol-coverage datasets.
