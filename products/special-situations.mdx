Special Situations is the authenticated API for following filing-derived M&A, tender offers, going-private transactions, spin-offs, activist campaigns, restructurings, and bankruptcies. Use it to monitor the current record and its filing trail while keeping current situations, calendar events, performance cohorts, and immutable weekly digest snapshots distinct.

## Find a current event and its filing trail

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/situations?types=merger&statuses=pending&limit=25"
```

Narrow list results by type, subtype, status, ticker, sector, market-cap bucket, country, announcement date, update time, and supported EDGAR forms. Use the returned situation ID to fetch current detail, its paginated filing timeline, a compact summary, a Markdown export, or an underwriting pack. Record filing identities, source references, update metadata, and `requestId` for a decision-critical record.

| --- | --- | --- | --- |

| `GET` | [`/v1/situations/by-form/{form}`](/api-reference/situations/get-v1-situations-by-form-form) | Start from an EDGAR form that can open or advance a situation. | A filtered situation list. |

| `GET` | [`/v1/situations/feed.rss`](/api-reference/situations/get-v1-situations-feed-rss) | Consume the authenticated paid event feed. | RSS feed. |
| `GET` | [`/v1/situations/calendar`](/api-reference/situations/get-v1-situations-calendar) | Check disclosed record dates, votes, expiries, and expected closes. | Calendar records. |
| `GET` | [`/v1/situations/stats`](/api-reference/situations/get-v1-situations-stats) | Summarize the current population by type, status, sector, or market-cap bucket. | Aggregate counts. |
| `GET` | [`/v1/situations/performance`](/api-reference/situations/get-v1-situations-performance) | Analyze closed-situation outcome cohorts. | Completion and timing metrics. |
| `GET` | [`/v1/situations/issues`](/api-reference/situations/get-v1-situations-issues) | Browse weekly digest snapshots. | Immutable numbered issues. |
| `GET` | [`/v1/situations/{situation_id}`](/api-reference/situations/get-v1-situations-situation-id) | Inspect one current situation and filing timeline. | Current detail. |
| `GET` | [`/v1/situations/{situation_id}/filings`](/api-reference/situations/get-v1-situations-situation-id-filings) | Page only the filing timeline. | Filing timeline records. |
| `GET` | [`/v1/situations/{situation_id}/summary`](/api-reference/situations/get-v1-situations-situation-id-summary) | Retrieve a compact current summary. | Rendered summary, terms, and latest event. |

Use `enrich=false` to request a smaller list projection when scanning a wide set. It changes the returned projection, not the billed request class. `limit` is 1 through 100; advance pagination only with the returned cursor and unchanged filters.

## Freshness, coverage, and decision boundaries

Situations are filing-derived. They do not confirm that a transaction will close or that a disclosed date, term, or status remains current. Digest issues are immutable publication-time snapshots, while current situation detail can change as new filings arrive. Calendar and performance responses answer different questions and should remain separate from current detail in a research record.

The authenticated read API uses an organization API key in `x-api-key`. Watchlist creation and deactivation are organization control-plane actions that use the signed-in Bearer session specified by their endpoint reference; do not use a read key as a dashboard-session substitute. Both are distinct from the anonymous `/v1/embed/situations*` discovery surface: embeds are deliberately capped and projected for public use, and are not a substitute for paid search, exports, archives, underwriting, or watch configuration. The paid RSS feed is likewise distinct from the capped anonymous embed RSS surface.

Coverage is limited to SEC filing-derived situations. TIKR-derived fundamentals, consensus estimates, screener results, and broad symbol coverage are internal-only and are not part of the public API, embeds, SDKs, CLI, MCP tools, or pricing. A missing situation or timeline is not proof that no related event exists.
