Special Situations is the authenticated API for following filing-derived M&A, tender offers, going-private transactions, spin-offs, activist campaigns, restructurings, and bankruptcies. Use it to monitor the current record and its filing trail while keeping current situations, calendar events, and immutable weekly digest snapshots distinct.

## Free public preview

The `/v1/embed/situations` routes are a free, anonymous preview for public sites and browser embeds. Do not send `x-api-key` to these routes:

```bash
curl -sS "https://api.secapi.ai/v1/embed/situations?types=merger&limit=10"
```

Preview reads are CORS-enabled for browser use and return public cache headers. They are intentionally limited to a capped, recent public projection, so a cached response is not a substitute for an authenticated research workflow.

| Use case | Route family | Access |
| --- | --- | --- |
| Public site or lightweight browser preview | [`/v1/embed/situations`](/api-reference/embed) | Anonymous, CORS-enabled, cacheable preview |
| Application, agent, or sustained monitoring workflow | [`/v1/situations`](/api-reference/situations) | Paid API with a server-held `x-api-key` |

To move beyond the preview, create an organization API key and call the `/v1/situations` routes from your server, worker, shell, SDK, CLI, or MCP client. Never put a paid API key in browser-delivered code.

## Find a current event and its filing trail

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/situations?types=merger&statuses=pending&limit=25"
```

Narrow list results by type, subtype, status, ticker, sector, market-cap bucket, country, announcement date, update time, and supported EDGAR forms. Use the returned situation ID to fetch current detail, its paginated filing timeline, a compact summary, a Markdown export, or an underwriting pack. Record filing identities, source references, update metadata, and `requestId` for a decision-critical record.

## Routes for discovery, monitoring, and evidence

| Method | Path | Job | Output |
| --- | --- | --- | --- |
| `GET` | [`/v1/situations`](/api-reference/situations) | Find current situations with filters and pagination. | A filtered situation list. |
| `GET` | [`/v1/situations/feed`](/api-reference/situations/get-v1-situations-feed) | Read reverse-chronological situation events. | JSON event feed. |
| `GET` | [`/v1/situations/feed.rss`](/api-reference/situations/get-v1-situations-feed-rss) | Consume the authenticated paid event feed. | RSS feed. |
| `GET` | [`/v1/situations/calendar`](/api-reference/situations/get-v1-situations-calendar) | Check disclosed record dates, votes, expiries, and expected closes. | Calendar records. |
| `GET` | [`/v1/situations/issues`](/api-reference/situations/get-v1-situations-issues) | Browse weekly digest snapshots. | Immutable numbered issues. |
| `GET` | [`/v1/situations/{situation_id}`](/api-reference/situations/get-v1-situations-situation-id) | Inspect one current situation and filing timeline. | Current detail. |
| `GET` | [`/v1/situations/{situation_id}/filings`](/api-reference/situations/get-v1-situations-situation-id-filings) | Page only the filing timeline. | Filing timeline records. |
| `GET` | [`/v1/situations/{situation_id}/export`](/api-reference/situations/get-v1-situations-situation-id-export) | Prepare a source-cited Markdown handoff. | Markdown brief. |
| `GET` | [`/v1/situations/{situation_id}/underwriting-pack`](/api-reference/situations/get-v1-situations-situation-id-underwriting-pack) | Assemble detail, timeline, and copyable source context. | Underwriting pack. |

Use `enrich=false` to request a smaller list projection when scanning a wide set. It changes the returned projection, not the billed request class. `limit` is 1 through 100; advance pagination only with the returned cursor and unchanged filters.

## Freshness, coverage, and decision boundaries

Situations are filing-derived. They do not confirm that a transaction will close or that a disclosed date, term, or status remains current. Digest issues are immutable publication-time snapshots, while current situation detail can change as new filings arrive. Calendar and performance responses answer different questions and should remain separate from current detail in a research record.

The authenticated API is distinct from public embed routes, and the paid RSS feed is distinct from a capped anonymous embed RSS surface. Coverage is limited to SEC filing-derived situations; it does not include broader market, fundamentals, estimates, valuation, or symbol-coverage datasets. A missing situation or timeline is not proof that no related event exists.
