---
title: "SEC Filing Search"
description: "Find SEC filing manifests by issuer, form, date, fiscal period, accession number, or indexed text."
---

Use filing search to find the filing before you read or extract it. Keep the returned accession number and SEC filing URL; they are the source trail for the next request.

## Find a recent annual report

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings?ticker=AAPL&form=10-K&sort=filing_date_desc&limit=5"
```

The response is a paginated list of filing manifests. A result identifies the filing with `accessionNumber`, `form`, `filingDate`, and a source URL. Use the accession number for an exact lookup or section retrieval.

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/0000320193-24-000123"

curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/0000320193-24-000123/sections/item_1a?ticker=AAPL&form=10-K"
```

## Filter deliberately

| Need                  | Parameters                        |
| --------------------- | --------------------------------- |
| One issuer            | `ticker`, `symbol`, or `cik`      |
| Filing family         | `form` or comma-separated `forms` |
| Filing-date window    | `date_from` and `date_to`         |
| Period or year        | `fp`, `filing_year`, or `fy`      |
| Exact filing          | `accession_number`                |
| Indexed manifest text | `q`                               |

`sort` accepts `filing_date_desc` or `filing_date_asc`. `limit` is 1 through 50. Send `nextCursor` back as `cursor` unchanged, and do not reuse it after changing filters.

## Production notes

- Pass `form=10-K` when that is the filing you need. Plain [latest filing](/api-reference/filings/get-v1-filings-latest) without `form` can return any filing type.
- A missing result is not proof that an issuer made no filing. Check the date, form, and issuer constraints before expanding the query.
- Preserve `requestId` and route-specific provenance and freshness metadata with cached or user-visible conclusions.
- Correct `400` requests instead of retrying them. Honor `Retry-After` on `429`; bound retries for `5xx` responses.
