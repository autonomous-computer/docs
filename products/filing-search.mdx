---
title: "SEC filing search"
description: "Find a reproducible SEC filing manifest and accession number before retrieval, extraction, or citation"
---

Start filing search when the job is to identify the exact SEC filing behind a workflow. It returns a filterable filing manifest and a stable accession-number handoff; it does not interpret the filing or decide which disclosure is material.

## Build a filing manifest

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings?ticker=AAPL&form=10-K&sort=filing_date_desc&limit=5"
```

Record the returned `accessionNumber`, form, filing date, filing URL, request parameters, `requestId`, and any provenance or freshness fields. The accession number is the handoff for filing-specific retrieval or section requests.

## Routes and selection rules

| Method | Path | When to use it | Output |
| --- | --- | --- | --- |
| `GET` | [`/v1/filings`](/api-reference/filings) | Start with issuer, form, date, fiscal-period, accession, or indexed-manifest text filters. | A paginated filing manifest. |
| `GET` | [`/v1/filings/{accession_number}`](/api-reference/filings) | A workflow already selected a filing. | One filing manifest. |
| `GET` | [`/v1/filings/{accession_number}/sections/{section_key}`](/api-reference/filings) | Retrieve a named section from that selected filing. | Section content and filing context. |
| `GET` | [`/v1/filings/latest`](/api-reference/filings/get-v1-filings-latest) | The workflow genuinely needs the current latest matching filing. | One latest filing. |
| `GET` | [`/v1/filings/latest/sections/{section_key}`](/api-reference/filings/get-v1-filings-latest-sections-section-key) | Read a named section from the current latest matching filing. | One latest section. |

The list route accepts ticker or `symbol`, CIK, form, filing dates, fiscal period, filing year, issuer fiscal year, accession number, `q`, sort, `limit`, and `cursor`. `limit` is 1 through 50; the cursor is a zero-based offset. Do not reuse a cursor after changing the filters.

## Evidence and availability limits

Ticker, symbol, and CIK must not conflict. Resolve ambiguous issuer input first. Filing availability follows SEC source material, so a missing result is a retrieval outcome, not proof that an issuer made no filing. A manifest is not filing text: use the accession number and source URL for review, and retrieve the selected section before quoting it.

Use a latest route only for a then-current workflow. For historical work, select a concrete accession. For language inside a filing, continue with [Filing content search](/products/search) and verify each returned section against its source.
