Start here after [your first request](/getting-started) works. Each request is runnable with an API key, but its records depend on current filings and issuer disclosures. Inspect the returned source, period, and request metadata before comparing or summarizing results.

## Compare reported 13F positions

Use a manager CIK to compare disclosed institutional positions:

```bash
curl --fail-with-body -sS -X POST \
  -H "content-type: application/json" \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/owners/13f/compare" \
  -d '{"cik":"0001067983","limit":25}'
```

Expect a comparison of filed reports. This is disclosure history, not a live ownership ledger. Preserve the returned reporting periods and filing references for each result you use.

## Inspect disclosed executive compensation

Retrieve normalized executive-compensation records derived from the latest DEF 14A filing:

```bash
curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/compensation?ticker=AAPL"
```

Expect normalized records from the latest proxy disclosure. Preserve the filing context and fiscal period returned for each record. Compare those disclosed periods and components, not labels alone.

## Find offering-related filings

```bash
curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/offerings?forms=S-1,424B4&limit=10"
```

Expect structured rows matching the bounded form query. An offering-related form is not, by itself, evidence that a transaction completed or has particular terms; follow the returned filing source before making that conclusion.

## Check the account before repeated work

```bash
curl --fail-with-body -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/billing"

curl --fail-with-body -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/limits"
```

These responses reflect the authenticated organization rather than a universal allowance. Before a repeated or potentially billable workflow, use [Agent billing](/agent-billing) to get a quote and define a stop condition. Account changes such as budgets, Checkout, and payment methods require the signed-in dashboard.

For a failed request, use [request diagnostics](/request-diagnostics). For documented endpoint parameters and fields, use the [API reference](/api-reference).
