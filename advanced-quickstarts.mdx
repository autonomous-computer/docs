Start here after your first filing request works. Each example is a real API call, but the returned records depend on issuer disclosures and current data. Retain the response identifier and source fields before you compare or summarize results.

## Compare 13F positions

Use a manager CIK to compare recent disclosed institutional positions:

```bash
curl --fail-with-body -sS -X POST \
  -H "content-type: application/json" \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/owners/13f/compare" \
  -d '{"cik":"0001067983","limit":25}'
```

The comparison describes filed reports, not current ownership. Keep the reporting periods and filing references returned for each position.

## Compare executive compensation

Use an issuer identifier to retrieve the comparison surface:

```bash
curl --fail-with-body -sS -X POST \
  -H "content-type: application/json" \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/compensation/compare" \
  -d '{"ticker":"NVDA","limit":10}'
```

Compare disclosed fiscal periods, not labels alone. Preserve the filing and period fields that establish what each value represents.

## Search offerings activity

This returns structured rows for a bounded form query:

```bash
curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/offerings?forms=S-1,424B4&limit=10"
```

An offering-related form is not, by itself, a conclusion about transaction terms or completion. Follow the returned filing sources before making that claim.

## Inspect billing before repeated work

The public rate catalog and organization-specific state are different surfaces. Read your current billing and limits with the same API key that will make requests:

```bash
curl --fail-with-body -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/billing"

curl --fail-with-body -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/limits"
```

Before a repeated or potentially billable step, use [Agent billing](/agent-billing) to obtain a quote and define a stop condition. Account changes such as budgets, Checkout, and payment methods require a signed-in browser session, not an API key.
