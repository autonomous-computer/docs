# Migrate from FactorsToday

This guide helps FactorsToday users map factor-history, valuation, portfolio, hedge, and factor-screening workflows to SEC API equivalents. SEC API uses its own factor methodology and data plane, with compact REST, SDK, and MCP interfaces designed for agent and LLM applications.

For a detailed comparison of factor construction methodologies, see [Factor Model Methodology](factor-model-methodology).

## Launch coverage posture

SEC API's launch factor set is built around U.S. market, style, sector, and industry factors. Launch factors target a `2010-01-01` history floor, but public 2010+ claims should be made only where row-level coverage, freshness, model version, and methodology metadata prove the factor. Shorter-history or beta factors are labeled in catalog metadata and should not be used for 2010+ coverage claims.

### Where SEC API leads

- **Agent-first response design**: `response_mode=compact` and `include` controls keep factor history, sparklines, valuations, attribution, and hedge payloads small enough for downstream agents.
- **Portfolio and model workflows**: Portfolio analysis, attribution, hedge, optimize, stress-test, and Model Builder factor analysis share one holdings contract.
- **Trust metadata**: Factor responses preserve `requestId`, `traceparent`, freshness, methodology, and materialization metadata so applications can audit stale or partial results.
- **MCP and SDK reach**: Core factor workflows are exposed through REST and hosted MCP, with SDK and CLI methods moving through the current release train.

## Authentication

FactorsToday uses a query parameter API key. SEC API uses the `x-api-key` header:

```bash
# FactorsToday
curl "https://api.factorstoday.com/api/factors/catalog?apiKey=YOUR_KEY"

# SEC API
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/factors/catalog"
```

## Example: factor catalog

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/factors/catalog"

# Filter by category
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/factors/catalog?category=industry"
```

## Example: historical factor returns

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/factors/returns?keys=VALUE,MOMENTUM,QUALITY&lookback=6m"
```

## Example: single-factor history

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/factors/history/VALUE?range=max&response_mode=compact"
```

## Example: factor valuation stocks

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/factors/valuations/stocks?factor=VALUE&signal=tailwind&limit=25&response_mode=compact"
```

## Example: portfolio factor decomposition

```bash
curl -X POST -H "x-api-key: $SECAPI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "holdings": [
      {"symbol": "AAPL", "weight": 0.3},
      {"symbol": "MSFT", "weight": 0.25},
      {"symbol": "JPM", "weight": 0.2},
      {"symbol": "XOM", "weight": 0.15},
      {"symbol": "JNJ", "weight": 0.1}
    ]
  }' \
  "https://api.secapi.ai/v1/portfolio/analyze"
```

## Example: portfolio hedge

```bash
curl -X POST -H "x-api-key: $SECAPI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "holdings": [
      {"symbol": "AAPL", "weight": 0.35},
      {"symbol": "MSFT", "weight": 0.30},
      {"symbol": "NVDA", "weight": 0.20},
      {"symbol": "JPM", "weight": 0.15}
    ],
    "objective": "factor_neutral",
    "constraints": {"maxHedges": 3, "maxPositionWeight": 0.10}
  }' \
  "https://api.secapi.ai/v1/portfolio/hedge?response_mode=compact"
```

## Example: factor rotation strategy

```bash
curl -X POST -H "x-api-key: $SECAPI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"country": "US", "lookback": "6m"}' \
  "https://api.secapi.ai/v1/strategies/factor-rotation"
```
