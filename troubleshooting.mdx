When a request fails, keep the HTTP status, error `code`, `Request-Id`, `traceparent` when present, endpoint, and SDK or client version. Those details turn a vague failure into a request someone can investigate.

## Start with credentials and limits

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/limits"
```

If this fails, resolve the key or account state before debugging a filing, statement, or MCP workflow. Machine API calls use `x-api-key`; dashboard configuration uses a signed-in browser session.

## Read the status and code

| Status | Typical meaning | Next action |
| --- | --- | --- |
| `400` | Invalid query, path value, or JSON body | Compare the request with the endpoint reference and remove unknown fields |
| `401 missing_api_key` | No API key reached the API | Export the key in the calling environment and send `x-api-key` |
| `401 authentication_failed` | The key is invalid, revoked, or belongs to a different organization | Create a replacement key in the dashboard and update the caller |
| `401 missing_browser_session` | A dashboard action was called as a machine request | Complete the action in the signed-in dashboard |
| `402` | Billing, plan, credit, or budget state blocks the request | Inspect billing and limits, then resolve the stated condition |
| `429` | A rate or quota limit applies | Wait for `Retry-After`; do not fan out retries |
| `5xx` | The request could not complete | Retry once with bounded backoff, then retain the request ID |

Read the [error code catalog](/error-code-catalog) for code-specific guidance.

## Check billing and quota state

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/billing"

curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/limits"
```

Stop automated retries for a billing, budget, or quota stop. Use the dashboard to update browser-only billing settings, then retry the original request once the account state is resolved.

## Fix MCP calls

First confirm that your client uses `https://api.secapi.ai/mcp` and sends `x-api-key`. Then refresh the tool list and use the exact tool name it returns.

- JSON-RPC `-32004` indicates a tool timeout; retry with a narrower request after backoff.
- JSON-RPC `-32005` indicates an AI-query quota stop; wait for reset or change the account's quota posture.
- JSON-RPC `-32006` indicates temporary tool protection; retry only after backoff.

Read [MCP installation](/mcp-install) for the connection pattern.

## Contact support with a reproducible request

Include the endpoint and method, request body with secrets removed, HTTP status, error code, `Request-Id`, trace context, and the outcome of `GET /v1/limits` with the same key. Do not include an API key.
