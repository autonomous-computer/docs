## Start in three steps

<Steps>
  <Step title="Resolve an issuer">
    Start with a company you know. Confirm the ticker, CIK, and name before you query filings or financial statements.
  </Step>
  <Step title="Retrieve evidence">
    Pull a filing, section, statement, or ownership record and retain the accession number or source URL with the result.
  </Step>
  <Step title="Choose your interface">
    Use REST to learn the shape, then move to an SDK, CLI, or MCP when it makes the surrounding workflow easier.
  </Step>
</Steps>

Wire SEC API into this project. ## Evaluate SEC API before you commit

If you are comparing SEC APIs, start with one workflow that matters to your team: filing search, 13F monitoring, section extraction, semantic search, or MCP tool use. A useful proof of concept should show the exact endpoint, request ID, accession number, source URL, freshness metadata, pricing posture, and the amount of payload your agent had to read.
