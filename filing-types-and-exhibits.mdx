SEC API can retrieve filing manifests broadly. Purpose-built extraction is narrower: use it only where a public route documents a structured result.

## Start with filing retrieval

Use [filing search](/api-reference/filings/get-v1-filings), [latest filing](/api-reference/filings/get-v1-filings-latest), or an [accession lookup](/api-reference/filings/get-v1-filings-accession-number) for an unfamiliar form.

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings?ticker=AAPL&forms=10-K,10-Q,8-K&limit=25"
```

The result identifies the SEC filing. It does not promise a normalized parser for every form or exhibit.

## Named workflows

| Filing family                  | Supported use                                                                                                      |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------ |
| `10-K`, `10-Q`                 | Generic retrieval and named-section extraction, including supported keys such as `item_1a`, `item_7`, and `item_8` |
| `10-K`, `10-Q`, `20-F`, `40-F` | Facts and statements workflows                                                                                     |
| `8-K`, `6-K`                   | Generic retrieval and persisted filing-event coverage where available                                              |
| `S-1`, `S-1/A`, `424B*`        | Generic retrieval and offerings workflows                                                                          |
| `13F-HR`, `13F-HR/A`           | Institutional ownership workflows                                                                                  |
| `SC 13D`, `SC 13G`, amendments | Beneficial-ownership workflows                                                                                     |
| Forms `3`, `4`, `5`            | Insider-ownership workflows                                                                                        |
| `DEF 14A`                      | Board and compensation workflows                                                                                   |
| `NPORT-P`, `NPORT-EX`          | N-PORT holdings workflows                                                                                          |

## Exhibit boundary

Workflows can read selected exhibits for a specific purpose. SEC API does not publish a general exhibit-enumeration and normalization API for every filing.

- The 13F information table is read inside institutional-ownership workflows.
- Selected EX-99 material can support transcript and guidance extraction.
- XBRL and inline XBRL support facts and statements; this is structured-data extraction, not raw-exhibit delivery.
- For an exhibit without a named workflow, retrieve the filing and follow its SEC filing URL.
