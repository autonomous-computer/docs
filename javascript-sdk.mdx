<Info>
The JavaScript SDK is the fastest way to wire SEC API into TypeScript services, server actions, cron jobs, and product backends without hand-rolling request headers or metadata plumbing.
</Info>

## Install in three steps

<Steps>
  <Step title="Install the package">
    Add the SDK with your package manager of choice.
  </Step>
  <Step title="Set the API key">
    Export `SECAPI_API_KEY` in the runtime where the code will run.
  </Step>
  <Step title="Verify one real investor workflow">
    Resolve an issuer, fetch a filing, and run one semantic investor workflow before you call the integration done.
  </Step>
</Steps>

```bash
npm install @secapi/sdk-js
```

## First useful calls

```ts
import { SecApiClient } from "@secapi/sdk-js"

const client = new SecApiClient({
  apiKey: process.env.SECAPI_API_KEY!,
})

const issuer = await client.resolveEntity({ ticker: "AAPL" })
const filing = await client.agentLatestFiling({ ticker: "AAPL", form: "10-K" })
const riskFactors = await client.agentSection({ ticker: "AAPL", form: "10-K", sectionKey: "item_1a" })
const income = await client.agentStatement("income_statement", { ticker: "AAPL", period: "annual", limit: 3 })
const holders = await client.agentInstitutionalHolders({ ticker: "NVDA", limit: 10 })
const form144 = await client.agentForm144({ ticker: "NVDA", limit: 10 })
const mcpSection = await client.callMcpTool(
  "sections.get",
  { ticker: "AAPL", form: "10-K", sectionKey: "item_1a", mode: "compact" },
  { id: "aapl-item-1a" },
)
```

## Reliability defaults

The JavaScript SDK retries transient failures with exponential backoff and a per-client circuit breaker.

- Auto-retried by default: `GET`, `HEAD`, `OPTIONS`
- Opt-in required: `POST`, `PUT`, `PATCH`, `DELETE`, MCP `tools/call`
- Always retried regardless of method: `429` rate limits, with `Retry-After` honored

Disable SDK retries when your application already owns retry behavior:

```ts
const client = new SecApiClient({
  apiKey: process.env.SECAPI_API_KEY!,
  retry: false,
  telemetry: false,
})
```

Opt into mutating retries only when the operation is idempotent:

```ts
await client.createArtifact(
  { kind: "markdown_bundle", ticker: "AAPL" },
  { retry: { enabled: true, idempotencyKey: "artifact-aapl-markdown-bundle" } },
)
```

See [SDK reliability](/sdk-reliability) for the full retry policy and migration notes.

## Dilution intelligence

```ts
// Top 10 highest-dilution-risk issuers
const ratings = await client.dilutionRatings({ overall_risk: "high", limit: 10 })

// Single-ticker dilution rollup
const score = await client.dilutionScore({ ticker: "BBBB" })

// ATM offerings only (boolean filter on the events list)
const atms = await client.dilutionEvents({ ticker: "BBBB", is_atm: true })

// Coverage + freshness rollup (no agent shape — already compact)
const coverage = await client.dilutionCoverage({ ticker: "BBBB" })
```

The full surface — `dilutionEvents`, `dilutionEventDetail`, `dilutionWarrants`, `dilutionConvertibles`, `dilutionRofr`, `dilutionLockups`, `dilutionCashPosition`, `dilutionCorporateActions`, `dilutionNasdaqCompliance`, `dilutionRatings`, `dilutionReverseSplits`, `dilutionScore`, `dilutionShareFloatHistory`, `dilutionCoverage` — mirrors the REST routes 1:1. Add `view: "agent"` to any list call (except `dilutionCoverage`) for the compact agent-mode shape.

## Error handling

```ts
import { SecApiClient, SecApiError } from "@secapi/sdk-js"

const client = new SecApiClient({
  apiKey: process.env.SECAPI_API_KEY!,
})

try {
  await client.requestDiagnostics("missing-request-id")
} catch (error) {
  if (error instanceof SecApiError) {
    console.error(error.status, error.code, error.requestId, error.message, error.body)
  }
}
```

`SecApiError` normalizes the common SEC API and JSON-RPC error shapes. Use `error.status`, `code`, `requestId`, `message`, and `body` in structured logs so support can connect customer reports to the exact failed request.

<Warning>
Do not drop metadata on the floor. In production, keep `requestId`, `traceparent`, provenance, freshness, and materialization fields in your logs or downstream artifacts so support and audit workflows stay intact.
</Warning>
