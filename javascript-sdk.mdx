---
title: "JavaScript SDK"
description: "Use the SEC API JavaScript client in TypeScript services and server-side applications"
---

Use the JavaScript SDK when SEC API calls belong in a TypeScript or JavaScript service. Keep the API key on the server and preserve filing identifiers with any result your application stores or displays.

## Install and configure

```bash
npm install @secapi/sdk-js
export SECAPI_API_KEY="secapi_..."
```

```ts
import { SecApiClient } from "@secapi/sdk-js"

const client = new SecApiClient({
  apiKey: process.env.SECAPI_API_KEY!,
})
```

Do not expose `SECAPI_API_KEY` to browser code. Call the SDK from a server route, worker, background job, or other trusted runtime.

## Make a first filing request

```ts
const issuer = await client.resolveEntity({ ticker: "AAPL" })
const filing = await client.agentLatestFiling({
  ticker: "AAPL",
  form: "10-K",
})

console.log({
  ticker: issuer.ticker,
  cik: issuer.cik,
  accessionNumber: filing.accessionNumber,
  filingUrl: filing.filingUrl,
  requestId: filing.requestId,
})
```

The `agentLatestFiling` helper returns the endpoint's compact agent projection. Use the full documented method when you need fields outside that projection.

## Extract the source text behind a result

```ts
const riskFactors = await client.agentSection({
  ticker: "AAPL",
  form: "10-K",
  sectionKey: "item_1a",
})
```

Keep the filing accession, URL, and request ID alongside an extracted section or conclusion. This lets a reader trace a result back to the SEC filing that supports it.

## Handle errors

```ts
import { SecApiClient, SecApiError } from "@secapi/sdk-js"

try {
  await client.resolveEntity({ ticker: "AAPL" })
} catch (error) {
  if (error instanceof SecApiError) {
    console.error({
      status: error.status,
      code: error.code,
      requestId: error.requestId,
      message: error.message,
    })
  }
}
```

The SDK retries safe read requests according to its retry policy. Only opt into retries for a mutating call when your operation is idempotent. Read [SDK reliability](/sdk-reliability) and [Troubleshooting](/troubleshooting) before adding application-level retries.
