---
title: "Make your first SEC API request"
description: "Resolve a public company with one API key, verify the response, and continue to a source filing"
---

Start with one small, read-only request: resolve a ticker into SEC API's canonical issuer identity. It proves that your key, base URL, and request format work before you build a filing or research workflow.

## Set your API key

Create an account at [secapi.ai/signup](https://secapi.ai/signup), then create an API key in the signed-in [dashboard](https://secapi.ai/app). Export it in the shell that will make the request:

```bash
export SECAPI_API_KEY="secapi_..."
```

For API-key authentication, send the key in `x-api-key`. An API key is not an `Authorization: Bearer` token; use the API-key header for this guide and for server-to-server requests.

## Resolve an issuer

Resolve Apple before requesting a filing. The response gives you the canonical CIK and identity metadata you can carry into later API calls.

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/entities/resolve?ticker=AAPL&view=agent"
```

## Expected response

The exact `id` and `requestId` vary. A successful response identifies Apple and includes CIK `0000320193`:

```json
{
  "object": "entity",
  "ticker": "AAPL",
  "cik": "0000320193",
  "name": "Apple Inc.",
  "requestId": "req_..."
}
```

`view=agent` is the compact, identifier-focused projection. Use the endpoint's default response when you need its full documented metadata.

## 2. Retrieve a filing you can cite

Once the identity request succeeds, retrieve Apple's latest 10-K with the same key:

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/filings/latest?ticker=AAPL&form=10-K&view=agent"
```

The result changes when Apple files a newer annual report. Preserve the returned `accessionNumber`, `filingDate`, `form`, `filingUrl`, and `requestId` with work derived from it. Those fields identify the specific filing behind a result.

For accepted identifiers, filters, response fields, and endpoint-specific limits, use the [entity resolution reference](/api-reference/entities/get-v1-entities-resolve) and [latest filing reference](/api-reference/filings/get-v1-filings-latest).

## If it does not work

For `401 missing_api_key`, check that the key is exported in the current shell and sent through `x-api-key`. For `401 authentication_failed`, verify the key in the dashboard and update the calling environment. For `402`, address the stated billing or entitlement condition. For `429`, wait for `Retry-After` before retrying rather than sending concurrent retries.
