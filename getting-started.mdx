Make one request you can inspect before building a larger workflow. Entity resolution is a small, read-only call that confirms your key, base URL, and request format before you request filings or statements.

## 1. Create and store an API key

Create an account at [secapi.ai/signup](https://secapi.ai/signup), then create an API key in the signed-in [dashboard](https://secapi.ai/app). Export it in the server, worker, notebook, or shell that will call the API:

```bash
export SECAPI_API_KEY="secapi_..."
```

<Warning>
Send machine API keys only in the `x-api-key` header. Keep them out of browser code, URLs, source control, and logs. An API key is not an `Authorization: Bearer` token.
</Warning>

## 2. Resolve a company

Resolve Apple before requesting a filing. The response gives you the canonical CIK and identity metadata you can carry into later API calls.

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/entities/resolve?ticker=AAPL&view=agent"
```

## Expected response

This shortened, representative excerpt shows the core identity fields. The exact `id` and `requestId` vary. A successful response identifies Apple and includes CIK `0000320193`:

```json
{
  "object": "entity",
  "ticker": "AAPL",
  "cik": "0000320193",
  "name": "Apple Inc.",
  "requestId": "req_..."
}
```

The full `view=agent` response can also include fields such as `id`, `primaryIdentifiers`, `matchConfidence`, and `matchBasis`. Use the endpoint's default response when you need its full documented metadata.

## 3. Retrieve a filing you can cite

Once the identity request succeeds, retrieve Apple's latest 10-K:

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/filings/latest?ticker=AAPL&form=10-K&view=agent"
```

You can also request one normalized annual statements bundle:

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/statements/all?ticker=AAPL&period=annual&limit=1"
```

The filing response changes when Apple files a newer annual report. Statement values and available periods change as issuers file. Preserve `requestId` with every result; for filing-derived work, also retain `accessionNumber`, `filingDate`, `form`, and `filingUrl` when returned.

For accepted identifiers, filters, response fields, and endpoint-specific limits, use the [entity resolution reference](/api-reference/entities/get-v1-entities-resolve), [latest filing reference](/api-reference/filings/get-v1-filings-latest), and [all statements reference](/api-reference/statements/get-v1-statements-all).

## If it does not work

For `401 missing_api_key`, confirm that the key is exported in the current process and sent through `x-api-key`. For `401 authentication_failed`, verify the key in the dashboard and update the calling environment. For `402`, resolve the billing or entitlement condition named in the response. For `429`, wait for `Retry-After` before retrying and avoid concurrent retries.

Read [Troubleshooting](/troubleshooting) for error-specific next steps and [API conventions](/api-conventions) for versioning, pagination, and response metadata.
