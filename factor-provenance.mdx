---
title: "Factor provenance"
description: "Retain the source, methodology, materialization, and limitation evidence needed to interpret factor results"
---

Factor output can fit in a chart or agent response while still requiring an audit trail. Request the trust envelope when a workflow needs to evaluate the result rather than merely display it.

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/factors/returns?keys=VALUE,MOMENTUM,QUALITY&lookback=6m&response_mode=compact&include=trust"
```

## Evidence to preserve

| Field | Use |
| --- | --- |
| `requestId` and `traceparent` | correlate a result with request and downstream tracing evidence |
| `freshness` | evaluate the route's applicable recency state |
| `materialization` | identify the returned read-model or computation version |
| `methodology` | identify model family, inputs, and construction notes when returned |
| `sourceRights` | understand the returned source-rights posture |
| `degradedState` | expose a partial result, fallback, or missing dependency |

The response is authoritative only for fields it returns. Compact mode can omit heavier metadata; `include=trust` requests it explicitly. Treat materialization versions as opaque identifiers and store them: a change can accompany a revision to data or methodology and therefore change historical results.

## Inputs are family-specific

Market factors use SEC API market-data inputs and construction metadata. Style factors use point-in-time returns and public-company characteristics, including SEC-derived fundamentals where the definition calls for them. Sector and industry factors use point-in-time returns and public classification mappings, including SIC-derived groupings. Portfolio attribution and hedge results also depend on caller-supplied holdings.

Catalog membership does not prove that a factor is available on every surface, for every period, or with a particular rights posture. Check the requested row's materialization, history, freshness, and returned source-rights evidence.

## Handle degraded output as data

A degraded response is neither an invisible success nor necessarily a generic failure. It can identify unavailable materialization, incomplete market-bar inputs, missing exposures, or constraints that leave no hedge candidates. Preserve its code, message, retryability, missing capabilities, and fallback indicator when supplied. Show the limitation to downstream users and agents.
