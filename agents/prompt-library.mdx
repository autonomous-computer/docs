Use these prompts with the hosted [SEC API MCP server](/mcp-install) when you need an agent to retrieve, organize, and cite public-company evidence. Replace the bracketed inputs before running one. Each prompt names the relevant tools and asks for the source record that lets another person reproduce the work.

<Note>SEC API returns public filing and market-derived data. A prompt can organize evidence; it cannot determine legal liability, suitability, underwriting terms, or investment merit.</Note>

## Run it with evidence

1. Replace every bracketed input, including the as-of date, issuer list, and output format.
2. Have the agent call `tools.describe` when its client has not loaded the relevant schemas.
3. Require the form, filing date, accession number, section or fact reference, and source URL beside every material statement.
4. Re-open the cited filing before publishing, advising, trading, underwriting, or otherwise acting on a material conclusion.

Start with `entities.resolve`, then retrieve the filing or section that supports the answer. `intelligence.query`, factor analytics, and ownership data can focus a review, but they do not replace the underlying filing.

See [MCP workflows](/mcp-workflows) for the retrieval pattern and [API conventions](/api-conventions) when you need a repeatable REST or SDK workflow.

## CLI access

```bash
secapi agents personas
secapi agents prompts list --persona law-firm
secapi agents prompts read law-firm-enforcement-history
```
