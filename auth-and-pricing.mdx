title: "API keys, usage, and billing"
description: "Set up machine authentication, inspect usage, and choose the right billing path for an SEC API integration"

Use an API key for REST, SDK, CLI, and server-to-server MCP requests. Create and rotate keys in the [SEC API dashboard](https://secapi.ai/app), keep them in server-side environment variables, and send them in `x-api-key`.

## Use the right credential

| Job | Credential | Where to do it |
| --- | --- | --- |
| Read SEC data from code, an SDK, the CLI, or MCP | API key in `x-api-key` | Your server, worker, terminal, or MCP client |
| Create, reveal, rotate, or revoke a key | Signed-in browser session | SEC API dashboard |
| Change billing, payment methods, or budgets | Signed-in browser session with billing access | SEC API dashboard |

```bash
export SECAPI_API_KEY="secapi_..."

curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/limits"
```

Do not put an API key in a browser bundle, URL, request body, source repository, log, or screenshot. API keys are not bearer tokens; do not send them as `Authorization: Bearer`.

## Start with one useful request

New organizations receive the current starter allowance. First, complete the [first request](/getting-started). Then use the dashboard to review the plan, usage, credits, and billing state before you increase automated traffic.

API keys can read account context such as limits, usage, billing state, rates, and a workflow quote:

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/billing"

curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/billing/rates"
```

Checkout, payment-method changes, credit purchases, budget changes, and key management require the dashboard. Keep that browser-authenticated boundary separate from machine requests.

## Read response usage metadata

SEC API includes request and usage metadata on API responses. Preserve the following values in application logs when a request affects a user-visible result:

- `Request-Id` and `traceparent` for support and correlation
- `SECAPI-Meter-Class` for the usage category
- `SECAPI-Estimated-Cost` and `SECAPI-Token-Count` for per-call planning
- `Retry-After` when the service asks a client to wait

Read [API conventions](/api-conventions) for the complete header behavior and [Troubleshooting](/troubleshooting) for billing and quota errors.

## Choose a plan

Use [Plans and pricing](/plans-and-pricing) to compare the current self-serve and commercial options. Contact SEC API before redistributing data, embedding it in a product for third parties, or reselling it; those uses can require commercial rights beyond a standard development integration.

## Rotate a key safely

1. Create a replacement key in the dashboard.
2. Update the server-side secret where your application runs.
3. Verify the replacement with `GET /v1/limits`.
4. Revoke the old key after callers have moved.

For a suspected leak, revoke the affected key first, then create and deploy the replacement.
