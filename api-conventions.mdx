---
title: "API conventions"
description: "Use SEC API versions, response metadata, pagination, and compact views consistently in production clients"
---

SEC API responses carry the identifiers and delivery context you need to reproduce a result. Build your client around those fields from the first integration, rather than trying to reconstruct them from a ticker or timestamp later.

## Send requests consistently

Use an API key in `x-api-key`. Production clients can pin a dated API version with `secapi-version`; unpinned requests use the current published version and return the resolved version in `SECAPI-Version`.

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/filings/latest?ticker=AAPL&form=10-K"
```

Additive fields can appear within a pinned version. Parse fields you use and tolerate additional fields in the response.

## Keep response metadata

Record these values with material results:

| Value | Why it matters |
| --- | --- |
| `Request-Id` | Identifies one API request when you need support or debugging |
| `traceparent` | Correlates the request with downstream trace context when present |
| `accessionNumber` and `filingUrl` | Identify the SEC filing behind a result |
| freshness and provenance fields | Describe the source and currentness of a derived result |
| `SECAPI-Estimated-Cost` and `SECAPI-Token-Count` | Support per-call budgeting |

You may send `x-request-id` to associate a request with your own log or job identifier.

## Choose a response view

Many endpoints support `view=default`, `view=compact`, or `view=agent`.

- `default` includes the full documented payload.
- `compact` keeps a smaller UI-oriented projection.
- `agent` keeps retrieval essentials and citation pointers for an agent or downstream model.

Use the endpoint reference to confirm view support and the exact projected fields. Do not assume that `agent` and `compact` contain the same fields.

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/latest?ticker=AAPL&form=10-K&view=agent"
```

## Page through lists

Paginated endpoints return `hasMore` and `nextCursor`. Send `nextCursor` as the next request's `cursor`; stop when `hasMore` is `false`. Endpoint-specific defaults and maximum `limit` values are documented in the API reference.

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings?ticker=AAPL&limit=25"
```

Some list-shaped responses return the complete set and do not include cursor fields. Treat the endpoint's documented response as authoritative.

## Handle errors and retries

Errors include a machine-readable `code`, a message, and a request ID. Read the response before retrying:

- `400`: correct the path, query, or JSON body.
- `401`: provide a valid `x-api-key` or use the dashboard for browser-only actions.
- `402`: resolve the stated billing or entitlement condition before retrying.
- `429`: wait for `Retry-After`; do not fan out retries.
- `5xx`: retry once with bounded backoff, then retain the request ID for support.

For the complete error catalog, read [Troubleshooting](/troubleshooting). For SDK retry behavior, read [SDK reliability](/sdk-reliability).

## Use statements carefully

Statement periods carry `periodBasis`. Balance-sheet values are point-in-time; quarterly income-statement and cash-flow values can be year-to-date as filed. Check `periodBasis` before comparing a quarterly value with a discrete quarter.

## REST and MCP

Use REST or an SDK when your application needs a defined wire response. Use hosted MCP when an MCP-aware client needs tools as part of an agent workflow. MCP tool calls use JSON-RPC and have their own quota and error envelopes. Read [MCP installation](/mcp-install) before connecting a client.
