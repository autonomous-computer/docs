description: "Qualify a macro workflow on SEC API route by route, with explicit country, series, timing, and availability checks."

Trading Economics integrations often begin with broad country and series discovery. SEC API macro routes should be qualified more narrowly: identify the country, indicator, date range, and decision your workflow needs, then inspect the returned coverage and timing evidence before replacing a production feed.

## Map the workflow

| Existing job | SEC API route | What to validate |
| --- | --- | --- |
| Retrieve an indicator series | `GET /v1/macro/indicators` | Country, indicator key, dates, units, observations, and source metadata. |
| Review scheduled macro events | `GET /v1/macro/calendar` | Country, requested window, event date, and returned coverage or status fields. |
| Inspect forward estimates | `GET /v1/macro/forecasts` | Country, horizon, source, and observation date. |
| Use market context | Relevant documented market route | Instrument identity, market date, units, and route-specific freshness. |

SEC API does not claim every country, series, calendar, or market representation a broad macro provider may offer. A returned response is evidence for that requested scope, not a promise of universal coverage.

## Prove one indicator

```bash
curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/macro/indicators?country=US&indicator=GDPC1"
```

Compare the observation dates, units, revisions if supplied, and the source fields your model or dashboard relies on. Store `requestId`, retrieval time, and any provenance, freshness, coverage, warning, or status fields returned by the route.

## Prove the calendar separately

```bash
curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/macro/calendar?country=US&days=7"
```

A calendar response and an indicator response answer different questions. Do not use an upcoming event to infer that a series value is current, or use a current series value to infer complete event coverage. Make unavailable, tentative, or degraded conditions visible to the caller.

## Production cutover rules

1. Dual-run one country and one indicator for a defined observation window.
2. Test the country and series inventory your users actually request, including gaps.
3. Compare timestamps, units, source records, and revision behavior before joining values to a model.
4. Move one low-risk read path first, with a reversible fallback during qualification.
5. Preserve response diagnostics so a stale or unavailable result can be investigated later.

For portfolio or factor interpretation, keep modeled output separate from the returned macro evidence. A macro observation can inform analysis; it does not prove a forecast or investment conclusion.
