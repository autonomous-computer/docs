## Version policy notes

Behavior rows are measured from the monorepo SDK source. Launch versions are the public registry versions customers install with unpinned commands. Language-specific hotfix rows are still launch-ready only when the release dashboard proves the hotfix version is the registry latest and public conformance passed at that launch version.

All SDK launch versions match the source versions used by the behavior contract.

## Alternate paths

| SDK | Capability | Why this is still launch-ready |
|---|---|---|
| CLI | Retry-After rate-limit handling | CLI delegates HTTP retry policy to the bundled JavaScript SDK client; CLI launch coverage asserts request routing, diagnostics, and command behavior. |
| CLI | Request ID error diagnostics | CLI exposes request diagnostics through commands and JSON stderr summaries rather than typed library exceptions. |
| CLI | Request timeout controls | CLI process invocations use command execution boundaries and the JavaScript SDK client; there is no public per-call timeout API. |
| CLI | Cursor auto-pagination helpers | CLI exposes page-sized commands and machine-readable output; streaming auto-pagination is a library SDK concern. |
| CLI | Explicit zero pagination caps | CLI pagination is controlled by explicit command flags instead of SDK iterator caps. |
| Rust | Explicit zero pagination caps | Rust pagination has explicit max_items zero behavior through PaginationOptions defaults and tests, but does not expose the same literal zero-cap token shape as JS/Python/Go. |
| CLI | Grouped service namespaces | CLI grouping is command-namespace based, not service-object based. |
| CLI | Agent workflow example | CLI ships `secapi examples` and `secapi agent-context` instead of a language-specific agent workflow source file. |
