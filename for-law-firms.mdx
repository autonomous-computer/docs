Use this guide to build a first-pass chronology for issuer, counterparty, or transaction diligence. Keep filing and source identifiers beside each statement. SEC API supports research and retrieval; it does not provide legal advice or replace primary-source review.

<Prompt>
For [TICKER] during [DATE RANGE], resolve the issuer and inventory 10-K, 10-Q, 8-K, and DEF 14A filings. Retrieve the latest 10-K Item 3 and Item 1A, recent Item 5.02 disclosures, and Form 4 activity. Deliver a chronology with form, filed date, accession, section, and short neutral description. Do not call something enforcement history unless an SEC or issuer source expressly says so. Use entities.resolve, filings.search, sections.get, sections.search, and insiders.list. For a date window, MCP `filings.search` and `sections.search` do not filter by date: fetch candidate records, then post-filter the returned `filingDate` locally before selecting sections or reporting results.
</Prompt>

## Assemble the chronology

| Matter task | Route | Required record |
| --- | --- | --- |
| Identify the issuer | `GET /v1/entities/resolve` | Resolved name, ticker, CIK, and query used. |
| Inventory relevant filings | `GET /v1/filings` | Form, filed date, accession number, and filing URL. |
| Read a known section | `GET /v1/filings/latest/sections/{section_key}` | Filing identity, section key, text availability, and warnings. |
| Locate sections by terms | `GET /v1/sections/search` | Candidate filing identities before selecting text for review. |
| Review a documented event record | `GET /v1/events/enforcement` or `GET /v1/events/officer-changes` | Returned record and its source context. |

Search results are candidates, not conclusions. Use the returned filing or release identity to read the underlying source before describing it in a memo.

## Build one issuer timeline

```bash
curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/entities/resolve?name=Tesla"

curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings?cik=0001318605&forms=8-K,10-K,10-Q,DEF%2014A&limit=25"

curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/sections/search?q=legal%20proceedings&ticker=TSLA&form=10-K&limit=5"
```

For a known annual filing section, retrieve the text separately:

```bash
curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/latest/sections/item_3?ticker=TSLA&form=10-K"
```

## Keep the record in bounds

Do not label a result as enforcement history unless the returned SEC record or issuer filing supports that statement. Do not infer liability, materiality, or legal outcome from a search hit. Preserve accession numbers, filing URLs, section keys, dates, `requestId`, and any warnings with the research file.
