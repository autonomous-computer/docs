# Request diagnostics

SEC API exposes request-scoped diagnostics so support can investigate a single workflow without database access.

## Endpoints

- `GET /v1/diagnostics/requests/{request_id}`
- `GET /v1/diagnostics/deliveries/summary`

## Request drilldown

```bash
curl "$SECAPI_BASE_URL/v1/diagnostics/requests/req_123" \
  -H "x-api-key: $SECAPI_API_KEY"
```

The drilldown includes:

- usage events for the request
- canonical events emitted for the request
- related webhook deliveries
- related stream events
- related artifacts
- grouped meter-class and event-type counts
- `summary.support`, a support-ready finding with next actions, latest usage
  status, delivery failure count, and missing artifact IDs

Use this first when support has a `Request-Id` from an API response.
Start with `summary.support.status`:

| Status | Meaning |
|---|---|
| `not_found` | Nothing was found for that request ID in the current org. Check the copied ID and tenant scope. |
| `request_error` | The latest metered response for the request was HTTP 4xx or 5xx. Start with the usage event status, path, and retry guidance. |
| `request_logged` | The request was metered but emitted no downstream events, deliveries, streams, or artifacts. |
| `side_effects_emitted` | The request produced downstream records without obvious delivery or artifact gaps. |
| `delivery_attention` | At least one webhook delivery failed or returned a non-2xx status. |
| `artifact_attention` | A canonical event referenced an artifact that diagnostics could not load. |

## Delivery summary

```bash
curl "$SECAPI_BASE_URL/v1/diagnostics/deliveries/summary?since=2026-03-14T00:00:00Z&limit=200" \
  -H "x-api-key: $SECAPI_API_KEY"
```

This returns grouped webhook and stream delivery activity so support can answer:

- which destinations are failing
- which event types are replaying frequently
- whether stream polling activity is flowing as expected

## CLI

```bash
secapi diagnostics request --request-id req_123
secapi diagnostics deliveries-summary --since 2026-03-14T00:00:00Z --limit 200
```

For support tickets or agent handoffs, generate one redacted bundle instead of
pasting several commands separately:

```bash
secapi diagnostics request --request-id req_123
```

The bundle includes local CLI config, `doctor` checks, request metadata, and the
request drilldown when a request id is provided. Credential values are redacted
recursively before output.
