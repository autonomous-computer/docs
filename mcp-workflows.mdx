# Hosted MCP Workflows

<Info>
Hosted MCP lets coding agents call SEC API directly. That means less glue code, less local wrapper drift, and faster time from prompt to useful investor workflow.
</Info>

## Why use hosted MCP

<CardGroup cols={3}>
  <Card title="Less wrapper code" icon="cable">
    Your agent can call the SEC API tool surface directly instead of waiting on a custom wrapper layer.
  </Card>
  <Card title="Same workflows" icon="equal">
    REST and hosted MCP sit on top of the same underlying SEC API workflows.
  </Card>
  <Card title="Faster setup" icon="rocket">
    A working MCP install gets you to a useful issuer, filing, or section workflow quickly.
  </Card>
</CardGroup>

## Install metadata

```bash
curl https://api.secapi.ai/mcp
```

Inspect the fields your client will use:

```bash
curl -s https://api.secapi.ai/mcp | jq '.distribution | {
  transport: .transport,
  auth: .auth.supported,
  claudeCode: .clients.claudeCode.addCommand,
  tools: (.capabilitiesSummary.tools | length),
  resources: (.capabilitiesSummary.resourceTemplates | length)
}'
```

That payload includes:

- the canonical streamable HTTP MCP transport URL
- auth metadata links
- sponsored bootstrap URLs for human token issuance and machine bootstrap
- install hints
- current tool and resource summaries

<Steps>
  <Step title="Fetch the install metadata">
    Confirm the hosted MCP endpoint is live and returning its transport and auth metadata.
  </Step>
  <Step title="Bootstrap the machine identity">
    Have an administrator mint a sponsor token first, then exchange it through `POST /v1/agent/bootstrap` before you start repeated MCP work.
  </Step>
  <Step title="Register the server in your client">
    Add the streamable HTTP MCP endpoint to Claude Code, Cursor, or another MCP-aware client.
  </Step>
  <Step title="Exercise a real tool path">
    Resolve an issuer, fetch a filing, and extract a section before calling the setup done.
  </Step>
</Steps>

## Auth and error shape

`GET /mcp` is public discovery metadata. Use it to confirm the deployed transport, auth expectations, and tool/resource summary before configuring a client.

`POST /mcp` is the authenticated JSON-RPC transport. Send machine credentials with the `x-api-key` header. Missing or invalid credentials fail before JSON-RPC dispatch and return REST-style auth errors; a missing key returns HTTP `401` with `code: "missing_api_key"`.

Transport-level gates such as auth, generic rate limits, billing, or entitlement checks can fail before JSON-RPC dispatch and use REST-style error bodies. Once the request reaches JSON-RPC dispatch, protocol, tool, AI quota, timeout, and resource failures return JSON-RPC error envelopes. Preserve `error.data.requestId` and `error.data.traceparent` from those envelopes when opening a support ticket; those fields identify the API request and distributed trace behind the failed MCP turn.

## AI query quota

MCP `tools/call` requests for `intelligence.query` and `signals.dilution.enhanced.get` count toward the monthly `ai_queries` quota. Successful calls include a top-level `_secapi.quota` envelope with projected post-call usage:

```json
{
  "jsonrpc": "2.0",
  "id": "quota_example",
  "result": {},
  "_secapi": {
    "quota": {
      "family": "ai_queries",
      "mcpToolName": "intelligence.query",
      "limit": 600,
      "used": 42,
      "remaining": 558,
      "resetAt": "2026-05-01T00:00:00.000Z"
    }
  }
}
```

When the quota is exhausted, hosted MCP returns HTTP `429` with JSON-RPC error code `-32005` and `error.data.code: "ai_query_quota_exceeded"`. Treat `-32004` as retryable timeout and `-32005` as non-retryable until quota reset or plan upgrade. If a tool circuit is open, hosted MCP returns HTTP `503` with JSON-RPC error code `-32006` and `error.data.code: "mcp_tool_circuit_open"`; retry it only after availability backoff.
