Hosted MCP gives an MCP-aware client access to SEC API tools without a local wrapper. Use it for a bounded research task with explicit evidence requirements, not as a substitute for checking the source fields a result depends on.

Before a repeated monitor or agent loop, review [pricing and limits](https://secapi.ai/pricing) and create an API key at [secapi.ai/signup](https://secapi.ai/signup).

## Start with a filing brief

Install the server first, then give your client a task that has a clear output and evidence trail:

<Prompt>
Use SEC API MCP to resolve NVDA, retrieve its latest 10-K, extract Item 1A, and summarize the filing. Include accessionNumber, filingDate, filingUrl, requestId, and any returned freshness or provenance fields with the summary.
</Prompt>

This workflow typically begins with issuer resolution and latest filing retrieval, then reads a specific section. Use the exact tool names returned by your client's tool list.

## Pick tools by job

| Job | Start with |
| --- | --- |
| Resolve a ticker or CIK | `entities.resolve` |
| Retrieve a current filing | `filings.latest` |
| Search filing text | `sections.search` or `filings.search` |
| Read a filing section | `sections.get` |
| Read filed numbers | `statements.get` or `facts.get` |
| Inspect institutional ownership | `owners.institutional_holders` |

The server's tool list is the current contract. Use it instead of relying on a cached client configuration or an older prompt.

## Keep the evidence trail

Ask the agent to preserve:

- `accessionNumber`, `filingDate`, and `filingUrl`
- `requestId` and `traceparent` when returned
- source URLs, citations, provenance, and freshness fields
- pagination or truncation signals when a tool returns a list

That context lets a reader verify the filing and lets your application distinguish a current result from a partial or historical one.

## Control tool calls

Make the requested scope explicit. For example, ask for one issuer, a named form, a date range, and the fields you need. For list-shaped MCP results, use the response controls documented by the tool when you need a smaller text representation; structured content remains the canonical result.

Do not automatically repeat a tool call after a quota or billing failure. Treat a timeout as a bounded retry candidate, and preserve the error's request ID before investigating.

## Debug an MCP request

If a client cannot connect, revisit [MCP installation](/mcp-install). If it connects but a call fails, capture the HTTP status or JSON-RPC error code, `error.data.code`, request ID, and trace context. Read [Troubleshooting](/troubleshooting) for the next action.
