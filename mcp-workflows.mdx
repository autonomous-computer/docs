---

Use SEC API MCP for a bounded research job. The reliable pattern is discover, inspect the schema, retrieve one source record, then summarize only what that record supports. Install the server first: [Install SEC API MCP](/mcp-install).

## A complete first job

Give the client this prompt after it has connected:

<Prompt>
Find NVIDIA's latest 10-K. Retrieve Item 1A from that filing. Return: company name, accession number, filing date, filing URL, section title, a five-bullet summary, requestId, traceparent when present, and any truncation or freshness signals. Do not infer facts that are not in the retrieved section.
</Prompt>

Expected output is one identified filing plus a bounded summary. A changed accession number on a later run is normal; a summary without its filing identity is not sufficient for a production report.

## Let the server choose the contract

The tool catalog is the current contract. Ask the client to call `tools.search` with a natural-language job, then call `tools.describe` on the selected tool before it supplies arguments. For the job above, useful searches are `company identity`, `latest filing`, and `filing section`.

`tools/list` can be filtered to `core` or a namespace when your client supports that parameter. Do not copy an old tool list into an agent instruction. Tool names, schemas, quotas, and annotations are server-owned and can change.

## Keep a source trail

Persist the returned `accessionNumber`, `filingDate`, `filingUrl`, `requestId`, and `traceparent` with the agent result. Keep route-provided provenance, freshness, pagination, and truncation fields when present. `structuredContent` is the canonical typed payload; compact text formatting is for the human or agent reading it.

## Limits and state changes

Start with a narrow issuer, form, date range, and result limit. If a tool reports pagination or truncation, inspect the next page or reduce the request before asking the model to draw a conclusion. Do not blindly retry authentication, billing, quota, or validation failures.

Most MCP tools are read-only. Inspect tool annotations before any stateful call; `situations.watch` and `exports.create` are not safe to parallelize or replay casually. Monitor, webhook, and stream control-plane changes remain human-authenticated dashboard actions.

## Production handoff

Run the same prompt against a fixture issuer in CI, record the output identifiers, and set a reviewer rule for material conclusions. Use REST or an SDK when your application needs a stable request loop, typed error handling, and its own retry policy. See [API conventions](/api-conventions), [MCP installation](/mcp-install), and [Troubleshooting](/troubleshooting).
