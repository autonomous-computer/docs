title: "Migrate from Fiscal.ai fund letters"
description: "Qualify one fund-letter research workflow on SEC API, preserving document identity and quoted-source evidence before cutover."
---

Use this guide when moving a fund-letter research workflow from Fiscal.ai to SEC API. SEC API provides fund-letter listings, manager and fund records, thesis retrieval, search, and document links through `/v1/fund-letters/*`. Fiscal.ai may remain the better fit where its historical scope, document rights, product workflow, or data fields meet a requirement that you have not qualified on SEC API.

## Start with one research job

Choose a job a reviewer can check: list letters for a period, find a manager, retrieve a thesis for one company, search letter text, or inspect a source document. Confirm the current Fiscal.ai endpoint, identifier format, pagination, and document-access terms in Fiscal.ai's own documentation before planning a field-for-field migration.

| Research job | SEC API path to evaluate | Evidence to retain |
| --- | --- | --- |
| List letters | `GET /v1/fund-letters` | Letter ID, manager, fund, period, status, and cursor behavior. |
| Find a manager | `GET /v1/fund-letters/managers` | Manager ID, name, and returned matching fields. |
| Find company theses | `GET /v1/fund-letters/theses?ticker=...` | Thesis ID, letter ID, issuer identity, quote fields, and source link. |
| Search letter text | `GET /v1/fund-letters/search` | Query, filters, matched record, and available source metadata. |
| Retrieve a document link | `GET /v1/fund-letters/{letter_id}/document` | Letter ID, document format, response behavior, and source URL. |

## Run a bounded proof

```bash
curl --fail-with-body -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/fund-letters?period=2026Q1&limit=25"

curl --fail-with-body -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/fund-letters/theses?ticker=NVDA&limit=25"

curl --fail-with-body -sS -L -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/fund-letters/LETTER_ID/document?format=markdown"
```

Replace `LETTER_ID` only with a returned identifier. For a quoted claim, retain the letter and thesis IDs, manager and fund identifiers when supplied, period, source URL, quote or anchor fields, request parameters, and `requestId`. Check the linked source before presenting text as a quotation.

## Fit and limits

Use SEC API when its documented fund-letter records and evidence fields meet your selected research job. Do not treat a letter listing or search result as a claim of complete manager, period, or document coverage. Document access can depend on source availability and rights; evaluate the exact documents your users need. Keep Fiscal.ai or another licensed source for any required historical or document-access scope that does not pass the dual run.

## Cut over one reversible workflow

1. Save the current request, response shape, and source record from the existing workflow.
2. Run the equivalent bounded SEC API request and compare identifiers, filters, pagination, nulls, and source evidence.
3. Verify a small set of research quotes against the linked document.
4. Move one reader or export behind a reversible control.
5. Add semantic search, sync, or additional corpus segments only after the first job passes.

## Next step

Choose one manager-period or issuer-thesis question and record its acceptance result. Expand only after a reviewer can trace each displayed quotation to its source.
