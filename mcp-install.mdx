---
title: "Install SEC API MCP"
description: "Connect an MCP-aware client to SEC API and prove the installation with a source-backed filing workflow"
---

Connect hosted MCP when your agent client supports streamable HTTP MCP and needs SEC filing, statement, ownership, or search tools. Start with an API key and prove one filing workflow before allowing an agent to use the tools in a broader task.

## Get the connection details

The public metadata endpoint describes the deployed transport and client hints:

```bash
curl -sS https://api.secapi.ai/mcp
```

The authenticated MCP endpoint is `https://api.secapi.ai/mcp`. Send a machine API key in `x-api-key` for tool calls.

```bash
export SECAPI_API_KEY="secapi_..."
```

## Connect Claude Code

```bash
claude mcp add --transport http secapi https://api.secapi.ai/mcp \
  --header "x-api-key: $SECAPI_API_KEY"
```

## Connect a JSON-configured client

Use this shape when your MCP-aware client accepts a project or user configuration file:

```json
{
  "mcpServers": {
    "secapi": {
      "url": "https://api.secapi.ai/mcp",
      "headers": {
        "x-api-key": "${SECAPI_API_KEY}"
      }
    }
  }
}
```

Keep the key in the environment rather than committing it to the configuration file.

## Prove the installation

Ask the client to do a short, verifiable job:

<Prompt>
Use SEC API MCP to resolve AAPL, retrieve the latest 10-K, extract Item 1A, and report the accession number, filing date, filing URL, and request ID with the result.
</Prompt>

The agent should return the filing evidence, not only a summary. If it cannot list or call a tool, check the configured URL, `x-api-key` header, and client logs before changing the prompt.

## Know the error boundary

`GET /mcp` is public discovery metadata. `POST /mcp` is the authenticated JSON-RPC transport. Authentication, billing, and generic rate-limit failures can return normal HTTP error bodies before JSON-RPC dispatch. Tool errors return JSON-RPC error envelopes once a request reaches the MCP server.

For setup patterns and tool selection, read [MCP workflows](/mcp-workflows). For response codes, read [Troubleshooting](/troubleshooting).
