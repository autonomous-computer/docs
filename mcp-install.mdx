---
title: "Install SEC API MCP"
description: "Connect an MCP client to the hosted SEC API server and verify one source-backed filing workflow"
---

SEC API hosts MCP at `https://api.secapi.ai/mcp`. MCP clients make authenticated JSON-RPC requests to that endpoint with an API key in `x-api-key`.

## Configure the server

Create an API key in the signed-in [dashboard](https://secapi.ai/app) and keep it in the client process environment:

```bash
export SECAPI_API_KEY="secapi_..."
```

### Claude Code

Run this supported external-user command:

```bash
claude mcp add --transport http secapi https://api.secapi.ai/mcp --header "x-api-key: $SECAPI_API_KEY"
```

### Other supported clients

`GET https://api.secapi.ai/mcp` is authenticated and requires the `x-api-key` header; it returns the hosted server's connection metadata, including its client install guidance.

```bash
curl --fail-with-body -sS -H "x-api-key: $SECAPI_API_KEY" https://api.secapi.ai/mcp
```

The SEC API CLI can generate the supported per-client configuration: `secapi mcp install --client <claude-code|claude-desktop|cursor|windsurf|project>`. For Claude Code, that command prints the command above; for the other clients, it writes their respective configuration file (or prints it with `--print`).

The portable HTTP-MCP configuration values are:

```json
{
  "mcpServers": {
    "secapi": {
      "url": "https://api.secapi.ai/mcp",
      "headers": { "x-api-key": "${SECAPI_API_KEY}" }
    }
  }
}
```

Use the environment-variable syntax supported by your client. Do not put the key in a repository, prompt, URL, or browser-delivered configuration.

## Verify discovery, then retrieve evidence

Restart the client so it reads the new environment, then use this request:

<Prompt>
Call tools.search for issuer resolution, latest filing, and filing section. Call tools.describe for each selected tool before invoking it. Resolve AAPL, retrieve its latest 10-K, and retrieve Item 1A. Return the company name, accession number, form, filing date, filing URL, requestId, and a short Item 1A summary. State when a field is absent instead of inventing it.
</Prompt>

`tools.search` and `tools.describe` are SEC API MCP tools; they are not JSON-RPC methods. The client uses the standard `tools/call` method to invoke them. Discovery avoids pinning an old catalog or schema in an agent instruction.

## Diagnose a failed connection

- No tools: reload the configuration and confirm `SECAPI_API_KEY` is available to the client process.
- `401` or `403`: repair or rotate the API key; do not substitute a browser bearer token.
- JSON-RPC validation error: send one JSON-RPC 2.0 request object with the required request ID. Batch requests are not supported.
- Validation, billing, quota, or timeout result: retain the returned request identifier, narrow the next call, and follow the response instead of replaying a broad job.

The API key grants machine data access, not dashboard authority. Continue with [MCP workflows](/mcp-workflows) for evidence retention and production use.
