---
title: "Install SEC API MCP"
description: "Connect an MCP client to the hosted SEC API server, authenticate with an API key, and prove tool discovery with one source-backed filing request."
---

SEC API serves hosted MCP at `https://api.secapi.ai/mcp`. Use an organization API key for a server-to-server client. `GET /mcp` publishes connection metadata; MCP requests are authenticated JSON-RPC `POST /mcp` calls with `x-api-key`.

## Add the hosted server

Create an API key in the signed-in [dashboard](https://secapi.ai/app), then expose it only to the client process:

```bash
export SECAPI_API_KEY="secapi_..."
claude mcp add --transport http secapi https://api.secapi.ai/mcp \
  --header "x-api-key: $SECAPI_API_KEY"
```

For another HTTP MCP client, configure the same URL and header:

```json
{
  "mcpServers": {
    "secapi": {
      "url": "https://api.secapi.ai/mcp",
      "headers": { "x-api-key": "${SECAPI_API_KEY}" }
    }
  }
}
```

Use your client's environment-variable syntax. Do not put the key in a repository, a shared prompt, or browser-delivered configuration.

## Verify discovery before research

Restart the client, ask it to list tools, then give it this precise request:

<Prompt>
Use tools.search to find an issuer-resolution tool and tools.describe before calling it. Resolve AAPL. Return the CIK, company name, requestId, and traceparent if returned. Then explain which tool you would use to retrieve the latest 10-K, but do not make that second call.
</Prompt>

A successful result identifies Apple and shows that the client can discover schemas rather than relying on an embedded catalog. It should not require a particular tool count or a fixed CIK in the prompt.

## Connection failures

- No tools: reload the client configuration and verify `SECAPI_API_KEY` is in that process environment.
- `401` or `403`: create or rotate the API key in the dashboard; do not substitute a browser bearer token.
- JSON-RPC validation error: send one object with `jsonrpc: "2.0"`; batch JSON-RPC requests are not supported.
- Quota, billing, or timeout response: retain the request id, narrow the next call, and follow the response guidance instead of replaying a broad request.

The key grants machine data access, not dashboard authority. A person configures [monitors](/monitors), webhook destinations, streams, billing, and key lifecycle in the signed-in dashboard. Continue with [MCP workflows](/mcp-workflows) for a source-backed filing job.
