# Macro Tier-1 Official Sources

## Source Modes

- `direct_official`: primary runtime fetches directly from an official government agency API or download endpoint.
- `official_bridge`: official agency data is accessed through a bridge such as DBnomics when a stable direct API is not yet available.
- `fallback_harmonized`: a harmonized source is used only as an explicit fallback and must be visible in metadata.

DBnomics is a bridge and fallback, not the product claim. When direct official APIs exist and are stable, SEC API treats those as the preferred source of truth.

## Example Calls

Search for an indicator key:

```bash
curl "https://api.secapi.ai/v1/macro/search?q=gdp&country=US" \
  -H "x-api-key: $SECAPI_API_KEY"
```

Fetch observations:

```bash
curl "https://api.secapi.ai/v1/macro/indicators?country=US&indicator=CPIAUCSL&limit=10&response_mode=compact" \
  -H "x-api-key: $SECAPI_API_KEY"
```

Fetch the compact high-signal pack:

```bash
curl "https://api.secapi.ai/v1/macro/high-signal-pack?country=US" \
  -H "x-api-key: $SECAPI_API_KEY"
```

Fetch upcoming scheduled releases only:

```bash
curl "https://api.secapi.ai/v1/macro/calendar?country=US&days=45&limit=12&response_mode=compact" \
  -H "x-api-key: $SECAPI_API_KEY"
```

## Freshness Ownership

Production macro writes are owned by Dagster. Read-only validation checks cover:

- source coverage and source health
- ClickHouse freshness for observations, releases, forecasts, regimes, revisions, and factor impacts
- methodology validation
- country-report and high-signal response shape

Every observation can carry source, revision, release-calendar, canonicality, fallback, freshness, materialization, and methodology metadata. Keep those fields when macro data supports an investment conclusion.
