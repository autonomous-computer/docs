---
title: "Insider trading data API"
description: "Query normalized Form 3, 4, and 5 records by issuer or reporting owner while retaining the filing evidence needed for review"
---

Use `GET /v1/insiders` to create a source-cited queue of reported insider transactions. The route returns normalized records from Forms 3, 4, and 5; it does not determine an insider's motive, whether a transaction was discretionary, or an investment conclusion.

## Get recent Form 4 records

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/insiders?ticker=AAPL&forms=4&limit=10"
```

Each row can include the reporting owner, role, form, filing date, transaction date, transaction code, direction, security title, shares, price, post-transaction ownership, accession number, and filing URL. Filing-derived values can be absent, so preserve null or missing values rather than filling them with estimates.

## Narrow the review queue

Filter Form 4 records by a reported transaction code:

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/insiders?ticker=JPM&forms=4&transaction_code=P&limit=25"
```

| Parameter | Purpose |
| --- | --- |
| `ticker` or `cik` | Select the issuer |
| `forms` | Select filing families such as `4` |
| `owner_name` or `owner_cik` | Select a reporting owner |
| `security_title` | Limit results to a disclosed security title |
| `transaction_code` | Select a Form 4 transaction code |
| `date_from` and `date_to` | Bound the filing-date window |
| `limit` and `cursor` | Page through results |

## Keep the fields that make a result reviewable

Keep `transactionCode` and `transactionDirection` together. Keep `transactionDate` separate from `filingDate`. Store the `accessionNumber`, `provenance.filingUrl`, and `requestId` with any alert or table row.

A code such as `P` does not prove open-market activity, and a sale can reflect tax withholding, an exercise, a gift, a plan, or another filing-specific fact. Read the source filing and footnotes before assigning motive or labeling activity in an investor-facing product.

## Handle pagination and failures

The list response supplies `hasMore` and `nextCursor`; use the returned cursor to continue. A non-2xx response is a failed lookup, not an empty transaction set. Retain `requestId` and `traceparent` for support or diagnostics.
