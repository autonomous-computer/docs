Use the insider-trading API to review transactions disclosed by officers, directors, and 10% owners. It returns normalized transaction rows from Forms 3, 4, and 5, together with the SEC accession number and filing URL needed to review the source.

An insider filing is evidence, not an investment conclusion. A sale can reflect taxes, diversification, or scheduled activity; a code alone does not explain the surrounding facts.

## Get recent Form 4 transactions

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/insiders?ticker=AAPL&forms=4&limit=10"
```

The list envelope contains `data`, `hasMore`, `nextCursor`, `requestId`, and `traceparent`. Each transaction row can include:

```json
{
  "ownerName": "Example reporting owner",
  "ownerTitle": "Chief Executive Officer",
  "form": "4",
  "filingDate": "2026-07-01",
  "transactionDate": "2026-06-30",
  "transactionCode": "P",
  "transactionDirection": "acquired",
  "securityTitle": "Common Stock",
  "transactionShares": 1000,
  "transactionPrice": 25.5,
  "transactionValue": 25500,
  "sharesOwnedFollowing": 51000,
  "accessionNumber": "0000000000-00-000000",
  "provenance": {
    "filingUrl": "https://www.sec.gov/Archives/..."
  }
}
```

The values above illustrate the response shape only. The source filing remains the authority for a transaction's terms, footnotes, and context.

## Filter the records you need

To retrieve Form 4 transactions reported with code `P`, pass `forms=4` and `transaction_code=P`:

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/insiders?ticker=JPM&forms=4&transaction_code=P&limit=10"
```

| Parameter | Use it for | Example |
| --- | --- | --- |
| `ticker` or `cik` | Scope to an issuer | `ticker=AAPL` |
| `forms` | Limit the filing family | `forms=4` |
| `owner_name` or `owner_cik` | Focus on a reporting owner | `owner_name=Jamie%20Dimon` |
| `security_title` | Separate common stock from another security | `security_title=Common%20Stock` |
| `transaction_code` | Select a Form 4 transaction code | `transaction_code=P` |
| `date_from` and `date_to` | Bound the filing-date window | `date_from=2026-01-01` |
| `limit` and `cursor` | Page through a result set | `limit=25&cursor=25` |

## What to preserve in an investor workflow

1. **The code and direction.** Keep both `transactionCode` and `transactionDirection`; they answer different questions.
2. **Dates.** `transactionDate` can differ from `filingDate`. Form 4 is generally due within two business days, but the filing date is the date this API query is filtering.
3. **The filing evidence.** Store `accessionNumber` and `provenance.filingUrl` with any alert or research note.
4. **Nulls and missing context.** Price, value, title, and post-transaction ownership are filing-derived and may be absent. Do not silently replace an unavailable field with an estimate.
