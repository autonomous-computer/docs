---
title: "SEC Financial Statements API"
description: "Retrieve annual or quarterly SEC financial statements with a clear filing trail for models, screens, and agent research."
---

Use the Financial Statements API when the job is to retrieve a filed income
statement, balance sheet, or cash-flow statement without turning a filing into
an XML parsing project. Start with one issuer, one period, and one statement;
then preserve the source and request metadata that let another person inspect
the result.

## Choose the response you need

| Job | Request |
| --- | --- |
| One normalized statement | `GET /v1/statements?statement=income_statement` |
| Income statement, balance sheet, and cash flow together | `GET /v1/statements/all` |
| A compact company-facing series | `GET /v1/companies/income-statements`, `GET /v1/companies/balance-sheets`, or `GET /v1/companies/cash-flow-statements` |
| A company snapshot across several financial surfaces | `GET /v1/companies/financials` |

All statement routes accept an issuer identifier such as `ticker` or `cik`.
Use `period=annual` or `period=quarterly`; use `fy`, `fy_from`, and `fy_to`
when the filing periods matter more than a default latest result.

## Retrieve one annual statement

This example asks for Apple's annual income statement in the compact agent
shape. Add `secapi-version` once you have selected the contract version your
application will hold to.

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/statements?statement=income_statement&ticker=AAPL&period=annual&limit=1&view=agent"
```

The useful result is not just a number. Keep the period, statement identity,
form, and filed date with the result. The default response can also include
provenance, freshness, materialization, validation, `requestId`, and
`traceparent`; compact and agent views intentionally return less envelope data.

## Retrieve the three statements together

Use the all-statements bundle when a model or agent needs the filing-period
relationship across the three core statements:

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/statements/all?ticker=AAPL&period=annual&fy=2025&limit=1"
```

Do not assume that a normalized line item settles an accounting question by
itself. Use the filing source and the relevant note when policy, segment,
currency, or presentation choices affect the conclusion.

## Company-data surfaces

`/v1/companies/overview` is the right starting point for a company briefing.
It accepts a ticker, symbol, or CIK and can request optional `segments`,
`footnotes`, `dilution`, and `factors` enrichments:

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/companies/overview?ticker=AAPL&include=segments,dilution"
```

Those enrichments are not a reason to discard the base company result. They
can degrade independently when an underlying extraction is unavailable. For a
small multi-company refresh, use `tickers=AAPL,MSFT,NVDA` with
`/v1/companies/financials` and inspect each row's status before treating a
batch as complete.

## What requires judgment

- Annual and quarterly periods are not interchangeable. State which one your
  model uses and why.
- A company may use an extension concept or a presentation that needs a
  tag-level fact or filing note to interpret.
- A partial company batch is evidence of a partial batch, not a zero or a
  missing value to silently fill.
