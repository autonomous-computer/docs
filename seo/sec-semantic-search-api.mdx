## Find language that literal search may miss

Use semantic search when the disclosure language may not repeat the words in your question. It returns passages to inspect, not findings about a company, risk, or accounting treatment.

## Search a constrained disclosure question

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/search/semantic?q=artificial%20intelligence%20regulatory%20risk&ticker=GOOGL&form=10-K&filing_year=2025&mode=hybrid&limit=5"
```

`q` is required. Scope the candidate set with `ticker`, `symbol`, or `cik`, plus `form` and `filing_year` when possible. `mode` accepts `keyword`, `semantic`, or `hybrid` and defaults to `hybrid`; `limit` bounds the single response.

Expect a search object with query and mode plus a section list. A result can include its accession, section key, source URL, ticker, highlighted snippet, and markdown `char_start`/`char_end` offsets; default responses can also carry retrieval detail and a degraded-state signal. Preserve these fields with any citation or downstream prompt.

## Verify before drawing a conclusion

1. Keep the result’s accession, section key, source URL, and offsets.
2. Retrieve the section by accession when the full context matters.
3. Treat offsets as positions in normalized section markdown, not raw SEC HTML.
4. When offsets are missing or the response reports degradation, open the source filing rather than slicing a passage programmatically.

Use `GET /v1/search/fulltext` for literal language, `GET /v1/sections/search` for cursor-paginated normalized-section search, and `GET /v1/filings` when the question is which filing exists rather than what it says.

## Implementation path

Use `client.semanticSearch({ q, ticker, form, filing_year, mode, limit })` from `@secapi/sdk-js`, or call `GET /v1/search/semantic` directly. The current hosted MCP tool list has `sections.search` for normalized section search, but no equivalent semantic-search tool; use REST or an SDK for semantic retrieval and hand the returned citation into the agent context.

## Retrieval limits

- Similarity or hybrid ranking is not a materiality score, a completeness guarantee, or a recommendation.
- A result list reflects indexed, normalized section content and its availability; it is not a claim that all EDGAR prose is searchable or that every related passage was returned.
- `view=agent` retains citation fields but omits `score` and `retrievalMode`. Use the default response to evaluate retrieval behavior.
- Treat a `503` or disclosed degraded state as an unavailable retrieval surface, not as a negative search result.
