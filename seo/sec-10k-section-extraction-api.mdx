---
title: "10-K Section Extraction API"
description: "Extract a named section from a known SEC filing or the latest matching 10-K while retaining the filing evidence."
---

Use section extraction when the research question maps to a disclosed filing item, such as `item_1a` for Risk Factors or `item_7` for MD&A. The accession route is the reproducible path; the latest route is appropriate only when a monitor intentionally follows the current annual filing.

## Pin the filing for a repeatable result

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/0000320193-25-000079/sections/item_1a?ticker=AAPL&form=10-K&mode=full"
```

The route is `GET /v1/filings/{accession_number}/sections/{section_key}`. `ticker`, `cik`, and `form` are optional resolution hints; avoid conflicting hints. A successful default/full response includes extracted `contentMd` and filing provenance, including the accession and source URL. Save them with the passage and `requestId`.

`mode=compact` shortens the body for triage. `view=agent` is a narrow identity-and-citation projection: it returns section identity such as `key`, `accessionNumber`, and `source_url`, not the section body. Use `mode=full` when the text itself is needed.

## Follow the newest annual filing intentionally

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/latest/sections/item_1a?ticker=AAPL&form=10-K&mode=compact"
```

`GET /v1/filings/latest/sections/{section_key}` defaults to `form=10-K`; set it explicitly in a production workflow. Unlike `GET /v1/filings/latest`, the route is section-oriented, but "latest" remains time-dependent. Persist the resolved accession each time the monitor runs.

## Evidence and extraction limits

Section text can omit layout-dependent tables, exhibits, cross-references, or surrounding definitions. A `404` can reflect an unresolved accession, an unavailable section key, or a nonextractable section; it is not evidence that the topic is absent from the filing. Do not assume every form implements every item key.

For a material claim, cite the returned accession, filing URL, form, filing date, and section key, then open the SEC source to check context. Use [filing search](/seo/sec-filing-search-api) to choose the correct historical filing before extraction.
