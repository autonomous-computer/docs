---
title: "10-K Section Extraction API"
description: "Retrieve a named section from a known SEC filing or the latest matching filing, with a compact option for review and agent use."
---

Use section extraction when you know the disclosure you need: risk factors,
MD&A, business description, or another named section. It is not a substitute
for reading the filing. It is a way to get to the relevant disclosure without
making HTML parsing part of every research loop.

## Choose a stable filing or the latest matching one

For a reproducible analysis, use the accession number from a filing search:

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/filings/0000320193-24-000123/sections/item_1a?mode=compact&view=agent"
```

For a monitoring loop, ask for the named section on the latest 10-K instead:

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/filings/latest/sections/item_1a?ticker=AAPL&form=10-K&mode=compact&view=agent"
```

The latest-section route defaults to `form=10-K`; keep `form` explicit in a
monitor so the expected filing type is clear. This differs from plain
`GET /v1/filings/latest`, where omitting `form` means the latest filing of any
type.

## What to retain with the text

Keep the accession number, section key, form, filing date, and SEC source URL
with an extracted passage. `mode=full` or `mode=compact` controls the section
body; `view=agent` is for a smaller handoff while preserving the fields needed
to identify the source. Do not infer that every response preserves tables,
contains every historical filing, or can be batched: those are separate
coverage and implementation questions.

## A practical review sequence

1. Find the filing with [`GET /v1/filings`](/api-reference/filings/get-v1-filings)
   when the filing is not already known.
2. Retrieve the section by accession for a stable comparison, or use the
   latest-section route for a current monitor.
3. Open the SEC source before relying on a sentence whose meaning depends on a
   table, definition, cross-reference, or surrounding qualification.
