---
title: "10-K Section Extraction API"
description: "Retrieve a named section from a known SEC filing or the latest matching 10-K, with a source trail for review."
---

Use section extraction when you know the disclosure you need. It takes HTML parsing out of the normal research loop; it does not replace reading the filing when tables, definitions, or qualifications matter.

## Use an accession for a repeatable result

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/0000320193-24-000123/sections/item_1a?ticker=AAPL&form=10-K&mode=compact&view=agent"
```

For a monitor that should follow the current annual filing instead, use the latest-section route:

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/latest/sections/item_1a?ticker=AAPL&form=10-K&mode=compact&view=agent"
```

The latest-section route defaults to `form=10-K`, but make the form explicit for a production monitor. This differs from plain `GET /v1/filings/latest`, where no `form` means the latest filing of any type.

## Keep the source with the text

Store the accession number, section key, form, filing date, SEC filing URL, and request ID with an extracted passage. `mode=compact` is a smaller section body; `mode=full` provides the full response. `view=agent` is a route-defined projection, not a generic field selector.

## Common failures

- A `404` can mean the accession, section key, or optional filing hints do not resolve to an extractable section. Find the filing first, then use its exact accession.
- A section result does not guarantee that surrounding tables or cross-references are represented in the passage. Open the SEC source before relying on such context.
- Do not infer that every form has every item key. Use the filing family and the specific extracted response as the evidence.
