# SEC Enforcement Actions API

SEC enforcement material is published across litigation releases, administrative proceedings, and Accounting and Auditing Enforcement Releases. `GET /v1/events/enforcement` gives developers one API surface for searching those public sources by respondent, issuer, date, normalized category, and disclosed penalty amount.

It is designed for evidence discovery. The returned record helps a researcher find and organize the relevant SEC release; it does not replace the release or establish liability for a person, company, or security mentioned in it.

## Search an enforcement category

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/events/enforcement?violation_type=fraud&date_from=2025-01-01&limit=25"
```

Use `source_type=aaer` when the accounting and auditing release stream is specifically relevant. Use `respondent`, `ticker`, or `cik` to focus a search, but verify the match against the source document because names and issuer references are not unique identifiers.

## What the API returns

The list response contains `data`, `hasMore`, `nextCursor`, and `requestId`. An enforcement row can include:

- `sourceType`, `publishedAt`, `releaseNumber`, `title`, and `excerpt`
- `respondents`, with any resolved `ticker` or `cik` exposed as an aid rather than a legal conclusion
- `violationType`, a normalized retrieval label derived from release language
- `penaltyAmount` and `relatedReleaseNumber` where the release makes those values available
- `documentUrl`, the SEC release to review, and `htmlUrl` when a rendered SEC page was found

## Supported normalized categories

`fraud`, `insider_trading`, `reporting_violation`, `market_manipulation`, `registration_violation`, `investment_adviser`, `broker_dealer`, `municipal_securities`, and `other`.

These categories make a broad release set easier to search. They are not a substitute for the SEC's allegations, legal theories, or final disposition.

## Common uses

**Research and diligence:** screen a watchlist or counterparty list, then link the relevant releases into a research record.

**Compliance triage:** find potentially relevant source material by respondent and date before a human review.

**Regulatory trend work:** group releases by source type or normalized category, while preserving the underlying source URLs for a reproducible study.
