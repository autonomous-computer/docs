---
title: "13F holdings comparison API"
description: "Compare the latest two parsable 13F reports for one manager and keep the dates and filing evidence that bound the result"
---

Use `POST /v1/owners/13f/compare` to identify differences between the latest two parsable 13F reports available for a manager CIK. The route is useful for building a review queue; it is not a way to select two arbitrary quarters or infer a manager's current book.

## Get a comparison

```bash
curl -X POST \
  -H "content-type: application/json" \
  -H "x-api-key: $SECAPI_API_KEY" \
  -d '{"cik":"0001350694","limit":50}' \
  "https://api.secapi.ai/v1/owners/13f/compare"
```

The response identifies the current and previous filing dates and returns position rows with a status and reported share/value differences. Keep `managerName`, both filing dates, the row `cusip`, the status, `provenance`, and `requestId` in a downstream report.

## Read the result correctly

| Response field | Use it for | Do not use it for |
| --- | --- | --- |
| `currentFilingDate` and `previousFilingDate` | Identifying the two disclosed reports | Inferring trade dates |
| `status` | Grouping added, removed, changed, and unchanged rows | Explaining manager intent |
| `currentShares` and `previousShares` | Measuring disclosed share-count change | Measuring complete exposure |
| reported value fields | Ranking rows for review | Current mark-to-market value |
| `provenance.filingUrl` | Opening the source filing | Replacing a review of the source filing |

## Retrieve a specific period instead

For a selected period, list available filings and request the reports separately:

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/owners/13f/filings?cik=0001350694&limit=8"
```

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/owners/13f?cik=0001350694&reportDate=2024-12-31&limit=50"
```

The compare endpoint has no `period_a` or `period_b` query parameters. Build a custom selected-period comparison only after retrieving and labeling the two source reports yourself.

## Failure and interpretation limits

The request requires a CIK and accepts a body `limit` from 1 through 50. A comparison can fail when the service cannot produce the two parsable reports; handle the non-2xx response, keep its request metadata, and avoid treating a failed comparison as an unchanged portfolio.

13F reports are delayed, limited disclosures. They do not reveal every position, hedge, cash balance, derivative, or trade after the report date.
