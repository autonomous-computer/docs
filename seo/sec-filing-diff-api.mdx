title: "SEC Filing Diff API"
description: "GET /v1/filings/diff is currently unavailable. Use supported filing retrieval and search endpoints to compare source disclosures."
---

`GET /v1/filings/diff` is currently unavailable and returns `404 filings_intelligence_not_available`. Do not use it as an active comparison dependency. The route is inactive in the deployed service; its availability is not a public-product guarantee.

When it becomes available, it will compare a selected issuer filing with a prior-period or explicitly selected counterpart and return section-level change information. Any result will be a review aid based on rendered filing text, not a determination of materiality, a restatement, legal significance, an accounting error, or an issuer's motive.

## Compare filings with supported endpoints

Use the supported filing endpoints to build a controlled review record today:

1. Identify the current filing for an issuer and form with [Filing search API](/seo/sec-filing-search-api) or [GET /v1/filings/latest](/api-reference/filings/get-v1-filings-latest). Retain the accession number, filing date, and source URL.
2. Retrieve the relevant disclosure from each selected filing with [10-K section extraction](/seo/sec-10k-section-extraction-api), then compare the source text in context.
3. Use [full-text search](/seo/sec-full-text-search-api) for exact language or [semantic search](/seo/sec-semantic-search-api) for related passages when you need to locate candidate disclosures before reading them.

The supported workflow requires you to choose and retain both filing identities. A missing search or retrieval result does not establish that two filings are identical.

## When filing diff is available

When the endpoint is enabled and documented as available, provide a ticker and form, preserve the selected filing citations, and treat its rendered-text comparison as a starting point for review. Confirm important changes in the underlying filings before publishing a summary.

Boilerplate edits, parsing differences, amendments, and substantive disclosure changes may appear as textual differences. Keep the request metadata and both source records with any comparison note so another reviewer can reproduce the work.
