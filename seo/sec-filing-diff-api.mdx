---
title: "SEC Filing Diff API"
description: "Compare the latest and prior filing of the same form, or name an explicit pair, then review the source filings behind each change."
---

Use filing diff to triage what changed between two filings of the same form. It is a comparison aid, not a determination that a change is material or an amendment is a restatement.

## Compare annual filings

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/diff?ticker=AAPL&form=10-K&sections=item_1a_risk_factors,item_7_mdna"
```

`ticker` and `form` are required. When `from` and `to` are absent, the API pairs the latest filing with the prior filing of the same form. To control the comparison, pass an accession number or ISO date for each selector:

```bash
curl -sS -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/diff?ticker=AAPL&form=10-K&from=2023-10-01&to=2024-10-01&include_hunks=false"
```

## What to review

The report identifies the paired filings and returns section changes. `include_hunks=false` removes word-level redline hunks when you only need per-section change information. Preserve both accession numbers, then fetch each filing record to obtain its SEC URL before opening a filing that affects a decision.

## Limits

- The route compares two filings of one form. It does not find every amendment or build a complete amendment chain.
- A textual difference is not proof of materiality, an accounting restatement, or a legal conclusion.
- The route does not accept the retired `accession_a` and `accession_b` parameters. Use `ticker`, `form`, `from`, and `to`.
