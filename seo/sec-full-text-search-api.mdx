---
title: "SEC Full-Text Search API"
description: "Search filing manifests and extracted section text with issuer, form, accession, and historical filters."
---

`GET /v1/search/fulltext` runs the same query against filing manifests and extracted filing sections. Use it to locate evidence for an exact term or query, then retrieve or open the returned filing before making a claim. It is not semantic search and an empty response is not proof that an issuer never used a concept.

## Run a scoped search

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/search/fulltext?q=material%20weakness&ticker=AAPL&form=10-K&limit=10"
```

`q` is required. Scope it with `ticker` (or `symbol`), `cik`, `form`, `accession_number`, `filing_year`, issuer fiscal year `fy`/`year`, or `date_from`/`date_to`. `limit` is 1 through 100 and is split between the filing and section search legs, so the result is not a single globally paginated list.

The response has `object: "fulltext_search"`, the echoed `query`, `filings`, `sections`, `requestId`, and `traceparent`. Each leg has `data` and `count`; section results can report `degradedState` when section-search capability is partially unavailable.

## Turn a hit into citable evidence

Prefer a section hit for a content claim. Retain its filing identity, section key, source URL/provenance, and request ID, then use the accession-specific section route or open the source filing. A filing-manifest hit can be useful for discovery but is not itself the passage that supports a statement.

Use [semantic search](/seo/sec-semantic-search-api) when the question is about related language rather than exact wording. Use [filing search](/seo/sec-filing-search-api) when the task is selecting filings rather than searching their content.

## Common errors

- Supplying different `ticker` and `symbol` values; they must agree.
- Treating a fiscal-year filter as a calendar filing-year filter. With an issuer identifier, `fy` maps to that issuer's fiscal filing window; `filing_year` is a calendar filing-year selector.
- Ignoring `degradedState` or a non-2xx response when a search backend is unavailable.
- Quoting a snippet without reading the source filing, table, definitions, and surrounding qualification.
