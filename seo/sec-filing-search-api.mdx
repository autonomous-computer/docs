Filing search answers тАЬwhich filing?тАЭ before another endpoint answers тАЬwhat did it say?тАЭ Use it to construct a reviewable universe, then carry the selected accession number into filing or section retrieval.

## Search a bounded filing universe

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings?ticker=NVDA&forms=10-K,10-Q&date_from=2024-01-01&date_to=2025-12-31&limit=25&sort=filing_date_desc"
```

Inputs include `ticker` or `cik`; `form` or `forms`; `date_from` and `date_to` in `YYYY-MM-DD`; `filing_year` or `fy`; `fp`; optional text `q`; `accession_number`; `limit`; and `cursor`. The list response contains records plus `hasMore` and `nextCursor`. Persist the manifestтАЩs `accessionNumber`, `filingDate`, form, filer identity, and filing URL before selecting an item.

Use the accession route when the document is already known:

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/0001045810-24-000029"
```

## Follow search with the right retrieval

| Need after discovery | Route |
| --- | --- |
| Latest filing for one issuer and form | `GET /v1/filings/latest` |
| Named content such as Item 1A | `GET /v1/filings/{accessionNumber}/sections/{sectionKey}` |
| Exact word search in content | `GET /v1/search/fulltext` |
| Related concepts in normalized sections | `GET /v1/search/semantic` |
| Textual changes across two same-form filings | `GET /v1/filings/diff` |

The manifest is metadata about a document, not an answer to a disclosure question. Search content or retrieve the cited section when the research question is about language, risk, or accounting policy.

## SDK and MCP implementation

The JavaScript SDK offers `client.searchFilings(params)`, `client.paginateFilings(params)`, `client.latestFiling(params)`, and `client.filingByAccession(accessionNumber, params)`. In hosted MCP, use `filings.search` for the bounded manifest query and `sections.get` for the follow-up content. `filings.search` accepts issuer, form, text query, and limit; use REST or an SDK when you need the full filter or cursor surface.

## Practical limits

- A form-only latest search can use a current daily-index path and may be terminal (`hasMore: false`); do not invent a page two.
- `filingDate` is when the SEC received the filing. It is not necessarily the report period, fiscal year, market event time, or availability time for every derived surface.
- A no-result list is not evidence that the issuer never filed; check identifier, form family, date bounds, and the [coverage boundary](/coverage-and-depth).
- Keep returned `requestId`, source URL, and accession number with a search result so a later reviewer can reproduce the selection.
