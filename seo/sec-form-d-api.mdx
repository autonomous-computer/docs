# SEC Form D API

Form D is a notice filing for certain exempt offerings. It can be useful for spotting a filing, identifying the issuer, and reading the disclosed offering information. It does not prove that an offering closed, that the entire amount was sold, or that every economic term is public.

SEC API provides two Form D routes:

| Route | Use it for |
| --- | --- |
| `GET /v1/forms/d` | Finding recent Form D and D/A filings by ticker, CIK, filing date, and page cursor. |
| `GET /v1/forms/d/{accessionNumber}` | Reading structured Form D data parsed from the XML for one known accession number. |

## Search filings first

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/forms/d?ticker=EXAMPLE&date_from=2025-01-01&limit=10"
```

The list is a standard JSON envelope with `data`, `hasMore`, `nextCursor`, and `requestId`. Its cursor may be prefixed with `db:` or `sec:` so pagination remains on the source selected for the first page. Pass the returned cursor through unchanged.

Use the `accessionNumber` in a list row to fetch structured data:

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/forms/d/0000000000-00-000000"
```

## How to use Form D responsibly

1. Treat the list route as discovery, not a private-market transaction feed.
2. Inspect the filing and its source URL when an offering matters to your diligence.
3. Keep the accession number with any downstream note or model output so another reader can reproduce the check.
4. Distinguish original Form D filings from amendments (`D/A`); an amendment can revise a prior notice without creating a new financing event.

## Useful filters

`GET /v1/forms/d` accepts `ticker`, `cik`, `date_from`, `date_to`, `limit`, `cursor`, and `submission_file_limit`.

The API does not claim to infer industry, financing round, total capital raised, or an offering outcome from the filing. Use the source filing and your own underwriting process for those conclusions.
