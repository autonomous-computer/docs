---
title: "XBRL API for SEC Filings"
description: "Retrieve normalized SEC XBRL facts and statements with the filing context needed to inspect and cite the result."
---

XBRL is the structured layer inside many SEC filings. The useful question is
rarely “can I parse XBRL?” It is usually “which filed value supports this
model, screen, or research conclusion, and how do I get back to the filing?”

SEC API gives you two complementary paths: normalized statements for a familiar
financial view, and tag-level facts when the accounting concept itself matters.

## Start with the right path

| Need | Start here |
| --- | --- |
| A filed income statement, balance sheet, or cash-flow statement | `GET /v1/statements` |
| The three core statements for one reporting period | `GET /v1/statements/all` |
| A time series for a concept such as revenue or assets | `GET /v1/facts` |
| A company snapshot with optional segment or dilution context | `GET /v1/companies/overview` |

## Retrieve a statement bundle

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/statements/all?ticker=MSFT&period=annual&limit=1"
```

Use `period=annual` or `period=quarterly`, then narrow the request with `fy`,
`fy_from`, or `fy_to` when the comparison requires specific fiscal years.

## Retrieve a fact series

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/facts?ticker=MSFT&tag=Revenues&period=annual&limit=4"
```

Check `requestedTag`, `resolvedTag`, and `aliasStrategy` before treating a
series as a direct match. A fact can include its unit, period, form,
accession number, and filing URL through provenance. Preserve those fields
with the value when an analyst or agent will make a conclusion from it.

## Taxonomy and foreign issuers

Facts default to the US-GAAP path. When neither taxonomy nor form is pinned
and the US-GAAP lookup is empty, the API can fall back to `ifrs-full` for
foreign private issuers. Pinning a taxonomy or form is deliberate: it prevents
that fallback and makes the query narrower. Use an explicit constraint when
your research process requires one.

## Limitations worth keeping visible

- A missing concept is not evidence that a company did not disclose the
  underlying economic item. It can be a tag, taxonomy, or presentation issue.
- Agent views are intentionally compact. Request the default view when your
  system needs full provenance, freshness, materialization, or validation
  envelopes.
- Statements help with comparison; the filing and its notes still resolve
  policy choices, nonstandard labels, and unusual periods.
