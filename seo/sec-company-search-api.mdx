title: "SEC Company Search and Company Data API"
description: "Resolve an SEC issuer, retrieve a company overview, and move from an identifier to source-aware company financial data."
---

Most SEC-data workflows start with an issuer identity problem: someone has a
ticker, a CIK, or a company name and needs the right entity before they can
retrieve filings or financial data. Resolve the issuer first. Then use the
company-data surfaces for the specific question in front of you.

## Resolve the issuer

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/entities/resolve?ticker=AAPL&view=agent"
```

Keep the returned CIK, ticker, match basis, and request ID. A CIK is the most
useful durable SEC identity when a ticker has changed or a filing workflow
needs a precise issuer key.

## Build a company briefing

`GET /v1/companies/overview` accepts a ticker, symbol, or CIK. The base
overview is intentionally light. Add only the enrichment your research needs:

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/companies/overview?ticker=AAPL&include=segments,footnotes,dilution,factors"
```

Segments, dilution, and factors can degrade independently when their source
work is unavailable. Preserve the base overview and inspect each enrichment's
status instead of treating a partial result as a complete company record.

## Choose the next endpoint

| Research job | Endpoint |
| --- | --- |
| Latest company snapshot | `GET /v1/companies/overview` |
| Financial data for one or a small set of issuers | `GET /v1/companies/financials` |
| A specific financial statement | `GET /v1/companies/income-statements`, `GET /v1/companies/balance-sheets`, or `GET /v1/companies/cash-flow-statements` |
| Ratio refresh for ticker-based company sets | `GET /v1/companies/ratios` |
| Product, geographic, or operating segments | `GET /v1/companies/segments` |

For a batch, use `tickers=AAPL,MSFT,NVDA` where the endpoint supports it and
inspect each result for a degraded state. The ratios endpoint accepts a ticker
or ticker batch; it is not a single-CIK lookup.

## Search is not diligence

Entity resolution helps you find the issuer. It does not settle whether a
company changed its reporting perimeter, presentation, or accounting policy.
When a company-data result informs an investment or compliance conclusion,
follow the filing URL, accession number, and relevant disclosure section.
