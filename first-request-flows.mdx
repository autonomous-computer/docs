Start with a company you recognize, then retrieve one filing you can trace back to the SEC. This verifies more than connectivity: it confirms your key, request format, and source record all work together.

## Make the first two requests

Resolve the issuer first. The response gives you its canonical CIK and identity metadata.

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/entities/resolve?ticker=AAPL&view=agent"
```

Then retrieve the issuer's latest annual report.

```bash
curl -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/latest?ticker=AAPL&form=10-K&view=agent"
```

The filing changes when Apple submits a newer annual report. Keep the returned `cik`, `form`, `accessionNumber`, `filingDate`, `filingUrl`, and `requestId`; together they identify the request and the filing behind your work.

## Choose an interface

| Interface | Use it when | Start here |
| --- | --- | --- |
| REST | You want direct HTTP control in any runtime. | [API reference](/api-reference/entities/get-v1-entities-resolve) |
| SDK | The request belongs in JavaScript, Python, Go, or Rust application code. | [Libraries and SDKs](/libraries-and-sdks) |
| CLI | You are working in a terminal or shell script. | [CLI](/cli) |
| MCP | An MCP-aware client needs SEC API tools for a research workflow. | [Install MCP](/mcp-install) |

Use the same issuer-and-filing check in every interface. Do not move on to filing extraction or analysis until you can preserve the source fields from the filing response.

## Continue from the terminal

The CLI is useful when you want to inspect the same workflow without writing application code.

```bash
export SECAPI_API_KEY="secapi_..."
secapi entities resolve --ticker AAPL
secapi filings latest --ticker AAPL --form 10-K
secapi sections get --ticker AAPL --form 10-K --section item_1a
```

## Continue from an MCP client

After you install MCP, ask the client for a bounded, source-backed result instead of a general summary.

<Prompt>
Resolve AAPL, retrieve its latest 10-K, and return the issuer CIK plus the filing form, accession number, filing date, source URL, and request ID. Do not summarize the filing until those fields are present.
</Prompt>

Use the client's tool catalog to discover the current tool names and input schemas. See [MCP workflows](/mcp-workflows) for a source-backed research pattern.

## If the request fails

For `401`, confirm that your key is exported in the calling environment and sent in `x-api-key`. For `402`, resolve the stated billing or entitlement condition. For `429`, wait for `Retry-After` when present; otherwise use bounded exponential backoff and avoid concurrent retries.

Read [API conventions](/api-conventions) for response metadata and error handling, or [Troubleshooting](/troubleshooting) for error-specific next steps.
