# Observability exports

SEC API exposes operator-only observability configuration and export surfaces so support operators can confirm provider seams before live credentials are wired.

Both endpoints require an API key with `admin:operator` scope. Use `SECAPI_OPERATOR_API_KEY` for local support workflows; `SECAPI_OPERATOR_API_KEY` remains supported for existing smoke and release scripts.

## Endpoints

- `GET /v1/observability`
- `GET /v1/observability/export`

## Observability config

```bash
curl "$SECAPI_BASE_URL/v1/observability" \
  -H "x-api-key: $SECAPI_OPERATOR_API_KEY"
```

This reports:

- which providers are configured at the environment level
- whether usage and webhook event logs are enabled
- the current quota table used by the API layer

No secrets are returned.

## Observability export

```bash
curl "$SECAPI_BASE_URL/v1/observability/export?limit=20" \
  -H "x-api-key: $SECAPI_OPERATOR_API_KEY"
```

The export bundles:

- observability config
- current usage summary
- current billing health summary
- recent canonical events

Use this when preparing a support ticket or validating pre-production environments.

The billing summary includes:

- whether a Stripe customer is configured
- current subscription status and billing state
- meter emission backlog counts
- recent billing webhook failure summary

## CLI

```bash
secapi observability show
secapi observability export --limit 20
```

Set `SECAPI_OPERATOR_API_KEY` before running these CLI commands.
