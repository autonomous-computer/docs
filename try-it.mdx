Use this page to make one small production request before you build a larger workflow. Resolve a company first, confirm the returned identity, then use that identity to retrieve a filing, statement, or search result.

<Warning>
These examples call the production API and can count against the authenticated organization's usage. Keep the shown limits while you validate response shapes, then inspect billing and limits before a repeated workload.
</Warning>

## Set an API key

Create an account at [secapi.ai/signup](https://secapi.ai/signup), create an API key in the signed-in [dashboard](https://secapi.ai/app), and expose it only to the process making the request.

```bash
export SECAPI_API_KEY="secapi_..."
```

Send the key in `x-api-key`. Do not put it in browser code, a checked-in file, or an `Authorization: Bearer` header. [API keys, usage, and billing](/auth-and-pricing) explains the boundary between machine credentials and dashboard actions.

## Resolve a company

This request resolves Apple and returns its canonical CIK and identity metadata. Keep the request ID with your logs.

<CodeGroup>
```bash cURL
curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/entities/resolve?ticker=AAPL&view=agent"
```

```javascript JavaScript
const response = await fetch(
  "https://api.secapi.ai/v1/entities/resolve?ticker=AAPL&view=agent",
  { headers: { "x-api-key": process.env.SECAPI_API_KEY } },
)

if (!response.ok) throw new Error(`SEC API request failed: ${response.status}`)
const entity = await response.json()
console.log(entity.ticker, entity.cik, entity.name, entity.requestId)
```

```python Python
import os
import requests

response = requests.get(
    "https://api.secapi.ai/v1/entities/resolve",
    params={"ticker": "AAPL", "view": "agent"},
    headers={"x-api-key": os.environ["SECAPI_API_KEY"]},
    timeout=30,
)
response.raise_for_status()
entity = response.json()
print(entity["ticker"], entity["cik"], entity["name"], entity["requestId"])
```

```bash CLI
secapi entities resolve --ticker AAPL
```
</CodeGroup>

Confirm that the response includes `ticker`, `cik`, `name`, and `requestId`. The [entity resolution reference](/api-reference/entities/get-v1-entities-resolve) documents accepted identifiers and response fields.

## Retrieve a filing after identity is confirmed

```bash
curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/latest?ticker=AAPL&form=10-K&view=agent"
```

Keep `accessionNumber`, `filingDate`, `filingUrl`, and `requestId` with anything derived from the filing. The [latest filing reference](/api-reference/filings/get-v1-filings-latest) documents filters and response fields.
