---
title: "Build a reviewable SEC filing research agent"
description: "Design a bounded research agent that retains source evidence and stops at authorization, billing, and capability boundaries"
---

An SEC research agent should leave behind a trail someone else can inspect: the question, exact source retrieved, fields carried into the answer, and conditions where it stopped. It should not turn a sparse response or ranked search result into a conclusion.

## Define one job

Start with one issuer, one filing family, one date or latest-filing rule, and one output artifact. For example: retrieve NVIDIA's latest 10-K, read Item 1A, and produce a five-bullet risk brief with the filing identity.

```bash
curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/latest?ticker=NVDA&form=10-K&view=agent"
```

The output record should retain `accessionNumber`, `form`, `filingDate`, `filingUrl`, and `requestId` when returned. Preserve route-provided provenance, freshness, materialization, warnings, and trace fields rather than replacing them with a generic confidence score.

## Put the operating policy in the task

| Observed condition | Required behavior |
| --- | --- |
| Evidence is missing, unavailable, degraded, or out of scope | Report that state and the source that was available. Do not fill the gap from model memory. |
| `400` | Correct the arguments from the endpoint reference or discovered MCP schema. |
| `401` or `403` | Stop and repair the credential or organization authorization. |
| `402` | Stop and surface the billing or entitlement requirement. |
| `429` | Honor `Retry-After` when supplied; use bounded backoff without retry fan-out. |
| `5xx` | Apply the caller's bounded transient-failure policy and keep the request identifier. |

Record the endpoint or tool, arguments, retrieval time, source artifact, and request identifier with the final answer. A filing section supports claims about that section; it does not establish facts outside it.

## Match the execution surface to the owner

- Use REST or an SDK when an application owns retries, timeouts, and persistence.
- Use the CLI for terminal inspection and scripts.
- Use hosted MCP when an MCP-aware client must discover tools and schemas during a research session.

For MCP, start with `tools.search`, then inspect the selected input with `tools.describe`. For REST, use the endpoint reference rather than copying an old tool schema into a service. The surfaces expose SEC API data through different contracts, so they do not share error handling or lifecycle behavior.

## Keep repeated runs bounded

Before a loop, read `GET /v1/billing` and `GET /v1/limits`; use `POST /v1/billing/quote` for a current estimate. Quotes do not reserve usage. [Billing-aware agent runs](/agent-billing) covers the account boundary and sponsored credential flow.
