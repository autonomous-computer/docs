---
title: "Migrate from financialdatasets.ai"
description: "Move statement, fact, and filing workflows to SEC API with a field-level dual-run instead of a blind endpoint swap."
---

Use this guide when your application already depends on normalized financial statements and you want to add SEC filing identity and route-specific evidence to the integration. Start with one statement workflow, not every dashboard at once.

## Map the job, not just the URL

| Current job | SEC API route | Validate before cutover |
| --- | --- | --- |
| Income statement rows | `GET /v1/statements/income_statement` | Period, units, statement rows, and source filing context. |
| Balance-sheet rows | `GET /v1/statements/balance_sheet` | Point-in-time date, units, and source filing context. |
| Cash-flow rows | `GET /v1/statements/cash_flow_statement` | Duration, period basis, units, and source filing context. |
| Tag-level reported observation | `GET /v1/facts` | Taxonomy tag, unit, form, fiscal period, and accession. |
| Filing inventory or latest filing | `GET /v1/filings` or `GET /v1/filings/latest` | Form, filing date, accession number, URL, and pagination. |
| Insider activity | `GET /v1/insiders` | Form family, transaction date, and returned coverage fields. |

Route names are not a field-level compatibility guarantee. Normalized values remain issuer disclosures, so compare reporting period, units, `periodBasis` where supplied, and returned provenance before treating two rows as equivalent.

## Dual-run one annual statement

```bash
curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/statements/income_statement?ticker=AAPL&period=annual&limit=4"
```

Record the requested ticker and parameters, retrieval time, `requestId`, source filing identifiers, period, units, and each field your application displays. A matching headline revenue number is useful, but it is not enough to prove equivalent period or filing provenance.

## Add facts when a statement row is too broad

Use the facts route when the application needs a named reported concept rather than a statement presentation:

```bash
curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/facts?ticker=AAPL&tag=Revenues&form=10-K&limit=4"
```

Do not coerce missing observations to zero or assume tags are comparable across issuers. Keep the selected tag, unit, fiscal period, and filing identity with the value.

## Cut over in small, reversible steps

1. Resolve one issuer and dual-run one statement request.
2. Compare the fields your application actually consumes, including nulls and periods.
3. Store the SEC API filing identity and response diagnostics beside the result.
4. Move one read path behind a feature flag or equivalent reversible release control.
5. Add filing or fact routes only after the statement acceptance checks pass.

For unavailable, partial, or degraded results, preserve the returned condition and show it to the caller. A filing's existence does not guarantee every normalized representation is ready or comparable.
