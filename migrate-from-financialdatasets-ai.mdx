description: "Dual-run one financial-statement read against SEC API and verify periods, units, and filing provenance before cutover."
---

Use this guide to move one normalized-statement read without losing the filing identity behind each value. Start with the screen, job, or downstream calculation affected by a changed period, unit, or source filing.

## Map the consuming contract

| Current job | SEC API route | What must match |
| --- | --- | --- |
| Income statement | `GET /v1/statements/income_statement` | Period, units, rows, and source filing context. |
| Balance sheet | `GET /v1/statements/balance_sheet` | Point-in-time date, units, and source filing context. |
| Cash-flow statement | `GET /v1/statements/cash_flow_statement` | Duration, period basis, units, and source filing context. |
| Reported concept | `GET /v1/facts` | Tag, unit, fiscal period, form, and accession number. |
| Filing inventory | `GET /v1/filings` | Form, filing date, accession number, URL, and pagination. |

Matching a headline number is not sufficient. A normalized value remains a reported disclosure, so compare the period, unit, presentation, and returned source fields your application uses.

## Dual-run one annual statement read

```bash
curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/statements/income_statement?ticker=AAPL&period=annual&limit=4"
```

Persist the request parameters, retrieval time, `requestId`, period, units, source filing identifiers, and each field that your application displays. Compare nulls as carefully as populated values.

## Test a named disclosure separately

When your consumer needs a reported concept rather than a statement presentation, qualify the tag directly:

```bash
curl --fail-with-body -sS \
  -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/facts?ticker=AAPL&tag=Revenues&form=10-K&limit=4"
```

Do not coerce absent observations to zero or assume the same tag is comparable across issuers. Keep the selected tag, unit, fiscal period, and filing identity with the value.

## Implement a reversible cutover

1. Dual-run one issuer and annual statement.
2. Compare the fields, periods, units, and nulls your application consumes.
3. Store SEC API source identifiers and diagnostics alongside the result.
4. Put one read path behind a reversible release control.
5. Add facts or additional statement families only after the acceptance test passes.

A filing existing in EDGAR does not guarantee that every normalized representation is ready or comparable. Preserve partial or unavailable conditions rather than turning them into a silent fallback.
