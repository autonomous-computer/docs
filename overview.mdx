## A practical path through the docs

<Steps>
  <Step title="Prove the request">
    Create an API key, resolve an issuer, and retrieve a filing. This verifies your credentials, request format, and the source fields you will retain.
  </Step>
  <Step title="Choose the integration surface">
    Start with REST when you need to understand a response. Move to an SDK, CLI, or MCP when it better fits the application or research environment.
  </Step>
  <Step title="Retain the source context">
    Keep accession numbers, filing URLs, filing dates, and request metadata with any conclusion derived from a response.
  </Step>
</Steps>

<Prompt>
