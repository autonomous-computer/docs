## Do new organizations receive included usage?

Every new organization receives `150` included calls. This starter grant is not a recurring monthly allowance. Check `GET /v1/billing` for the authenticated organization's remaining included usage and current billing state.

## What happens after the included calls run out?

Billable requests require an active paid billing path. You can activate Pay As You Go or choose a paid plan in the signed-in dashboard. A request that cannot proceed returns a billing or entitlement error with the condition to resolve.

## Can an agent bootstrap itself?

Not anonymously. A signed-in administrator creates a short-lived sponsor token through `POST /v1/agent/bootstrap_tokens`; the agent exchanges it once through `POST /v1/agent/bootstrap`. Read [agent billing](/agent-billing) before using this flow in an automated environment.

## Do I need a subscription to use the API?

No. Pay As You Go is the default paid path. Personal and Team plans suit sustained internal usage, while Commercial covers external-facing use and negotiated terms.

## Which plans include redistribution rights?

Commercial. Use a commercial agreement before redistributing, embedding, white-labeling, or reselling SEC API-derived output. See [plans and pricing](/plans-and-pricing) for the rights boundary.

## Where do I view invoices and payment methods?

Use the Billing area of the signed-in dashboard. Payment methods, invoices, checkout, and billing-portal actions require a browser session with billing access; API keys cannot perform them.

## How do I keep an agent from spending too much?

Use `POST /v1/billing/quote` before an expensive loop. A billing administrator can set organization-level budget controls in the dashboard; your application should handle the billing condition returned when a control blocks a request.

## What counts as an AI query?

The `ai_queries` quota applies to documented intelligence and enhanced dilution routes. The current response is the source of truth for whether a request counted and how much quota remains. The quota-counted routes include:

- Every `/v1/intelligence/*` request (security, company, country report, footnotes query, portfolio, watchlist, query/poll)
- `GET /v1/signals/dilution/enhanced` (and any sub-paths)
- MCP `tools/call` requests for `intelligence.query` and `signals.dilution.enhanced.get`

Other meter classes, such as filing reads, factor lookups, and market data, do not consume this quota.

## What are the per-tier ai_queries limits?

| Plan | Monthly AI queries |
| --- | --- |
| Sandbox grant | 10 |
| Pay As You Go | Uncapped; metered per call |
| Personal | Uncapped; governed by budget controls |
| Team | Uncapped; governed by budget controls |
| Commercial | Uncapped; governed by budget controls |

Paid plans do not have a fixed monthly `ai_queries` cap. Their usage is governed by billing and budget controls instead. The sandbox grant retains its 10-query monthly allocation.

## When does my AI query quota reset?

Calendar-month reset, in UTC. The `SECAPI-AI-Quota-Reset` response header carries the exact ISO timestamp (the first day of the next UTC month at `00:00:00Z`).

## What happens when I exhaust my ai_queries quota?

Only the sandbox grant has a fixed `ai_queries` cap. When it is exhausted, quota-counted REST routes return `429 Too Many Requests` with `code: "ai_query_quota_exceeded"`. MCP returns an HTTP `429` JSON-RPC error with the same machine-readable code. Inspect `SECAPI-AI-Quota-*` headers on quota-counted responses for current usage and reset information.

For paid plans, resolve the billing budget condition returned by the API rather than expecting the quota-exhaustion response.

## Can I see my current ai_queries usage?

`GET /v1/billing` includes `monthlyQuotas.ai_queries` with `limit`, `used`, `remaining`, and `resetAt`. Quota-counted responses also include `SECAPI-AI-Quota-*` headers.

## Is public account creation available?

Create an account at [secapi.ai/signup](https://secapi.ai/signup), then create API keys in the signed-in dashboard. The signup flow presents the current account and terms requirements.

## Can an API key manage billing or create more keys?

No. API keys are machine credentials for data access. Creating or revoking keys, changing budgets, managing billing, updating dashboard settings, and requesting account deletion require a signed-in browser session with the necessary permissions.
