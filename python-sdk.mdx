Use the Python SDK for research code, jobs, notebooks, and services that need the SEC API contract without hand-writing HTTP requests. Keep the API key outside notebooks shared with other people and outside client-side applications.

## Install and configure

```bash
pip install secapi-client
export SECAPI_API_KEY="secapi_..."
```

```python
import os
from secapi_client import SecApiClient

client = SecApiClient(api_key=os.environ["SECAPI_API_KEY"])
```

The client uses `https://api.secapi.ai` by default. Use a different base URL only for an environment you control.

## Make a first filing request

```python
issuer = client.resolve_entity(ticker="AAPL")
filing = client.agent_latest_filing(ticker="AAPL", form="10-K")

print({
    "ticker": issuer["ticker"],
    "cik": issuer["cik"],
    "accessionNumber": filing["accessionNumber"],
    "filingUrl": filing["filingUrl"],
    "requestId": filing["requestId"],
})
```

Use `agent_latest_filing` when the compact retrieval shape is enough. Check the [latest filings reference](/api-reference/filings/get-v1-filings-latest) for the full response and supported filters.

## Retrieve a cited section

```python
risk_factors = client.agent_section(
    "item_1a",
    ticker="AAPL",
    form="10-K",
)
```

Keep the filing accession, filing URL, request ID, and returned freshness or provenance fields with a generated memo, table, or model input.

## Handle errors

```python
from secapi_client import SecApiError

try:
    client.resolve_entity(ticker="AAPL")
except SecApiError as error:
    print({
        "status": error.status,
        "code": error.code,
        "requestId": error.request_id,
        "message": error.message,
    })
```

The SDK retries safe read requests. Do not retry a mutation unless your caller can safely repeat it. For response codes and bounded backoff, read [Troubleshooting](/troubleshooting) and [SDK reliability](/sdk-reliability).
