The Python SDK is built for research, ETL, notebooks, and backend services that want REST parity without turning every request into a small framework project.
</Info>

<Steps>

## First useful calls

```python
from secapi_client import SecApiClient

client = SecApiClient(
    api_key="secapi_test_...",
    base_url="https://api.secapi.ai",
)

issuer = client.resolve_entity(ticker="AAPL")
filing = client.agent_latest_filing(ticker="AAPL", form="10-K")
risk_factors = client.agent_section("item_1a", ticker="AAPL", form="10-K")
income = client.agent_statement("income_statement", ticker="AAPL", period="annual", limit=3)
holders = client.agent_institutional_holders(ticker="NVDA", limit=10)
form144 = client.agent_form_144(ticker="NVDA", limit=10)
mcp_section = client.call_mcp_tool(
    "sections.get",
    {"ticker": "AAPL", "form": "10-K", "sectionKey": "item_1a", "mode": "compact"},
    id="aapl-item-1a",
)
```

## Reliability defaults

The Python SDK retries transient failures with exponential backoff and a per-client circuit breaker.

- Auto-retried by default: `GET`, `HEAD`, `OPTIONS`
- Opt-in required: `POST`, `PUT`, `PATCH`, `DELETE`, MCP `tools/call`
- Always retried regardless of method: `429` rate limits, with `Retry-After` or structured retry timing honored
- Request timeout: `30s` by default, including when retries are disabled

Disable SDK retries when your application already owns retry behavior:

```python
client = SecApiClient(api_key="secapi_test_...", retry=False, telemetry=False, timeout=10)
```

Set `timeout=None` only when your own transport wrapper enforces socket deadlines.

Opt into mutating retries only when the operation is idempotent:

```python
artifact = client.create_artifact(
    {"kind": "markdown_bundle", "ticker": "AAPL"},
    retry={"enabled": True, "idempotency_key": "artifact-aapl-markdown-bundle"},
)
```

See [SDK reliability](/sdk-reliability) for the full retry policy and migration notes.

## Error handling

```python
from secapi_client import SecApiClient, SecApiError

client = SecApiClient()

try:
    client.resolve_entity(ticker="INVALID")
except SecApiError as error:
    print(error.status)
    print(error.code)
    print(error.request_id)
    print(error.message)
    print(error.hint)
    print(error.docs_url)
    print(error.action)
    print(error.retryable)
    print(error.details)
    print(error.payload)
```

`SecApiError` keeps the raw `payload` and also exposes `code`, `message`, `request_id`, `hint`, `docs_url`, `action`, `retryable`, and `details` convenience properties. Log those fields with the failing workflow so support can trace the exact request without asking users to parse JSON by hand.

## Dilution intelligence

```python
# Top 10 highest-dilution-risk issuers
ratings = client.dilution_ratings(overall_risk="high", limit=10)

# Single-ticker dilution rollup
score = client.dilution_score(ticker="BBBB")

# ATM offerings only (boolean filter on the events list)
atms = client.dilution_events(ticker="BBBB", is_atm=True)

# Coverage + freshness rollup (no agent shape — already compact)
coverage = client.dilution_coverage(ticker="BBBB")
```

The full surface — `dilution_events`, `dilution_event_detail`, `dilution_warrants`, `dilution_convertibles`, `dilution_rofr`, `dilution_lockups`, `dilution_cash_position`, `dilution_corporate_actions`, `dilution_nasdaq_compliance`, `dilution_ratings`, `dilution_reverse_splits`, `dilution_score`, `dilution_share_float_history`, `dilution_coverage` — mirrors the REST routes 1:1. Add `view="agent"` to any list call (except `dilution_coverage`) for the compact agent-mode shape.

## Admin paths

The observability methods require an admin-scoped key (`admin:operator`).

```python
admin_client = SecApiClient(api_key=os.environ["SECAPI_API_KEY"])
artifact = client.create_artifact({
    "ticker": "AAPL",
    "form": "10-K",
    "sectionKey": "item_1a",
    "kind": "markdown_bundle",
})
manifest = client.artifact_manifest(artifact["id"])
observability = admin_client.observability()
```

<Warning>
If you are caching responses in a notebook or long-running worker, keep the freshness block next to the payload. Otherwise you lose the context that tells you whether a result is still safe to trust.
</Warning>
