Use the Python SDK in research code, jobs, notebooks, and services when you want a client wrapper around the REST contract. Keep the API key out of shared notebooks and client-side applications.

## Install and configure

```bash
pip install secapi-client
export SECAPI_API_KEY="secapi_..."
```

```python
from secapi_client import SecApiClient

client = SecApiClient()
```

By default, the client reads `SECAPI_API_KEY` and uses `https://api.secapi.ai`. Use a different base URL only for an environment you control.

## Make a first request

```python
issuer = client.resolve_entity(ticker="AAPL")
filing = client.agent_latest_filing(ticker="AAPL", form="10-K")

print({
    "ticker": issuer["ticker"],
    "cik": issuer["cik"],
    "accessionNumber": filing["accessionNumber"],
    "filingUrl": filing["filingUrl"],
    "requestId": filing["requestId"],
})
```

`resolve_entity` calls `GET /v1/entities/resolve`. `agent_latest_filing` calls the latest-filings endpoint with `view=agent`. Values such as the accession number and request ID vary, so use this output to confirm the request and preserve it with downstream work. Check the [entity resolution reference](/api-reference/entities/get-v1-entities-resolve) and [latest filings reference](/api-reference/filings/get-v1-filings-latest) for accepted parameters, response fields, and endpoint limits.

## Retrieve a cited section

```python
risk_factors = client.agent_section(
    "item_1a",
    ticker="AAPL",
    form="10-K",
)
```

The helper retrieves the latest matching section in compact mode. Keep the filing accession, filing URL, and request ID with a generated memo, table, or model input.

## Handle errors

```python
from secapi_client import SecApiError

try:
    client.resolve_entity(ticker="AAPL")
except SecApiError as error:
    print({
        "status": error.status,
        "code": error.code,
        "requestId": error.request_id,
        "message": error.message,
    })
```

The SDK's documented retry policy covers safe reads. For `429`, let the client honor the returned retry timing; do not add concurrent retry loops. Do not retry a mutation unless your caller can safely repeat it. For response codes and bounded backoff, read [Troubleshooting](/troubleshooting) and [SDK reliability](/sdk-reliability).
