Return recent normalized insider trading records derived from Forms 3, 4, and 5 with date filters and cursor pagination

<Info>
Audience: application and coding agent.
</Info>

## Agent mode

Pass `?view=agent` to receive the compact agent shape: trade essentials (ticker, form, dates, code, direction, security title, shares, price, value, accession) + lifted `filingUrl`. Ownership flags (`isDirector`, `isOfficer`, `sharesOwnedPrior/Following`) and metadata envelopes are dropped. Typical reduction: **~55%**. See [api conventions → response formats](/api-conventions) for the full `?view=` contract.

## Canonical metadata

- `requestId`
- `traceparent`
- `provenance`
- `freshness`
- `materialization`

## Example request

<RequestExample>
```bash
curl -X GET -H "x-api-key: $SECAPI_API_KEY" -H "secapi-version: 2026-03-19" "https://api.secapi.ai/v1/insiders?ticker=AAPL&forms=4&limit=10"
```
</RequestExample>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
