---
title: "GET /v1/embed/situations/:id"
description: "Public redacted situation detail for embeddable discovery surfaces"
---

Public, anonymous detail for a recent situation and its filing-event timeline. No API key is required.

<Info>
The response is a public projection. Provider-owned keys and verification internals are not exposed.
</Info>

## Path parameters

- `id` - Canonical situation ID in the form `sit_` followed by 20 lowercase hexadecimal characters.

## Example request

<RequestExample>
```bash
curl "https://api.secapi.ai/v1/embed/situations/sit_0123456789abcdef0123"
```
</RequestExample>

## Response fields

The top-level object is a `situation`. It includes identity and status, issuer fields, `headline`, optional `summaryMd` and `businessProfile`, structured `terms`, `keyDates`, and `market` data, plus `sourceAccessions`, `eventCount`, and `latestEventAt`.

`events` contains public filing events, including occurrence time, SEC accession number, form type, optional 8-K items and category, title, optional summary, status transition, and optional `documentUrl`.

The `verification` object is retained for schema compatibility, but source-span, extraction-trace, confidence, cross-validation, model-version, and verification-time fields are redacted on this public surface.

## Failure posture

- Treat non-2xx responses as contract-aware errors.
- Preserve `requestId` and `traceparent` from error responses in logs.
- Use `sourceAccessions`, event accession numbers, and `documentUrl` when present as public source references; do not infer unreturned verification details.
