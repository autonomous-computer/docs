# GET /v1/filings/latest/sections/{section_key}

Extract a section from the latest filing for an entity and form

<Info>
Use this endpoint from server-side applications, SDKs, or agent workflows.
</Info>

## Agent mode

Pass `?view=agent` to receive citation fields and a snippet without the full `contentMd` section body. Use `mode=compact` when you still need `contentMd` but want it capped for token efficiency.

## Agent response fields

- `requestId`
- `object`
- `ticker`
- `key`
- `snippet`
- `accessionNumber`
- `accession`
- `section_key`
- `source_url`
- `startOffset`
- `endOffset`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/filings/latest/sections/item_1a?ticker=AAPL&form=10-K&view=agent"
```
</RequestExample>

## Errors and partial results

- Treat non-2xx responses as API errors and log the response body for debugging
- Log `requestId`, `accessionNumber`, and `source_url` so SEC API support can trace the request
