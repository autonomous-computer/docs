Extract disclosed pension and retiree benefit payments for a target year

<Info>
Audience: application and coding agent.
</Info>

## Coverage and source scope

Extract expected pension and retiree benefit payments for a requested target year from a selected issuer filing when the rendered source discloses a usable schedule. Supply an issuer, filing year, and target year; preserve the source accession and provenance. An unavailable result is not a zero payment or a complete benefits forecast.

## Disclosure-aware schedule extraction

Supply one issuer selector plus the required `filing_year` and `target_year`; use `accession_number` when the disclosure vintage must be fixed. The route extracts a disclosed schedule from rendered filing tables, so preserve provenance and treat unavailable rows as unavailable disclosure rather than zero expected payments. See [filing and source conventions](/api-conventions) before comparing schedule values across filing vintages.

## Canonical metadata

- `requestId`
- `traceparent`
- `provenance`
- `freshness`
- `materialization`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/filings/pension-benefit-schedule?ticker=VZ&filing_year=2022&target_year=2024"
```
</RequestExample>

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
