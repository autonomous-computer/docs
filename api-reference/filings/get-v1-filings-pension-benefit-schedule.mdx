# GET /v1/filings/pension-benefit-schedule

Return structured expected pension and retiree benefit payments for a target year when the filing discloses the schedule in rendered tables

<Info>
Use this endpoint from server-side applications, SDKs, or agent workflows.
</Info>

## Issuer lookup

Use `ticker`, `symbol`, or `cik` to choose the issuer. `symbol` is an alias for `ticker`; if both are provided, they must match.

## Response metadata

- `requestId`
- `traceparent`
- `provenance`
- `freshness`
- `materialization`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/filings/pension-benefit-schedule?symbol=VZ&filing_year=2022&target_year=2024"
```
</RequestExample>

## Give this prompt to your agent

Use SEC API GET /v1/filings/pension-benefit-schedule with `ticker`, `symbol`, or `cik` plus `filing_year` and `target_year` to return structured expected pension and retiree benefit payments when the filing discloses the schedule in rendered tables. Preserve `requestId`, `traceparent`, `provenance`, `freshness`, `materialization` in the output. Keep the response fields your application uses for follow-up requests.

## Errors and partial results

- Treat non-2xx responses as API errors and log the response body for debugging
- Log `requestId` and `traceparent` so SEC API support can trace the request
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
