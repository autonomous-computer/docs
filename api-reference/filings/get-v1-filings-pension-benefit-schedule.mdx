# GET /v1/filings/pension-benefit-schedule

Return structured expected pension and retiree benefit payments for a target year when the filing discloses the schedule in rendered tables

<Info>
Audience: application and coding agent.
</Info>

## Issuer lookup

Use `ticker`, `symbol`, or `cik` to choose the issuer. `symbol` is an alias for `ticker`; if both are provided, they must match.

## Canonical metadata

- `requestId`
- `traceparent`
- `provenance`
- `freshness`
- `materialization`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/filings/pension-benefit-schedule?symbol=VZ&filing_year=2022&target_year=2024"
```
</RequestExample>

## Give this prompt to your agent

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
