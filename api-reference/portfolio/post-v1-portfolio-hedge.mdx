# POST /v1/portfolio/hedge

Return constrained factor hedge candidates for a portfolio with compact residual exposure and trust metadata

<Info>
Use this endpoint from server-side applications, SDKs, or agent workflows.
</Info>

## Response metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X POST \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  -H "content-type: application/json" \
  -d '{
    "holdings": [
      {"symbol": "AAPL", "weight": 0.35},
      {"symbol": "MSFT", "weight": 0.30},
      {"symbol": "NVDA", "weight": 0.20},
      {"symbol": "JPM", "weight": 0.15}
    ],
    "objective": "factor_neutral",
    "constraints": {
      "maxHedges": 3,
      "maxPositionWeight": 0.10,
      "allowedInstrumentTypes": ["etf"]
    }
  }' \
  "https://api.secapi.ai/v1/portfolio/hedge?response_mode=compact"
```
</RequestExample>

## Give this prompt to your agent

<Prompt description="Copy this SEC API prompt for your agent.">
Use SEC API POST /v1/portfolio/hedge to generate constrained hedge candidates for unwanted factor exposures. Preserve candidate instruments, expected exposure reduction, residual exposure, constraints, freshness, and methodology metadata. Treat the output as a hedge construction input, not an execution instruction.
</Prompt>

## Errors and partial results

- Treat non-2xx responses as API errors and log the response body for debugging
- Log `requestId` and `traceparent` so SEC API support can trace the request
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
