# POST /v1/portfolio/stress-test

Run portfolio stress scenarios across factor and macro shock definitions with compact traceable outputs

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X POST \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  -H "content-type: application/json" \
  -d '{"country":"US","lookback":"12m","category":"style","keys":["VALUE","MOMENTUM","QUALITY"],"scenarioKey":"higher_for_longer","holdings":[{"symbol":"AAPL","weight":0.35},{"symbol":"MSFT","weight":0.3},{"symbol":"NVDA","weight":0.2},{"symbol":"JPM","weight":0.15}]}' \
  "https://api.secapi.ai/v1/portfolio/stress-test?response_mode=compact&include=trust"
```
</RequestExample>

## Give this prompt to your agent

<Prompt>
Use SEC API POST /v1/portfolio/stress-test for scenario-based factor stress testing. Preserve `scenarioKey`, `scenarioLabel`, `estimatedDrawdownPercent`, `factorShocks`, `macroShocks`, `conditioningNotes`, `summaryMd`, `freshness`, `methodology`, `requestId`, and `traceparent`.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
