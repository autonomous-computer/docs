# POST /v1/intelligence/query

Compile an allocator prompt into a deterministic one-call intelligence workload with compact traced output.

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X POST \
  -H "x-api-key: $OMNI_DATASTREAM_API_KEY" \
  -H "omni-version: 2026-03-19" \
  -H "content-type: application/json" \
  -d '{
    "query": "Decompose AAPL'\''s last 6M return into factors, macro drivers, filing-specific risks, then suggest hedges.",
    "entities": ["AAPL"],
    "lookback": "6m",
    "response_mode": "compact_json_and_md"
  }' \
  "https://api.secapi.ai/v1/intelligence/query"
```
</RequestExample>

## Launch prompt patterns

- return decomposition: `"Decompose AAPL's last 6M return into factors, macro drivers, filing-specific risks, then suggest hedges."`
- factor neutralization: `"Give me a recommendation for how to make this portfolio factor neutral."`
- regime-aware screen: `"Run a regime-aware quality screen for tightening cycles."`
- company brief: `"Give me a compact brief on NVDA with filings, factor context, and catalysts."`

## Async lifecycle

- bounded workloads return `200` with the final compact result
- wide workloads return `202` with `jobId`, `pollUrl`, and `artifactId`
- poll with [`GET /v1/intelligence/query/{jobId}`](/api-reference/intelligence/get-v1-intelligence-query-jobId)

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
