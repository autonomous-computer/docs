## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- preserve `queryPlan` and `queryPlanManifest` when handing results to downstream agents so they can see matched signals, missing inputs, and validation state
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
- the planner now requires an explicit issuer, portfolio, manager CIK, or country for the matching workload instead of silently defaulting broad prompts onto an arbitrary target
