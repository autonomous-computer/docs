# GET /v1/intelligence/company

Return a one-call company bundle spanning market, filings, ownership, macro, factor, and risk context

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $OMNI_DATASTREAM_API_KEY" \
  -H "omni-version: 2026-03-19" \
  "https://api.secapi.ai/v1/intelligence/company?ticker=AAPL"
```
</RequestExample>

## Example response

<ResponseExample>
```json
{
  "object": "company_intelligence_bundle",
  "symbol": "AAPL",
  "reference": { "symbol": "AAPL", "exchange": "XNAS" },
  "estimates": [{ "metric": "eps", "fiscalPeriod": "Q2", "fiscalYear": 2026, "value": 1.42 }],
  "macroRegimes": [{ "key": "higher_for_longer", "country": "US" }],
  "factorExposures": [{ "factorKey": "MKT_US", "beta": 1.18 }],
  "keyRisks": [{ "key": "regulatory_legal", "severity": "medium" }],
  "requestId": "req_example_123"
}
```
</ResponseExample>

## Choose the right surface

- use `company` when you want market, estimates, macro, factor, and filing risk in one read
- use [`security`](/api-reference/intelligence/get-v1-intelligence-security) when you want a lighter quote-and-catalyst bundle
- use [`query`](/api-reference/intelligence/post-v1-intelligence-query) when the caller already has a natural-language allocator task

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
