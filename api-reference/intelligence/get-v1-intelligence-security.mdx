# GET /v1/intelligence/security

Return a one-call security bundle with snapshot, catalysts, filings, and factor context

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $OMNI_DATASTREAM_API_KEY" \
  -H "omni-version: 2026-03-19" \
  "https://api.secapi.ai/v1/intelligence/security?ticker=AAPL"
```
</RequestExample>

## Example response

<ResponseExample>
```json
{
  "object": "security_intelligence_bundle",
  "symbol": "AAPL",
  "snapshot": { "symbol": "AAPL", "price": 273.76, "changePercent": -0.0044 },
  "reference": { "symbol": "AAPL", "exchange": "XNAS" },
  "recentBars": [{ "timestamp": "2026-03-24T20:00:00.000Z", "close": 273.76 }],
  "catalysts": [{ "headline": "Apple Inc. 8-K", "sourceName": "SEC EDGAR" }],
  "factorExposures": [{ "factorKey": "MKT_US", "beta": 1.18 }],
  "requestId": "req_example_123"
}
```
</ResponseExample>

## Choose the right surface

- use `security` for quotes, short-horizon catalysts, recent bars, and filing traces
- use [`company`](/api-reference/intelligence/get-v1-intelligence-company) when you also need estimates, macro regime, and filing risk
- use [`query`](/api-reference/intelligence/post-v1-intelligence-query) for prompt-driven one-call analysis

## Give this prompt to your agent

<Prompt>
Use Omni Datastream GET /v1/intelligence/security to return a one-call security bundle with snapshot, catalysts, filings, and factor context. Preserve `requestId`, `traceparent` in the output. Return a concise markdown summary plus the exact structured payload fields that a downstream engineer or agent should keep using this result.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
