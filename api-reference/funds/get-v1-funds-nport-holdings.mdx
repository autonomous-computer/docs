# GET /v1/funds/nport/holdings

Return the latest SEC N-PORT holdings roster with explicit capability semantics and balance-unit metadata

<Info>
Use this endpoint from server-side applications, SDKs, or agent workflows.
</Info>

## Agent mode

Pass `?view=agent` to receive the compact agent shape: fund identifiers (ticker, cik, fundName, form, filingDate, accessionNumber, holdingCount) + lifted `filingUrl` + nested `holdings[]` trimmed to 6 fields per holding (issuer, cusip, valueUsd, balance, balanceUnit, pctNetAssets). Title, isin, assetType, issuerType, country, currency, `summaryMd`, and metadata envelopes are dropped. Typical reduction: **~60%**. See [api conventions → response formats](/api-conventions) for the full `?view=` contract.

## Response metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SEC_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/funds/nport/holdings"
```
</RequestExample>

## Errors and partial results

- Treat non-2xx responses as API errors and log the response body for debugging
- Log `requestId` and `traceparent` so SEC API support can trace the request
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
