# GET /v1/companies/overview

Return a first-party SEC company overview — a due-diligence briefing combining identity/classification (name, CIK, tickers, sector/industry from SIC, state of incorporation, exchange, former names) with the latest material filing and a sector-aware financial snapshot derived from multi-year annual SEC XBRL facts. A companyType discriminator (operating / financial_institution / pre_revenue) drives which metrics are valid: financial institutions null gross/operating margin and free cash flow and lean on net margin, ROE, ROA, book value, and total assets. The revenue concept is recency-selected (the candidate whose latest fiscal year is most recent wins) so deprecated tags and wrong sub-lines never win. The base overview is intentionally light; pass include=segments,footnotes,dilution,factors to attach opt-in, bounded enrichments (footnotes is returned as a reference link rather than rendered inline; factors is a quantified, time-ranged quant factor-exposure snapshot — an exposure snapshot, not a forecast). Ships beta. Requires ticker or cik.

## Enrichment architecture and latency posture

The base overview (identity + latest material filing + financial snapshot) is a
LIGHT read: it serves the cached SEC submissions payload plus companyfacts-backed
XBRL reads. It never renders a filing and never runs segmented submission-file
extraction, so it stays well within the customer-facing request budget.

Heavy work is opt-in via `include` (comma list of `segments`, `footnotes`,
`dilution`, `factors`) and is bounded so it can never block or fail the base
snapshot:

- `dilution` runs inline — a single light database read mapped to a compact
  risk band (`overallRisk`, `dilutionRisk`, `numericScore`).
- `segments` runs inline but only through a shared, process-wide concurrency
  slot (the same guard `/v1/companies/segments` uses). If the slot is at
  capacity the enrichment returns `status: "error"` rather than a 429 on the
  whole overview.
- `factors` runs inline via the cache-first light read that backs
  `/v1/factors/exposures` — it never triggers a factor recompute or
  materialization. It surfaces the issuer stock's exposure to the launch factors
  as a **quantified, time-ranged snapshot**: each exposure carries `beta` (the
  model loading), `percentileVsUniverse` (where the stock ranks vs the factor
  universe, 0–100; `null` when the model did not materialize a percentile —
  beta + `asOf` + `window` are still returned, never a heavy computed
  percentile), `asOf`, and the estimation `window`, plus a top-level `asOf` and a
  `methodology` link. It requires a ticker (exposures are per-stock); a cik-only
  request that cannot resolve a ticker returns `status: "error"` with code
  `factors_requires_ticker`. This is an **exposure snapshot, not a forecast** —
  it never implies predictive power.
- `footnotes` is returned as a **reference**, not rendered inline. A latest 10-K
  render is CPU-heavy and uninterruptible, so the overview returns the endpoint +
  query the client should call on its own request budget (`POST
  /v1/intelligence/footnotes/query`).

Each enrichment carries its own status field (`included` / `reference` /
`error` / `not_requested`). A failed or referenced enrichment never affects the
base overview.

## companyType-aware financial snapshot

`financialSnapshot.companyType` is one of `operating`, `financial_institution`,
or `pre_revenue` and drives which metrics are emitted:

- `financial_institution` (SIC 6000–6999, or null revenue with interest/premium
  facts): gross margin, operating margin, and free cash flow are `null`; the
  snapshot leans on net margin, ROE, ROA, book value, and total assets.
- `pre_revenue` (clinical-stage filers with negligible revenue vs operating
  expense): margins that would divide by ~zero revenue are `null`.
- `operating`: the full metric set.

The revenue series is recency-selected: among the candidate revenue concepts the
one whose latest fiscal year is most recent wins (tiebreak: longest series), so
deprecated tags and narrow sub-lines never displace the current top line. Any
metric that cannot be sourced reliably is `null` rather than guessed.

<RequestExample>

```bash
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/companies/overview?ticker=AAPL&include=dilution,footnotes,factors"
```
</RequestExample>
