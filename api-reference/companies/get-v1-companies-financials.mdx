# GET /v1/companies/financials

Return unified financial statements (income statement, balance sheet, cash flow statement) for a company in a single call

<Info>
Audience: application and coding agent.
</Info>

## Query parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `ticker` | string | Stock ticker symbol (e.g., AAPL) |
| `cik` | string | SEC CIK number |
| `period` | string | `annual` or `quarterly` (default: `annual`) |
| `limit` | integer | Number of periods to return (default: 4, max: 50) |

## Canonical metadata

- `requestId`
- `traceparent`
- `lineage` — source attribution (`sec_xbrl`)

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $OMNI_DATASTREAM_API_KEY" \
  -H "omni-version: 2026-03-19" \
  "https://api.secapi.ai/v1/companies/financials?ticker=AAPL&period=annual&limit=4"
```
</RequestExample>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
