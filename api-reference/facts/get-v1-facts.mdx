# GET /v1/facts

Return normalized SEC company facts for an issuer, concept, and optional form or unit

<Info>
Use this endpoint from server-side applications, SDKs, or agent workflows.
</Info>

## Agent mode

Pass `?view=agent` to receive the compact agent shape: concept + value + unit + period fields + lifted `accessionNumber` / `filingUrl` from provenance. Metadata envelopes are dropped. See [api conventions → response formats](/api-conventions) for the full `?view=` contract.

## Response metadata

- `requestId`
- `traceparent`
- `provenance`
- `freshness`
- `materialization`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/facts?ticker=AAPL&tag=Revenues&period=quarter&limit=4&view=agent"
```
</RequestExample>

## Period aliases

Use `period=annual` for annual company facts or `period=quarterly` for quarterly company facts. The shorter `period=quarter` and `period=q` aliases are also accepted for quarterly data.

## Query parameters

- `ticker` or `cik` identifies the issuer. One is required.
- `tag` is the XBRL concept for concept-specific lookups, such as `Revenues`, `Revenue`, `NetIncomeLoss`, or `PropertyPlantAndEquipmentNet`. If omitted, the API returns common financial metrics.
- `taxonomy`, `unit`, and `form` pin the lookup when you do not want the default us-gaap plus foreign-private-issuer fallback.
- `fy`, `year`, `fy_from`, `fyFrom`, `fy_to`, and `fyTo` bound fiscal years.
- `include=geographic_segments` adds bounded revenue geography when available. Use `segment_limit` and `submission_file_limit` to control the enrichment budget.
- `view=agent` returns token-efficient fact rows with filing pointers lifted onto each item.

## Give this prompt to your agent

<Prompt>
Use SEC API GET /v1/facts with `ticker` or `cik`, an XBRL `tag` for concept-specific lookups, and `period=annual|quarterly|quarter|q` to return normalized SEC company facts. Use `view=agent` for compact fact rows with `accessionNumber` and `filingUrl`; use `include=geographic_segments` only when revenue geography is needed. Preserve `requestId` and any provenance/freshness/materialization metadata present in default responses.
</Prompt>

## Errors and partial results

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- retry with either `ticker` or `cik`, and add a valid `tag` when you need one specific concept
- invalid `period` values return `400 invalid_query_parameter` with `details.acceptedValues`
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
