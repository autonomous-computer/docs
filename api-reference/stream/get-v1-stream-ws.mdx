# GET /v1/stream/ws

Upgrade to a WebSocket connection for live stream frames

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
# Create a WebSocket stream subscription first:
curl -X POST \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  -H "content-type: application/json" \
  -d '{"transport":"websocket","eventTypes":["monitor.match"]}' \
  "https://api.secapi.ai/v1/stream_subscriptions"

# Mint the short-lived ticket:
curl -X POST \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  -H "content-type: application/json" \
  -d '{}' \
  "https://api.secapi.ai/v1/stream/tickets"

npx wscat -c "wss://api.secapi.ai/v1/stream/ws?ticket=$SECAPI_STREAM_TICKET&stream_id=$SECAPI_STREAM_ID" \
  -H "secapi-version: 2026-03-19"
```
</RequestExample>

<Info>
Mint the short-lived `$SECAPI_STREAM_TICKET` first with `POST /v1/stream/tickets` so you do not expose a long-lived API key in a browser-visible WebSocket URL.
</Info>
