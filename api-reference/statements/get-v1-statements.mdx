# GET /v1/statements

Return a normalized statement backed by SEC company facts. Resolves us-gaap (10-K/10-Q) filers and IFRS foreign private issuers (ifrs-full on 20-F/6-K) automatically.

<Info>
Audience: application and coding agent.
</Info>

## Agent mode

Pass `?view=agent` to receive the compact agent shape: rows + periods with tag/value/period metadata. Provenance, completeness, freshness, materialization, and validation envelopes are dropped. See [api conventions → response formats](/api-conventions) for the full `?view=` contract.

## Canonical metadata

- `requestId`
- `traceparent`
- `provenance`
- `freshness`
- `materialization`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/statements?statement=income_statement&symbol=AAPL&period=quarter"
```
</RequestExample>

## Query parameters

- `ticker`, `symbol`, or `cik` identifies the issuer. `symbol` is accepted as an alias for `ticker`.
- `statement` selects the statement key, such as `income_statement`, `balance_sheet`, or `cash_flow_statement`.
- `period=annual` returns annual statements. `period=quarterly`, `period=quarter`, and `period=q` return quarterly statements.
- `fy`, `year`, `fy_from`, `fyFrom`, `fy_to`, and `fyTo` bound fiscal years.
- `limit` caps periods returned, up to `40`.
- `view=agent` returns compact rows and periods for agent workflows; `view=compact` returns period-indexed row arrays.

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
