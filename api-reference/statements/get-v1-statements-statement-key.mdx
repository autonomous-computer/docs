# GET /v1/statements/{statement_key}

Return a specific normalized statement bundle keyed by statement type

<Info>
Use this endpoint from server-side applications, SDKs, or agent workflows.
</Info>

## Response metadata

- `requestId`
- `traceparent`
- `provenance`
- `freshness`
- `materialization`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/statements/income_statement?symbol=AAPL&period=quarter"
```
</RequestExample>

## Query parameters

- `statement_key` is the statement key in the path, such as `income_statement`, `balance_sheet`, or `cash_flow_statement`.
- `ticker`, `symbol`, or `cik` identifies the issuer. `symbol` is accepted as an alias for `ticker`.
- `fy`, `year`, `fy_from`, `fyFrom`, `fy_to`, and `fyTo` bound fiscal years.
- `limit` caps periods returned, up to `40`.
- `view=agent` returns compact rows and periods for agent workflows; `view=compact` returns period-indexed row arrays.

## Period aliases

Use `period=annual` for annual statements or `period=quarterly` for quarterly statements. The shorter `period=quarter` and `period=q` aliases are also accepted for quarterly data.

## Errors and partial results

- Treat non-2xx responses as API errors and log the response body for debugging
- Log `requestId` and `traceparent` so SEC API support can trace the request
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
