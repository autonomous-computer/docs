# GET /v1/entities/resolve

Resolve an entity by ticker, symbol, CIK, FIGI-family identifier, ISIN, CUSIP, name, or query with confidence and match-basis metadata

<Info>
Audience: application and coding agent.
</Info>

## Agent mode

Pass `?view=agent` to receive the compact agent shape: entity identifiers (id, ticker, cik, name) + primary identifiers + match metadata. Aliases, jurisdiction, full identifiers list, and metadata envelopes are dropped. See [api conventions → response formats](/api-conventions) for the full `?view=` contract.

## Canonical metadata

- `requestId`
- `traceparent`
- `provenance`
- `freshness`
- `materialization`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/entities/resolve?ticker=AAPL&view=agent"
```
</RequestExample>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
