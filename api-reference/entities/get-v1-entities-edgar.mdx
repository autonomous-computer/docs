Audience: engineers and agents that need SEC EDGAR-oriented entity discovery.

curl -X GET -H "x-api-key: $SECAPI_API_KEY" -H "secapi-version: 2026-03-19" "https://api.secapi.ai/v1/entities/edgar?q=example&limit=25"

</RequestExample>

## Pagination and source scope

This is an EDGAR-oriented discovery surface, not a promise that every result has the canonical enrichment available from [entity resolution](/api-reference/entities/get-v1-entities-resolve). Continue with `nextCursor` while `hasMore` is true, then resolve the selected identifier before using it as an issuer key.

## Errors

Invalid pagination returns `400 invalid_query_parameter`; source-search failures return a contract-aware error. Preserve `requestId` when reporting a source-coverage gap.
