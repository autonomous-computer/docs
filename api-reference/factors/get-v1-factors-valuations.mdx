# GET /v1/factors/valuations

Return valuation-factor opportunity signals ranked from SEC API-owned factor return state with trust metadata

<Info>
Use this endpoint from server-side applications, SDKs, or agent workflows.
</Info>

## Response metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/factors/valuations?category=style&signal=tailwind&limit=10&response_mode=compact"
```
</RequestExample>

## Example response

<ResponseExample>
```json
{
  "requestId": "req_example_123",
  "traceparent": "00-4bf92f3577b34da6a3ce929d0e0e4736-00f067aa0ba902b7-01"
}
```
</ResponseExample>

## Give this prompt to your agent

<Prompt description="Copy this SEC API prompt for your agent.">
Use SEC API GET /v1/factors/valuations to rank factor-level valuation opportunities. Preserve signal, side, score, lookback, latest return context, freshness, and methodology metadata. Treat this as factor opportunity discovery, not a direct stock recommendation.
</Prompt>

## Errors and partial results

- Treat non-2xx responses as API errors and log the response body for debugging
- Log `requestId` and `traceparent` so SEC API support can trace the request
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
