# GET /v1/factors/valuations

Return valuation-factor opportunity signals ranked from SecAPI-owned factor return state with trust metadata

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/factors/valuations?category=style&signal=tailwind&sort=opportunity_score&weighting_mode=long_short_equal&limit=10&response_mode=compact&include=trust"
```
</RequestExample>

## Give this prompt to your agent

<Prompt>
Use SEC API GET /v1/factors/valuations for factor-level opportunity discovery. Preserve `rank`, `factorKey`, `factorName`, `category`, `valuationMetric`, `signalDirection`, `weightingMode`, `opportunityScore`, `zScore`, `scaledReturn`, `stockDrilldown`, `opportunitySummary`, `freshness`, `materialization`, `methodology`, `requestId`, and `traceparent`.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
