# GET /v1/factors/dashboard

Return a one-call factor dashboard with intraday, regime, rotation, spotlight-security, and optional model-portfolio drill-down sections

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/factors/dashboard"
```
</RequestExample>

## Give this prompt to your agent

<Prompt>
Use SEC API GET /v1/factors/dashboard for a one-call factor dashboard with intraday, regime, rotation, spotlight-security, and model-portfolio sections. Preserve dashboard section keys, ranked factor rows, `asOf`, freshness, degraded-state notes, methodology metadata, `requestId`, and `traceparent` so downstream UIs and agents can explain what is current.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
