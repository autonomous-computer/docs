Discover a custom thematic factor pack from factor-overlap peers plus deterministic filing/news signatures

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X POST \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  -H "content-type: application/json" \
  -d '{"symbol":"NVDA","candidates":["AMD","AVGO","TSM","ASML"],"lookback":"6m","limit":5}' \
  "https://api.secapi.ai/v1/factors/custom"
```
</RequestExample>

## Give this prompt to your agent

<Prompt>
Use SEC API POST /v1/factors/custom to discover a thematic factor pack from factor-overlap peers plus filing/news signatures. Preserve `symbol`, `anchorTheme`, `themeLabels`, `factorKeySuggestion`, `semanticSignature`, `namingWorkflow`, `peers`, `evidence`, `summaryMd`, `freshness`, `methodology`, `requestId`, and `traceparent`.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
