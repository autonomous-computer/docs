# GET /v1/factors/exposures

Return compact factor exposures for securities, portfolios, or watchlists, with diagnostics and provenance available on request.

<Info>
Use this endpoint from server-side applications, SDKs, or agent workflows. Use `symbol=AAPL` for the common single-security lookup, or `symbols=AAPL,MSFT` for batches.
</Info>

## Response metadata

- `requestId`
- `traceparent`

## Response controls

- `response_mode=compact` returns the factor beta, percentile, confidence, model, and as-of timestamp.
- `include=diagnostics` adds regression diagnostics such as `rSquared`, `tStat`, `observationCount`, and `nActiveFactors`.
- `include=trust` adds provenance, freshness, materialization, source-rights, methodology, revision, and degraded-state metadata.
- `response_mode=standard` returns the full exposure row shape.

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/factors/exposures?symbol=AAPL&response_mode=compact"
```
</RequestExample>

## Expanding diagnostics

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/factors/exposures?symbol=AAPL&response_mode=compact&include=diagnostics"
```
</RequestExample>

Use `include=diagnostics` when a model, risk report, or quant workflow needs regression fit details. Use `include=trust` when the answer will cite provenance, freshness, methodology, or source-rights evidence.

## Give this prompt to your agent

<Prompt description="Copy this SEC API prompt for your agent.">
Use SEC API GET /v1/factors/exposures with `symbol=AAPL` or the ticker the user provides. Preserve `requestId` and `traceparent`. Summarize the largest positive and negative factor betas, percentile, confidence, and model name. If the workflow needs regression fit statistics, call again with `include=diagnostics`; if it needs proof or methodology evidence, call again with `include=trust`.
</Prompt>

## Errors and partial results

- Treat non-2xx responses as API errors and log the response body for debugging
- Log `requestId` and `traceparent` so SEC API support can trace the request
- Use `include=diagnostics` before citing regression fit statistics such as `rSquared` or `tStat`
- Use `include=trust` before citing proof, provenance, freshness, or source-rights metadata
