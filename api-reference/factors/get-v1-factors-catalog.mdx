# GET /v1/factors/catalog

Return compact SEC API factor definitions, with trust and methodology metadata available on request.

<Info>
Use this endpoint from server-side applications, SDKs, or agent workflows. The default response is compact so catalog lookups are cheap to put in a prompt, table, or picker.
</Info>

## Response metadata

- `requestId`
- `traceparent`

## Response controls

- `response_mode=compact` keeps each factor definition focused on identity, category, launch status, benchmark, equation, and expansion hints.
- `include=trust` expands compact rows with `launchReadiness`, `qualityProof`, provenance, freshness, source-rights, methodology, materialization, revision, and degraded-state metadata.
- `response_mode=standard` returns the full factor definition shape without requiring an `include` value.

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/factors/catalog?response_mode=compact"
```
</RequestExample>

## Expanding trust metadata

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/factors/catalog?response_mode=compact&include=trust"
```
</RequestExample>

Use `include=trust` when an answer will cite freshness, coverage, methodology, source rights, or launch-readiness proof. Compact rows keep the trust envelope out of the default payload, but the response still points callers to the expansion.

## Give this prompt to your agent

<Prompt description="Copy this SEC API prompt for your agent.">
Use SEC API GET /v1/factors/catalog to return the compact factor catalog. Preserve `requestId` and `traceparent`. Summarize the available factor keys, categories, benchmark symbols, and launch statuses. If the workflow needs proof, methodology, provenance, freshness, or source-rights evidence, call the endpoint again with `include=trust`.
</Prompt>

## Errors and partial results

- Treat non-2xx responses as API errors and log the response body for debugging
- Log `requestId` and `traceparent` so SEC API support can trace the request
- Use `include=trust` before citing proof, provenance, freshness, or source-rights metadata
