# GET /v1/factors/sparklines

Return compact multi-factor sparkline series with 1D through MAX summary windows for dashboard, sector, index, and model UIs

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/factors/sparklines?keys=VALUE,MOMENTUM,QUALITY&range=1y&points=64&response_mode=compact&include=trust"
```
</RequestExample>

## Give this prompt to your agent

<Prompt>
Use SEC API GET /v1/factors/sparklines for compact multi-factor chart rows in sector, index, dashboard, and model UIs. Preserve each row's `factorKey`, `factorName`, `category`, `range`, `metric`, `latestValue`, `summaryWindows`, `points`, `historyStartDate`, `historyEndDate`, `qualityProof`, `requestId`, and `traceparent`. Use `points` to bound token cost.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
