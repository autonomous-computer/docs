# GET /v1/factors/sparklines

Return compact multi-factor sparkline series with 1D through MAX summary windows for dashboard, sector, index, and model UIs

<Info>
Use this endpoint from server-side applications, SDKs, or agent workflows.
</Info>

## Response metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/factors/sparklines?keys=VALUE,MOMENTUM,QUALITY&range=1y&points=64&response_mode=compact"
```
</RequestExample>

## Example response

<ResponseExample>

```json
{

```

## Give this prompt to your agent

<Prompt description="Copy this SEC API prompt for your agent.">
Use SEC API GET /v1/factors/sparklines to fetch compact sparkline data for multiple factors. Keep summary windows, latest return, row-level coverage, and freshness metadata intact. Use `keys` or `factors` for explicit factors and `response_mode=compact` for dashboard or LLM context handoff.
</Prompt>

## Errors and partial results

- Treat non-2xx responses as API errors and log the response body for debugging
- Log `requestId` and `traceparent` so SEC API support can trace the request
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
