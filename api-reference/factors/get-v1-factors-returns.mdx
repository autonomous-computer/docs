# GET /v1/factors/returns

Return factor return history, z-scores, and volatility-scaled series for supported factor families

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/factors/returns"
```
</RequestExample>

## Give this prompt to your agent

<Prompt>
Use SEC API GET /v1/factors/returns for multi-factor return windows, z-scores, and volatility-scaled series. Preserve each row's `factorKey`, `factorName`, `category`, `window`, `rawReturn`, `pureReturn`, `scaledReturn`, `zScore`, `qualityProof`, `freshness`, `materialization`, `methodology`, `requestId`, and `traceparent`.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
