# GET /v1/factors/valuations/stocks

Return stock candidates exposed to a valuation-factor signal using latest materialized factor exposures

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/factors/valuations/stocks?factor=VALUE&stance=beneficiaries&weighting_mode=long_short_equal&limit=25&response_mode=compact&include=trust"
```
</RequestExample>

## Give this prompt to your agent

<Prompt>
Use SEC API GET /v1/factors/valuations/stocks to drill from a valuation-factor signal into exposed securities. Preserve `rank`, `symbol`, `factorKey`, `factorName`, `stance`, `impact`, `score`, `expectedFactorImpactScore`, `exposureBeta`, `exposurePercentile`, `rankingRationale`, `opportunitySummary`, `freshness`, `materialization`, `methodology`, `requestId`, and `traceparent`.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
