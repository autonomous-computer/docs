Compile a natural-language alert request into a structured monitor spec

<Info>
Audience: application and coding agent.
</Info>

## Coverage and source scope

Turns a plain-English trigger (e.g. "notify me about mergers at AAPL") into a validated structured monitor subscription spec plus a ready-to-POST create body. Read-only: it returns a spec, it does not create the monitor.

## Draft before creation

The compiler returns a validated structured subscription specification and a ready-to-submit create body. It does not persist a monitor or reserve a delivery destination. The endpoint can be unavailable, and invalid prompts or unsupported filters return validation errors that should be shown for correction.

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X POST \
  -H "Authorization: Bearer $SECAPI_BEARER_TOKEN" \
  -H "secapi-version: 2026-03-19" \
  -H "content-type: application/json" \
  -d '{"prompt":"Notify me about mergers at AAPL"}' \
  "https://api.secapi.ai/v1/monitors/compile"
```
</RequestExample>

## Give this prompt to your agent

<Prompt>
Call SEC API POST /v1/monitors/compile to validate a natural-language request and obtain a structured draft. It is read-only: review the returned create body and create a monitor separately after a human approves it.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
