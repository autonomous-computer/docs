Retrieve the public free-tier plan descriptor (included features, alert limits, and upgrade paths)

<Info>
Audience: application and coding agent.
</Info>

## Availability

This public descriptor exists only when the alerts surface is enabled. A `404` means it is not currently exposed; it does not mean that the plan has zero limits or that a registration request succeeded.

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/alerts/plan"
```
</RequestExample>

## Example response

<ResponseExample>
```json
{
  "requestId": "req_2ZK8Q1W9F4M6P7R3"
}
```
</ResponseExample>

## Give this prompt to your agent

<Prompt>
Call SEC API GET /v1/alerts/plan only to read the currently exposed public alert-plan descriptor. Treat a 404 as unavailable rather than an empty plan, and preserve the returned limits and requestId exactly.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
