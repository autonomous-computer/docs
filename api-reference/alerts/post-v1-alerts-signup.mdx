Register an email for the free alerts tier (idempotent lead capture)

<Info>
Audience: application and coding agent.
</Info>

## Registration behavior

Submit one valid email address. Repeating the same registration is safe, but this endpoint records registration only; it does not create a saved monitor, authenticate an organization, or prove that any alert will be delivered.

The alerts surface can be disabled. Treat `404` as unavailable and show the returned result rather than promising a fallback subscription.

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X POST \
  -H "content-type: application/json" \
  -H "secapi-version: 2026-03-19" \
  -d '{"email":"alerts@example.com"}' \
  "https://api.secapi.ai/v1/alerts/signup"
```
</RequestExample>

</ResponseExample>

## Give this prompt to your agent

<Prompt>
Call SEC API POST /v1/alerts/signup with one valid email address to register interest in the exposed public alerts tier. The registration is idempotent, but a 404 means the public alerts surface is unavailable; do not retry it as a delivery failure.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
