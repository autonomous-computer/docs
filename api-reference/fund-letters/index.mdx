Reference pages for Fund Letters endpoints.

<Note>
The sidebar labels inside this family are the literal calls you make over HTTP. Descriptions stay in the body so the navigation stays sharp and scannable.
</Note>

## Operations in this family

<CardGroup cols={2}>
<Card title="GET /v1/fund-letters" href="/api-reference/fund-letters/get-v1-fund-letters">List investor letters from hedge funds, partnerships, and registered funds, filterable by manager, fund, company, period, source, and distribution tier.</Card>
<Card title="GET /v1/fund-letters/{letter_id}" href="/api-reference/fund-letters/get-v1-fund-letters-letter-id">Retrieve one letter with letter-level narratives, performance figures, and inline theses (capped at 25; thesisCount carries the total). Superseded/merged IDs resolve forever via aliases.</Card>
<Card title="GET /v1/fund-letters/{letter_id}/document" href="/api-reference/fund-letters/get-v1-fund-letters-letter-id-document">Retrieve the original letter document (format=pdf, 302 redirect) or page-segmented markdown whose bytes verify every anchor (format=markdown).</Card>
<Card title="GET /v1/fund-letters/changes" href="/api-reference/fund-letters/get-v1-fund-letters-changes">Keyset-paginated delta feed of fund-letter events (letter.published, letter.updated, letter.superseded, thesis.extracted, manager.added). Never response-cached.</Card>
<Card title="GET /v1/fund-letters/companies" href="/api-reference/fund-letters/get-v1-fund-letters-companies">Company coverage index: every company with at least one extracted thesis, with thesis/manager/letter counts and the latest stance.</Card>
<Card title="GET /v1/fund-letters/funds" href="/api-reference/fund-letters/get-v1-fund-letters-funds">List funds that publish letters, with per-fund letter counts.</Card>
<Card title="GET /v1/fund-letters/funds/{fund_id}" href="/api-reference/fund-letters/get-v1-fund-letters-funds-fund-id">Retrieve one fund. Merged fund IDs resolve forever via aliases; the response carries the canonical id plus requestedId when they differ.</Card>
<Card title="GET /v1/fund-letters/managers" href="/api-reference/fund-letters/get-v1-fund-letters-managers">List letter-publishing managers (firms) with coverage stats: letters, funds, companies, theses, period range, and cadence.</Card>
<Card title="GET /v1/fund-letters/managers/{manager_id}" href="/api-reference/fund-letters/get-v1-fund-letters-managers-manager-id">Retrieve one manager profile. Merged manager IDs resolve forever via aliases; the response carries the canonical id plus requestedId when they differ.</Card>
<Card title="GET /v1/fund-letters/managers/{manager_id}/overview" href="/api-reference/fund-letters/get-v1-fund-letters-managers-manager-id-overview">Fund Overview: one token-efficient briefing per manager — canonical name, description, founders, website, coverage counts, and the latest letter's highlights with up to 5 headline theses.</Card>
<Card title="GET /v1/fund-letters/search" href="/api-reference/fund-letters/get-v1-fund-letters-search">Full-text search over letter bodies; hits carry page-anchored highlights.</Card>
<Card title="GET /v1/fund-letters/semantic" href="/api-reference/fund-letters/get-v1-fund-letters-semantic">Semantic (vector) search over letter content; hits carry similarity scores, matched chunks, and page anchors.</Card>
<Card title="GET /v1/fund-letters/theses" href="/api-reference/fund-letters/get-v1-fund-letters-theses">Cross-cutting thesis screen: structured per-company extractions (relationship, stance, conviction, narratives, verbatim anchored quotes) across all letters.</Card>
</CardGroup>
