Find exact terms across SEC filing content and extracted section text, then narrow the combined result set by issuer, form, filing identity, fiscal year, or filing date.

<Info>
Audience: application and coding agent.
</Info>

## Scope and filters

`q` is required. Use exactly one of `ticker`, `symbol`, or `cik` to narrow to an issuer; `symbol` is an alias for `ticker`, and supplying both requires matching values. Add `form`, `accession_number`, `filing_year` or `fy`, and `date_from` / `date_to` when the workflow needs a filing-level boundary.

`limit` is the maximum combined search budget before the response splits work across the filing and section legs. Use a smaller limit for a review queue and retain the returned filing and section context when presenting matches.

## Example request

<RequestExample>
```bash
curl -X GET -H "x-api-key: $SECAPI_API_KEY" -H "secapi-version: 2026-03-19" "https://api.secapi.ai/v1/search/fulltext?q=liquidity&ticker=EXAMPLE&form=10-K&limit=20"
```
</RequestExample>

## About this example

This response uses a synthetic issuer, accession, and `example.com` URL to illustrate the two result legs. It does not identify or quote a live SEC filing.

## Give this prompt to your agent

<Prompt>
Use SEC API GET /v1/search/fulltext to find exact terms across filings and extracted sections. Start with `q`, then narrow by one issuer identifier and, when useful, `form`, filing year, accession number, or filing-date range. Preserve each result's filing identifier, section context, source URL, `requestId`, and `traceparent`; treat `limit` as a combined budget across the filing and section search legs.
</Prompt>

## Failure posture

- send `ticker` or its `symbol` alias, not conflicting values for both; matching values are required when both are present
- treat an invalid query or filter response as a request-shape problem and retain the returned validation details
- preserve filing and section context with every match; a keyword hit is not a conclusion about the issuer
- preserve `requestId` and `traceparent` when retrying a transient failure
