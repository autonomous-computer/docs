# GET /v1/search/fulltext

Full-text search across filing content and section text

<Info>
Use this endpoint from server-side applications, SDKs, or agent workflows.
</Info>

## Response metadata

- `requestId`
- `traceparent`

Use `ticker`, `symbol`, or `cik` to scope the search to an issuer. `symbol` is an alias for `ticker`; if both are provided, they must match.

Use `accession_number` to scope the filing result leg to an exact filing, `filing_year` for calendar-year filing scope, `fy` or `year` for issuer fiscal-year scope, and `date_from` / `date_to` for explicit filing-date bounds.

## Example request

<RequestExample>
```bash
curl -X GET \
 -H "x-api-key: $SECAPI_API_KEY" \
 -H "secapi-version: 2026-03-19" \
 "https://api.secapi.ai/v1/search/fulltext?q=going+concern&symbol=AAPL&form=10-K&filing_year=2025&date_from=2025-01-01&date_to=2025-12-31"
```
</RequestExample>

## Errors and partial results

- Treat non-2xx responses as API errors and log the response body for debugging
- if `ticker` and `symbol` are both provided, they must match
- `filing_year`, `fy`, `year`, `date_from`, and `date_to` scope both filing and section search legs
- Log `requestId` and `traceparent` so SEC API support can trace the request
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
