# GET /v1/search/semantic

Semantic vector search across SEC filing section content via Voyage AI + Pinecone, with hybrid keyword + vector RRF and citation fields on every result row

<Info>
Audience: application and coding agent.
</Info>

## Citation fields

Every result row carries seven additive top-level fields:

| Field | Description |
|---|---|
| `accession` | SEC accession number for the source filing |
| `section_key` | Canonical section identifier (e.g. `item_1a`) |
| `char_start` / `char_end` | Half-open offsets into the section markdown text (`section_snippets.content_md`) — section-text-relative, stable across reparses |
| `highlighted_snippet` | ±150 char window with query terms wrapped in `**…**`, sentence-boundary truncated, ≤320 chars |
| `source_url` | Public SEC.gov filing URL |
| `ticker` | Issuer ticker, when known |

When the validator can't produce a span (no section text, no query-term match, or offsets out of bounds), `char_start`/`char_end`/`highlighted_snippet` drop to `null` and `_citation_degraded` records the reason. Identity fields (`accession`, `section_key`, `source_url`, `ticker`) are always populated when known. See [api conventions → citation + char-range spans](/api-conventions#citation-char-range-spans) for the full contract.

## Agent mode

Pass `?view=agent` to drop `score` and `retrievalMode` from each row. The seven citation fields are preserved unconditionally. See [api conventions → response formats](/api-conventions) for the full `?view=` contract.

Valid `view` values are `default`, `compact`, and `agent`. `view=compact` is accepted for response-format consistency and currently returns the default search row shape; use `view=agent` when you need token-efficient rows. Invalid view values return `400 invalid_query_parameter` with `details.acceptedValues`.

Use `ticker`, `symbol`, or `cik` to scope the search to an issuer. `symbol` is an alias for `ticker`; if both are provided, they must match.

## Retrieval mode

Use `mode=hybrid` for the default keyword + vector blend, `mode=semantic` for vector retrieval, or `mode=keyword` for exact-text fallback. Invalid mode values return `400 invalid_query_parameter` with `acceptedValues`.

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/search/semantic?q=supply+chain+risk&symbol=AAPL&form=10-K&mode=hybrid&view=agent"
```
</RequestExample>

## Give this prompt to your agent

<Prompt>
Use SEC API GET /v1/search/semantic with a required `q` query and `mode=hybrid`, `mode=semantic`, or `mode=keyword` to search SEC filing sections with citation fields on every result row. Preserve `requestId`, `traceparent` in the output. Return a concise markdown summary plus the exact structured payload fields that a downstream engineer or agent should keep using this result.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- if `ticker` and `symbol` are both provided, they must match
- invalid `mode` values return `invalid_query_parameter` with `acceptedValues`
- invalid `view` values return `invalid_query_parameter` with `details.acceptedValues`
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
