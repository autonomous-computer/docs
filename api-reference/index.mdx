## Start Here

<Steps>
<Step title="Pick the domain you care about">Start from the family page that matches your workflow: entities, filings, statements, ownership, market data, webhooks, or platform APIs.</Step>
<Step title="Open the exact operation page">Each operation page is titled with the literal REST call, such as `GET /v1/entities/resolve`.</Step>
<Step title="Hand it to your agent">Every operation page includes a copyable prompt, example request, example response, metadata expectations, and error handling.</Step>
</Steps>

## Choose by job

| If you need to... | Start with |
|---|---|
| Resolve a ticker, CIK, fund, adviser, or insider | [Entities](/api-reference/entities) |
| Find, render, download, export, or section a filing | [Filings](/api-reference/filings) |
| Read normalized statements or XBRL facts | [Statements](/api-reference/statements) and [Facts](/api-reference/facts) |
| Search filing text by keyword or meaning | [Search](/api-reference/search) and [Sections](/api-reference/sections) |
| Monitor filings, streams, events, and webhooks | [Monitors](/api-reference/monitors), [Streams](/api-reference/streams), and [Webhooks](/api-reference/webhooks) |
| Work with usage, billing, diagnostics, or agent bootstrap | [Platform](/api-reference/platform), [Diagnostics](/api-reference/diagnostics), and [Agent Bootstrap](/api-reference/agent) |

## Canonical wire contract

- OpenAPI JSON: [`/openapi/sec-api.v1.json`](/openapi/sec-api.v1.json)
- Base URL: `https://api.secapi.ai`
- Version header: `secapi-version` set to the current published version
