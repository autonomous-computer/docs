Exchange a sponsor token for the first org-scoped API key, billing snapshot, limits, and MCP install metadata

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `organization`
- `apiKey`
- `billing`
- `limits`
- `distribution`
- `sponsorToken`

## Example request

<RequestExample>
```bash
curl -X POST \
  -H "secapi-version: 2026-03-19" \
  -H "content-type: application/json" \
  -d '{"token":"agtok_example_sponsor_token","label":"research-agent","scopes":["read:sec"]}' \
  "https://api.secapi.ai/v1/agent/bootstrap"
```
</RequestExample>

## Authentication

This exchange is intentionally pre-auth. Do not send x-api-key or Authorization; submit the short-lived, single-use sponsor token in the JSON body. A human organization member creates that sponsor token through [POST /v1/agent/bootstrap_tokens](/api-reference/agent/post-v1-agent-bootstrap-tokens). The issued API-key secret is returned once and must not be logged.

## Failure posture

- bootstrap_token_invalid returns 404; obtain a valid sponsor token
- bootstrap_token_expired returns 410; request a new sponsor token
- bootstrap_token_already_used returns 409; sponsor tokens are single-use
- bootstrap_token_scope_invalid returns 400; self-serve exchange scopes are limited to read:sec
- api_key_limit_reached returns 402; resolve the organization's active API-key limit before retrying
