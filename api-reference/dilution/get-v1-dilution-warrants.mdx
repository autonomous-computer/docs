# GET /v1/dilution/warrants

List warrant schedules parsed from offering exhibits, including price-protection and ratchet metadata, with the full verification block on every row.

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`
- `verification`
- `provenance`

## Query parameters

| Param | Type | Description |
|---|---|---|
| `ticker` | string | Issuer ticker. |
| `cik` | string | SEC CIK. |
| `accession_number` | string | Match a specific filing's warrants. |
| `form_type` | string | Filing form (e.g. `424B5`, `S-1`). |
| `filed_at_from` / `filed_at_to` | ISO date | Bounds on `filed_at`. |
| `cursor` | int | From prior page's `nextCursor`. |
| `limit` | int (1–50, default 10) | Page size. |
| `view` | enum | Response shape (default \| compact \| agent). |

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $OMNI_DATASTREAM_API_KEY" \
  -H "omni-version: 2026-03-19" \
  "https://api.secapi.ai/v1/dilution/warrants?ticker=ACME&limit=5"
```
</RequestExample>

## Give this prompt to your agent

<Prompt>
Use Omni Datastream `GET /v1/dilution/warrants` to list warrant schedules and assess overhang. Preserve `requestId`, `traceparent`, and the per-row `verification` block. Flag warrants whose `priceProtection` is true (price-reset / ratchet exposure) and report total potential dilutive shares from `underlyingSharesCalculated` when present.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- when `verification.confidence < 0.7`, treat the row as advisory only
