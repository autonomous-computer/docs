# GET /v1/dilution/score

Per-issuer Dilution Score — singleton view returning the latest dilution rating row for a ticker. Requires `?ticker`.

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`
- `verification`
- `provenance`

## Query parameters

| Param | Type | Description |
|---|---|---|
| `ticker` | string (required) | Issuer ticker. Returns the latest rating row. |
| `view` | enum | Response shape (default \| compact \| agent). |

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $OMNI_DATASTREAM_API_KEY" \
  -H "omni-version: 2026-03-19" \
  "https://api.secapi.ai/v1/dilution/score?ticker=ACME"
```
</RequestExample>

## Errors

- `400 missing_ticker` — `?ticker` is required.
- `404 dilution_score_not_found` — no rating row for the ticker yet.

## Give this prompt to your agent

<Prompt>
Use Omni Datastream `GET /v1/dilution/score?ticker={TICKER}` to fetch the latest composite Dilution Score for a single issuer. Preserve `requestId`, `traceparent`, and `verification`. Surface `numericScore`, `overallRisk`, and the sub-factor map so the human reviewer sees the breakdown.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent`
- on `404 dilution_score_not_found`, fall back to `GET /v1/dilution/ratings?ticker=…` for the historical timeseries
