# GET /v1/dilution/events/{event_id}

Retrieve a single dilution event by id, including the full verification block, offering economics, and provenance pointer.

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`
- `verification`
- `provenance`

## Path parameters

| Param | Type | Description |
|---|---|---|
| `event_id` | string | Primary key of the dilution event row (e.g. `dil_evt_acme_424b5_2026q1`). |

## Query parameters

| Param | Type | Description |
|---|---|---|
| `view` | `default` \| `compact` \| `agent` | Response shape — agent compresses verification via `summarizeVerification`. |

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $OMNI_DATASTREAM_API_KEY" \
  -H "omni-version: 2026-03-19" \
  "https://api.secapi.ai/v1/dilution/events/dil_evt_acme_424b5_2026q1"
```
</RequestExample>

## Errors

- `404 dilution_event_not_found` — no row matches `event_id`.

## Give this prompt to your agent

<Prompt>
Use Omni Datastream `GET /v1/dilution/events/{event_id}` to fetch the full record for one dilution event. Always preserve `requestId`, `traceparent`, and the `verification` block. If `verification.confidence < 0.7` or any `crossValidations` entry has `passed: false`, surface that — and the `sourceSpan` — instead of treating the row as a confirmed observation.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- on `404 dilution_event_not_found`, do not retry; the id is wrong or the row is not yet extracted
