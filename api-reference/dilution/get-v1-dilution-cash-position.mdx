# GET /v1/dilution/cash-position

Cash runway, burn, and management commentary computed from quarterly filings. Ordered by `period_ended desc`. Verification block on every row.

<Info>
Use this endpoint from server-side applications, SDKs, or agent workflows.
</Info>

## Response metadata

- `requestId`
- `traceparent`
- `verification`
- `provenance`

## Query parameters

| Param | Type | Description |
|---|---|---|
| `ticker` | string | Issuer ticker. |
| `period_ended_from` / `period_ended_to` | ISO date | Bounds on `period_ended`. |
| `cursor` | int | From prior page's `nextCursor`. |
| `limit` | int (1–50, default 10) | Page size. |
| `view` | enum | Response shape. |

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SEC_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/dilution/cash-position?ticker=ACME&limit=4"
```
</RequestExample>

## Give this prompt to your agent

<Prompt description="Copy this SEC API prompt for your agent.">
Use SEC API `GET /v1/dilution/cash-position` to assess runway. Preserve `requestId`, `traceparent`, and the `verification` block. Flag tickers with `cashRemainingMonths < 12` as near-term dilution candidates and API `mgmtCommentary` verbatim when present.
</Prompt>

## Errors and partial results

- Treat non-2xx responses as API errors and log the response body for debugging
- preserve `requestId` and `traceparent`
