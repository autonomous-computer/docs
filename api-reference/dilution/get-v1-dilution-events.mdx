# GET /v1/dilution/events

List dilution events parsed from S-1, 424B*, FWP, and S-3 offering filings, with the full programmatic verification block on every row.

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`
- `verification` (programmatic per-row block ‚Äî see API conventions)
- `provenance`

## Query parameters

| Param | Type | Description |
|---|---|---|
| `ticker` | string | Issuer ticker (case-insensitive). |
| `cik` | string | SEC CIK as digits. |
| `accession_number` | string | Match a specific filing's events. |
| `form_type` | string | E.g. `S-1`, `424B5`, `FWP`, `S-3`. |
| `offering_type` | string | Free-text classification (e.g. `atm`, `pipe`). |
| `is_atm` | boolean | Filter to at-the-market offerings. |
| `filed_at_from` | ISO date/timestamp | Inclusive lower bound on `filed_at`. |
| `filed_at_to` | ISO date/timestamp | Inclusive upper bound on `filed_at`. |
| `cursor` | int (offset) | Cursor from prior page's `nextCursor`. |
| `limit` | int (1‚Äì50, default 10) | Page size. |
| `view` | `default` \| `compact` \| `agent` | Response shape (default leaves verification block intact; agent compresses via `summarizeVerification`). |

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $OMNI_DATASTREAM_API_KEY" \
  -H "omni-version: 2026-03-19" \
  "https://api.secapi.ai/v1/dilution/events?ticker=ACME&form_type=424B5&limit=2"
```
</RequestExample>

## Give this prompt to your agent

<Prompt>
Use Omni Datastream `GET /v1/dilution/events` to list dilution events for the issuer of interest, then summarize ATM activity, offering pace, and warrant overhang. Always preserve `requestId`, `traceparent`, and the per-row `verification` block (or its agent-mode summary). When `confidence < 0.7` or `crossValidationsPassed` reports failures, treat the row as unverified and surface the `sourceSpan` for human review.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if the verification block reports `confidence < 0.5` or any `crossValidations` entry fails, surface that signal ‚Äî do not paper over it
