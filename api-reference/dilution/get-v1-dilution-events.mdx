Find filed offering and issuance events by issuer, filing, date, or ATM status

<Info>
Audience: investors, research systems, and developers tracing filed offering activity.
</Info>

## Canonical metadata

- `data`
- `hasMore`
- `nextCursor`
- `per-row: id, ticker, accessionNumber, filedAt, verification.confidence`
- `requestId`
- `traceparent (standard response/header)`

## Example request

<RequestExample>
```bash
curl -X GET -H "x-api-key: $SECAPI_API_KEY" -H "secapi-version: 2026-03-19" "https://api.secapi.ai/v1/dilution/events?ticker=EXAMPLE&is_atm=true&limit=10"
```
</RequestExample>

## Evidence and limits

Each row is a filing-derived event, not a prediction of future issuance or price impact. Use the accession, filing URL, and verification fields to inspect the underlying disclosure before drawing an investment conclusion. `is_atm=true` narrows disclosed ATM events; it does not establish that an ATM was used after the cited filing. See [filing and source conventions](/api-conventions) and the [Dilution Score guide](/dilution-score).

## Failure posture

- preserve accession, filing URL, verification, and pagination metadata with each event
- treat an empty filtered page as no matching stored disclosure, not proof that no financing occurred
- distinguish filed offering terms from an inference about future share issuance
