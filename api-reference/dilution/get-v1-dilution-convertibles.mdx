# GET /v1/dilution/convertibles

List convertible debt and convertible-preferred schedules with conversion price, maturity, and ratchet metadata. Verification block on every row.

<Info>
Use this endpoint from server-side applications, SDKs, or agent workflows.
</Info>

## Response metadata

- `requestId`
- `traceparent`
- `verification`
- `provenance`

## Query parameters

| Param | Type | Description |
|---|---|---|
| `ticker` | string | Issuer ticker. |
| `cik` | string | SEC CIK. |
| `accession_number` | string | Match a specific filing. |
| `form_type` | string | Filing form. |
| `filed_at_from` / `filed_at_to` | ISO date | Bounds on `filed_at`. |
| `cursor` | int | From prior page's `nextCursor`. |
| `limit` | int (1–50, default 10) | Page size. |
| `view` | enum | Response shape. |

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/dilution/convertibles?ticker=ACME"
```
</RequestExample>

## Give this prompt to your agent

<Prompt description="Copy this SEC API prompt for your agent.">
Use SEC API `GET /v1/dilution/convertibles` to list convertible debt / preferred schedules. Preserve `requestId`, `traceparent`, and the `verification` block. Flag rows where `priceProtection` is true and `sharePrice < conversionPrice` — those are at risk of triggering a reset.
</Prompt>

## Errors and partial results

- Treat non-2xx responses as API errors and log the response body for debugging
- preserve `requestId` and `traceparent`
- treat `verification.confidence < 0.7` as advisory only
