# GET /v1/dilution/nasdaq-compliance

List Nasdaq deficiency notices with status and remediation tracking. Ordered by `date desc`. Verification block on every row.

<Info>
Use this endpoint from server-side applications, SDKs, or agent workflows.
</Info>

## Response metadata

- `requestId`
- `traceparent`
- `verification`
- `provenance`

## Query parameters

| Param | Type | Description |
|---|---|---|
| `ticker` | string | Issuer ticker. |
| <code>status</code> | string | Filter (`pending`, `in_compliance`, `delinquent`). |
| `date_from` / `date_to` | ISO date | Bounds on `date`. |
| `cursor` | int | From prior page's `nextCursor`. |
| `limit` | int (1–50, default 10) | Page size. |
| `view` | enum | Response shape. |

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/dilution/nasdaq-compliance?ticker=ACME"
```
</RequestExample>

## Give this prompt to your agent

<Prompt description="Copy this SEC API prompt for your agent.">
Use SEC API `GET /v1/dilution/nasdaq-compliance` to retrieve Nasdaq deficiency notices and remediation status. Preserve `requestId`, `traceparent`, and `verification`. Status transitions (e.g. `pending → in_compliance`) fire the `dilution.nasdaq_compliance.status_changed` webhook event.

## Errors and partial results

- Treat non-2xx responses as API errors and log the response body for debugging
- preserve `requestId` and `traceparent`
