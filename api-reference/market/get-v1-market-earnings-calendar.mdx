# GET /v1/market/earnings-calendar

Return upcoming and recent earnings events with consensus estimates, actuals, and surprise data

<Info>
Audience: application and coding agent.
</Info>

## Query parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `date_from` | string | **Required.** Start date (YYYY-MM-DD) |
| `date_to` | string | **Required.** End date (YYYY-MM-DD) |
| `symbol` | string | Filter to a single ticker symbol |
| `limit` | integer | Page size (default 100, max 1000) |

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $OMNI_DATASTREAM_API_KEY" \
  -H "omni-version: 2026-03-19" \
  "https://api.secapi.ai/v1/market/earnings-calendar?date_from=2026-03-24&date_to=2026-03-31&limit=5"
```
</RequestExample>

## Give this prompt to your agent

<Prompt>
Use Omni Datastream GET /v1/market/earnings-calendar to retrieve earnings events within a date range. Each event includes consensus estimate, actual result, surprise percentage, analyst count, and fiscal period. Filter by symbol for single-ticker views. Both EPS and revenue metrics are returned. Preserve `requestId` and `traceparent` in the output.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- `date_from` and `date_to` are required; omitting either returns a 400
