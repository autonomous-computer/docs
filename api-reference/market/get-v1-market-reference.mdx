# GET /v1/market/reference

Return canonical market reference metadata for a security, including identifiers, exchange, and listing context.

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`
- `provenance`
- `freshness`
- `materialization`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $OMNI_DATASTREAM_API_KEY" \
  -H "omni-version: 2026-03-19" \
  "https://api.secapi.ai/v1/market/reference?ticker=AAPL"
```
</RequestExample>

## Keep these fields

- canonical identifiers: `symbol`, `cik`, `compositeFigi`, `shareClassFigi`
- listing context: `exchange`, `securityType`, `currency`
- issuer context: `marketCap`, `listDate`, `homepageUrl`, `sicDescription`
- trust metadata: `provenance`, `freshness`, `materialization`, `sourceRights`

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
