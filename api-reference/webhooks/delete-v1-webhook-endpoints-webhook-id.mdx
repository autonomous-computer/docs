Delete a webhook endpoint while preserving delivery audit history

<Info>
Audience: application and admin.
</Info>

## Deletion and history

Deletion removes the destination from future fanout while preserving its delivery audit history. It does not revoke events already received by the remote system. A missing endpoint returns `404`.

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X DELETE \
  -H "Authorization: Bearer $SECAPI_BEARER_TOKEN" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/webhook_endpoints/wh_example_123"
```
</RequestExample>

<Info>
This organization control-plane action accepts bearer organization REST authentication. API keys cannot mutate webhook endpoints. The dashboard is optional.
</Info>

## Example response

<ResponseExample>
```json
{
  "object": "webhook_endpoint.deleted",
  "id": "wh_2ZK8Q1W9F4M6P7R3",
  "deleted": true,
  "requestId": "req_2ZK8Q1W9F4M6P7R3"
}
```
</ResponseExample>

## Give this prompt to your agent

<Prompt>
Delete the endpoint with bearer organization REST authentication while retaining delivery audit history. API keys cannot mutate webhook endpoints; the dashboard is optional.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
