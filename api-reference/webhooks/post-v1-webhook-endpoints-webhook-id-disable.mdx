Disable a webhook endpoint without deleting delivery history

<Info>
Audience: application and admin.
</Info>

## Pause without erasing evidence

Disable an endpoint to remove it from future fanout while retaining its configuration and recorded delivery history. This does not cancel a request that a remote receiver has already accepted.

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X POST \
  -H "Authorization: Bearer $SECAPI_BEARER_TOKEN" \
  -H "secapi-version: 2026-03-19" \
  -H "content-type: application/json" \
  -d '{}' \
  "https://api.secapi.ai/v1/webhook_endpoints/wh_example_123/disable"
```
</RequestExample>

<Info>
This organization control-plane action accepts bearer organization REST authentication. API keys cannot mutate webhook endpoints. The dashboard is optional.
</Info>

## Give this prompt to your agent

<Prompt>
Disable the endpoint with bearer organization REST authentication to stop future matching events without erasing delivery history. API keys cannot mutate webhook endpoints; the dashboard is optional.
</Prompt>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
