Return the latest normalized 13F ownership report for a manager CIK, with optional quarter-specific selection

<Info>
Audience: application and coding agent.
</Info>

## Agent mode

Pass `?view=agent` to receive the compact agent shape: manager name, form, filingDate, accessionNumber + lifted `filingUrl` + nested `holdings[]` capped to the first 50 positions and trimmed to 6 fields per position (issuer, cusip, valueUsdThousands, shares, shareType, putCall). `holdingCount` reports the parsed filing total, `holdingsReturned` reports the agent payload rows, and `holdingsTruncated` makes partial payloads explicit. Classes, investment discretion, and metadata envelopes are dropped. Typical reduction: **~35–45%** depending on position count. See [api conventions → response formats](/api-conventions) for the full `?view=` contract.

## Canonical metadata

- `requestId`
- `traceparent`
- `provenance`
- `freshness`
- `materialization`

## Example request

<RequestExample>
```bash
curl -X GET -H "x-api-key: $SECAPI_API_KEY" -H "secapi-version: 2026-03-19" "https://api.secapi.ai/v1/owners/13f?cik=0001067983&limit=20"
```
</RequestExample>

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
