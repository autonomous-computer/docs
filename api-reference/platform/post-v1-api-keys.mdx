# POST /v1/api_keys

Create a new API key for the current organization

<Info>
Use this endpoint from server-side applications, SDKs, or agent workflows.
</Info>

## Response metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X POST \
  -H "x-api-key: $SEC_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  -H "content-type: application/json" \
  -d '{}' \
  "https://api.secapi.ai/v1/api_keys"
```
</RequestExample>

## Example response

<ResponseExample>
```json
{
  "requestId": "req_example_123",
  "traceparent": "00-4bf92f3577b34da6a3ce929d0e0e4736-00f067aa0ba902b7-01"
}
```
</ResponseExample>

## Give this prompt to your agent

Use SEC API POST /v1/api_keys to create a new API key for the current organization. Preserve `requestId`, `traceparent` in the output. Keep the response fields your application uses for follow-up requests.
</Prompt>

## Errors and partial results

- Treat non-2xx responses as API errors and log the response body for debugging
- Log `requestId` and `traceparent` so SEC API support can trace the request
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
