# Platform

Identity, organization, auth metadata, billing, usage, and event endpoints.

<Note>
The sidebar labels inside this family are the literal calls you make over HTTP. Descriptions stay in the body so the navigation stays sharp and scannable.
</Note>

<Card title="GET /v1/billing/rates" href="/api-reference/platform/get-v1-billing-rates">Return the public pricing catalog, plan metadata, and meter-family launch rates</Card>
<Card title="GET /v1/events" href="/api-reference/platform/get-v1-events">List canonical event, webhook delivery, and stream records for the current organization</Card>
<Card title="GET /v1/events/auditor-changes" href="/api-reference/platform/get-v1-events-auditor-changes">Return 8-K Item 4.01 auditor change events with change-type classification (dismissal, resignation, engagement)</Card>

<Card title="GET /v1/events/export" href="/api-reference/platform/get-v1-events-export">Export filtered canonical events as JSON or NDJSON</Card>
<Card title="GET /v1/events/officer-changes" href="/api-reference/platform/get-v1-events-officer-changes">Return 8-K Item 5.02 officer and director change events with change-type classification (appointment, departure, resignation, termination)</Card>
<Card title="GET /v1/events/restatements" href="/api-reference/platform/get-v1-events-restatements">Return 8-K Item 4.02 restatement and non-reliance events with severity classification and affected-period extraction</Card>
<Card title="GET /v1/events/voting-results" href="/api-reference/platform/get-v1-events-voting-results">Return 8-K Item 5.07 voting results with structured proposals, vote counts, and approval outcomes</Card>
<Card title="GET /v1/me" href="/api-reference/platform/get-v1-me">Return the current authenticated principal and organization context</Card>
<Card title="GET /v1/org" href="/api-reference/platform/get-v1-org">Return the current organization profile</Card>
