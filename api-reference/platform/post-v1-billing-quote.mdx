Quote a billable workflow or meter class against the current billing plan and budget gates

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`
- `planKey`
- `meterClass`
- `units`
- `budget`
- `budgetGate`

## Example request

<RequestExample>
```bash
curl -X POST \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  -H "content-type: application/json" \
  -d '{"path":"/v1/facts","method":"GET","units":1}' \
  "https://api.secapi.ai/v1/billing/quote"
```
</RequestExample>

## Request body

Provide meterClass or path. method is used with path and defaults to GET; planKey and positive units are optional. The quote reflects the current plan and budget gate. It does not reserve usage or guarantee a later request's billing state.

## Failure posture

- treat non-2xx responses as contract-aware failures, not free-form errors
- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
