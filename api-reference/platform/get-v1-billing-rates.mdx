Return the public pricing catalog, starter grant metadata, plan metadata, and meter-family launch rates

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X GET \
  -H "x-api-key: $SECAPI_API_KEY" \
  -H "secapi-version: 2026-03-19" \
  "https://api.secapi.ai/v1/billing/rates"
```
</RequestExample>

<Info>
`starterGrant` is included even though it is not a paid checkout plan. Agents should read it together with `GET /v1/billing` and `GET /v1/limits` to explain the included-call grant, overflow behavior, meter rates, and current account state without inferring from marketing copy.
</Info>

- preserve `requestId` and `traceparent` in logs and downstream reports
- if provenance or freshness metadata is present, return it unchanged so trust is not lost in the handoff
