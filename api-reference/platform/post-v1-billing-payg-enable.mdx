Enable pay-as-you-go in one flow: save a card, make an initial credit top-up, and optionally turn on auto-top-up (returns a PaymentIntent client secret)

<Info>
Audience: application and coding agent.
</Info>

## Canonical metadata

- `requestId`
- `traceparent`

## Example request

<RequestExample>
```bash
curl -X POST \
  -H "Authorization: Bearer $SECAPI_BEARER_TOKEN" \
  -H "secapi-version: 2026-03-19" \
  -H "content-type: application/json" \
  -d '{"amountUsd":10,"autoTopupThresholdCents":1000,"autoTopupAmountCents":2500}' \
  "https://api.secapi.ai/v1/billing/payg/enable"
```
</RequestExample>

<Info>
Billing funding changes require a WorkOS bearer session with `billing:manage`. API keys are rejected so payment setup, refunds, and auto-top-up controls stay limited to interactive billing administrators.
</Info>
