---
title: "Build an SEC Filing Monitor"
description: "Poll supported filing search endpoints, send idempotent alerts, and retain provenance."
---

This Node.js job polls three issuers for new 8-K filings and writes a
source-linked alert list. Run it in cron, a container scheduler, or a serverless
job; the local JSON state is only suitable for one worker.

## Prerequisites

- Node.js 18+ and `SECAPI_API_KEY`.
- Issuer tickers to poll. The example uses `AAPL`, `MSFT`, and `NVDA`.

```bash
export SECAPI_API_KEY="your_api_key"
```

Create `monitor-filings.mjs`:

```js
import { readFile, writeFile } from "node:fs/promises";

const statePath = new URL("./state.json", import.meta.url);
const apiKey = process.env.SECAPI_API_KEY;
if (!apiKey) throw new Error("SECAPI_API_KEY is required");

let state;
try {
  state = JSON.parse(await readFile(statePath, "utf8"));
} catch (error) {
  if (error?.code !== "ENOENT") throw error;
  state = { seen: [] };
}
const seen = new Set(state.seen ?? []);
const alerts = [];
for (const ticker of ["AAPL", "MSFT", "NVDA"]) {
  const query = new URLSearchParams({ ticker, forms: "8-K,8-K/A", limit: "50" });
  const response = await fetch(`https://api.secapi.ai/v1/filings?${query}`, {
    headers: { "x-api-key": apiKey },
  });
  if (!response.ok) throw new Error(`${ticker}: ${response.status}`);
  const payload = await response.json();
  for (const filing of payload.data ?? []) {
    const accession = filing.accessionNumber ?? filing.accession_number;
    if (accession && !seen.has(accession)) {
      alerts.push({
        accession,
        filingUrl: filing.filingUrl ?? filing.provenance?.filingUrl ?? null,
        requestId: payload.requestId,
      });
      seen.add(accession);
    }
  }
}
await writeFile(statePath, JSON.stringify({ seen: [...seen].slice(-5000) }, null, 2));
console.log({ newFilings: alerts.length, alerts });
```

Run `node monitor-filings.mjs`. The first run may list existing filings because
the state is empty. A successful result prints `newFilings` and source-linked
records with the accession, filing URL when returned, and request ID.

## Common errors and production use

- `401` or `403` means the runtime did not send a valid entitled API key.
- An empty page is not proof that an issuer made no filing; continue through
  `nextCursor` while `hasMore` is true when the chosen window can exceed a page.
- Use a transactional idempotency key based on accession number, overlap polling
  windows, and retain query filters, response metadata, and alert status.

Alerting identifies disclosures for review. It does not determine materiality.
