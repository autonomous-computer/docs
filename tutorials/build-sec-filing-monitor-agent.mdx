---
title: "Build an SEC Filing Monitor"
description: "Poll supported filing search endpoints, send idempotent alerts, and retain provenance."
---

This Node.js job polls for new 8-K and 8-K/A filings. Run it in cron, a
container scheduler, or a serverless job; the local JSON state is for one worker.

```js
import { readFile, writeFile } from "node:fs/promises";

const statePath = new URL("./state.json", import.meta.url);
const apiKey = process.env.SECAPI_API_KEY;
if (!apiKey) throw new Error("SECAPI_API_KEY is required");

let state;
try {
  state = JSON.parse(await readFile(statePath, "utf8"));
} catch (error) {
  if (error?.code !== "ENOENT") throw error;
  state = { seen: [] };
}
const seen = new Set(state.seen ?? []);
const alerts = [];
for (const ticker of ["AAPL", "MSFT", "NVDA"]) {
  const query = new URLSearchParams({ ticker, forms: "8-K,8-K/A", limit: "50", sort: "filing_date_desc" });
  const response = await fetch(`https://api.secapi.ai/v1/filings?${query}`, {
    headers: { "x-api-key": apiKey },
  });
  if (!response.ok) throw new Error(`${ticker}: ${response.status}`);
  const payload = await response.json();
  for (const filing of payload.data ?? []) {
    const accession = filing.accessionNumber ?? filing.accession_number;
    if (accession && !seen.has(accession)) {
      alerts.push({
        accession,
        filingUrl: filing.provenance?.filingUrl ?? null,
        requestId: payload.requestId,
      });
      seen.add(accession);
    }
  }
}
await writeFile(statePath, JSON.stringify({ seen: [...seen].slice(-5000) }, null, 2));
console.log({ newFilings: alerts.length, alerts });
```

The first run may return existing filings because the state is empty. In
production, use a transactional idempotency key based on accession number,
continue with `nextCursor` when `hasMore` is true, overlap polling windows, and
reconcile a recent date range. Retain request IDs, filing URLs, query filters,
and alert status. Alerting discovers disclosures; it does not determine
materiality.
