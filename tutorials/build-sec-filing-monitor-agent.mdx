---
title: "Build a scheduled filing review queue"
description: "Poll a bounded SEC filing query, de-duplicate accession numbers, and send evidence-linked items to a reviewer"
---

Use this job when your scheduler needs checkpoints and you do not want an inbound webhook receiver. It polls recent filings, records new accession numbers, and emits a review queue. It does not determine whether a filing is material or provide a complete EDGAR archive.

## Prove the query before scheduling it

`GET /v1/filings` accepts ticker, `forms`, filing-date filters, cursor pagination, and filing-date sorting. Check a deliberately small result before turning it into a recurring job.

```bash
curl --get --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  --data-urlencode "ticker=AAPL" \
  --data-urlencode "forms=8-K,8-K/A" \
  --data-urlencode "limit=10" \
  --data-urlencode "sort=filing_date_desc" \
  "https://api.secapi.ai/v1/filings" \
  | jq '{data, hasMore, nextCursor, requestId, traceparent}'
```

Confirm that the company and form scope match the job you intend to run. A broad query creates an expensive and noisy review queue.

## Poll and de-duplicate

Create `monitor.mjs`. The example uses a local JSON file only to show the key; use a durable store in production.

```js
import { readFile, writeFile } from "node:fs/promises";

const apiKey = process.env.SECAPI_API_KEY;
if (!apiKey) throw new Error("SECAPI_API_KEY is required");
const statePath = new URL("./state.json", import.meta.url);
const state = JSON.parse(await readFile(statePath, "utf8").catch(() => '{"seen":[]}'));
const seen = new Set(state.seen ?? []);
const alerts = [];

for (const ticker of ["AAPL", "MSFT", "NVDA"]) {
  const query = new URLSearchParams({
    ticker, forms: "8-K,8-K/A", limit: "50", sort: "filing_date_desc",
  });
  const response = await fetch(`https://api.secapi.ai/v1/filings?${query}`, {
    headers: { "x-api-key": apiKey },
  });
  if (!response.ok) throw new Error(`${ticker}: ${response.status} ${await response.text()}`);
  const page = await response.json();
  for (const filing of page.data ?? []) {
    if (!filing.accessionNumber || seen.has(filing.accessionNumber)) continue;
    alerts.push({
      ticker, form: filing.form, filingDate: filing.filingDate,
      accessionNumber: filing.accessionNumber,
      filingUrl: filing.filingUrl ?? filing.provenance?.filingUrl ?? null,
      requestId: page.requestId, traceparent: page.traceparent,
    });
    seen.add(filing.accessionNumber);
  }
}

await writeFile(statePath, JSON.stringify({ seen: [...seen].slice(-5000) }, null, 2));
console.log(JSON.stringify({ checked: 3, newFilings: alerts.length, alerts }, null, 2));
```

Run the script with `node monitor.mjs`. The output is a source-linked queue. The first run is a backfill and may report older filings; do not label it as a real-time alert.

## Preserve the evidence boundary

Each queue item should carry accession number, filing date, source URL when returned, `requestId`, and `traceparent`. A downstream agent may summarize the filing's disclosed facts, but it should cite the accession and source URL and must not infer materiality from a form label or title.

Follow `nextCursor` while `hasMore` is true whenever the requested window exceeds one page. Run a short overlapping-window reconciliation because late processing or transient failures can otherwise leave a gap.

## Failure posture

`401` and `403` mean the scheduler lacks a valid entitled key. Treat accession number as the filing identity and enforce it as a unique key to prevent duplicate alerts. In production, write the accession record and enqueue the review in one transaction, use bounded retry/backoff, and measure pages, new records, and failed jobs.
