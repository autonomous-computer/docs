---
title: "Find Risk Factors with Semantic Search"
description: "Search SEC filing passages by meaning, preserve citation metadata, and use the results as a research queue rather than a conclusion engine."
---

Semantic search can surface related language that a literal keyword search misses. It does not establish materiality or completeness. Treat the response as a ranked route into the underlying filing and retain the returned citation fields with every excerpt.

## Prerequisites

- An API key exported as `SECAPI_API_KEY`.
- `curl` and `jq`, or Node.js 18+.
- A natural-language research question. This example searches semiconductor supply-chain risk.

```bash
export SECAPI_API_KEY="your_api_key"
```

## 1. Run a scoped hybrid search

`GET /v1/search/semantic` requires `q`. The `mode` parameter accepts `keyword`, `semantic`, or `hybrid`; hybrid is a practical starting point when you want both terminology and conceptual relevance. Scope by `ticker`, `cik`, `form`, and `filing_year` when the question calls for it.

```bash
curl --get --fail-with-body \
  -H "x-api-key: $SECAPI_API_KEY" \
  --data-urlencode "q=supply chain disruption risk" \
  --data-urlencode "ticker=NVDA" \
  --data-urlencode "form=10-K" \
  --data-urlencode "mode=hybrid" \
  --data-urlencode "limit=10" \
  "https://api.secapi.ai/v1/search/semantic" \
  | jq '{sections, requestId, traceparent}'
```

**Result:** ranked matching passages in the search response. Before publishing or scoring a result, retain `accession`, `source_url`, `section_key`, `char_start`, `char_end`, and `highlighted_snippet`; open the cited filing to review the passage in context.

## 2. Compare a question across annual filings

Use the same query and issuer scope for each filing year. A changed rank or a new match is a research lead, not proof that the risk changed.

```js
const base = "https://api.secapi.ai/v1/search/semantic";
const headers = { "x-api-key": process.env.SECAPI_API_KEY ?? "" };

for (const filingYear of [2023, 2024, 2025]) {
  const query = new URLSearchParams({
    q: "artificial intelligence regulatory risk",
    ticker: "MSFT",
    form: "10-K",
    filing_year: String(filingYear),
    mode: "hybrid",
    limit: "5",
  });
  const response = await fetch(`${base}?${query}`, { headers });
  if (!response.ok) throw new Error(`SEC API ${response.status}: ${await response.text()}`);

  const payload = await response.json();
  const hits = payload.sections?.data ?? payload.data ?? [];
  console.log(JSON.stringify({
    filingYear,
    requestId: payload.requestId,
    hits: hits.map((hit) => ({
      companyName: hit.companyName ?? null,
      form: hit.form ?? null,
      filingDate: hit.filingDate ?? null,
      accession: hit.accession,
      source_url: hit.source_url,
      section_key: hit.section_key,
      char_start: hit.char_start,
      char_end: hit.char_end,
      highlighted_snippet: hit.highlighted_snippet,
    })),
  }, null, 2));
}
```

## Common failure modes

- `400`: `q` is required; confirm `mode` is `keyword`, `semantic`, or `hybrid` and `limit` is 1-100.
- Empty or weak matches: narrow by issuer, form, or filing year, then try both a concrete phrase and a business question. Do not compensate by inventing a relevance threshold.
- Lost evidence: a result without `accession`, `source_url`, `section_key`, `char_start`, `char_end`, and `highlighted_snippet` cannot be independently reviewed.
- Unstable comparisons: compare like with like, including form type and reporting period; an annual report and an interim report answer different questions.

## Production notes

Store the query, filters, retrieval mode, request ID, and every citation field. Cache repeated research queries, bound the number of results you present to a reviewer, and retrieve the cited filing section before using an excerpt in a report. Model outputs or search ranks should never be represented as legal, accounting, or investment advice.
