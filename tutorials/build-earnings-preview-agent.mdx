---
title: "Build an Earnings Preview Agent"
description: "Create a source-preserving pre-earnings research brief from the earnings-preview, filing, and insider-data routes."
---

This tutorial builds a small Node.js job that produces a pre-earnings review packet for one issuer. It collects an earnings-preview response, the latest annual filing identity, and recent Form 4 rows, then writes the raw evidence and a compact handoff. It is not a forecast, recommendation, or statement that a reported date will occur.

## Prerequisites

- Node.js 18+ and an API key in `SECAPI_API_KEY`.
- A ticker. This example uses `AAPL`.
- A local directory where it is appropriate to retain SEC-derived research output.

```bash
export SECAPI_API_KEY="your_api_key"
mkdir earnings-preview-agent && cd earnings-preview-agent
```

## 1. Check the three evidence sources

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/intelligence/earnings-preview?ticker=AAPL" \
  | jq '{requestId, traceparent, provenance, freshness, data}'

curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/latest?ticker=AAPL&form=10-K" \
  | jq '{accessionNumber, filingDate, filingUrl, requestId, provenance}'

curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/insiders?ticker=AAPL&forms=4&limit=20" \
  | jq '{data, hasMore, nextCursor, requestId, traceparent}'
```

Before automating, inspect the returned shapes and source links. The earnings-preview response is a scoped research input, the 10-K is the annual filing record, and Form 4 rows describe reported transactions. Keep those roles distinct.

## 2. Write the preview packet

Create `preview.mjs`:

```js
import { mkdir, writeFile } from "node:fs/promises";

const ticker = (process.argv[2] ?? "AAPL").toUpperCase();
const apiKey = process.env.SECAPI_API_KEY;
const baseUrl = "https://api.secapi.ai";
if (!apiKey) throw new Error("SECAPI_API_KEY is required");

async function get(path) {
  const response = await fetch(`${baseUrl}${path}`, { headers: { "x-api-key": apiKey } });
  if (!response.ok) throw new Error(`${path}: ${response.status} ${await response.text()}`);
  return response.json();
}

const encodedTicker = encodeURIComponent(ticker);
const [earnings, annualFiling, insiders] = await Promise.all([
  get(`/v1/intelligence/earnings-preview?ticker=${encodedTicker}`),
  get(`/v1/filings/latest?ticker=${encodedTicker}&form=10-K`),
  get(`/v1/insiders?ticker=${encodedTicker}&forms=4&limit=20`),
]);

const packet = {
  generatedAt: new Date().toISOString(), ticker,
  earningsPreview: earnings,
  annualFiling: {
    accessionNumber: annualFiling.accessionNumber,
    filingDate: annualFiling.filingDate,
    filingUrl: annualFiling.filingUrl ?? annualFiling.provenance?.filingUrl ?? null,
    requestId: annualFiling.requestId,
    provenance: annualFiling.provenance ?? null,
  },
  insiderRows: insiders.data ?? [],
  insiderRequest: { requestId: insiders.requestId, traceparent: insiders.traceparent,
    hasMore: insiders.hasMore, nextCursor: insiders.nextCursor },
};

await mkdir("output", { recursive: true });
await writeFile(`output/${ticker.toLowerCase()}-earnings-packet.json`, JSON.stringify(packet, null, 2));
console.log(JSON.stringify({ ticker, output: `output/${ticker.toLowerCase()}-earnings-packet.json`,
  annualFiling: packet.annualFiling, insiderRows: packet.insiderRows.length,
  earningsPreviewRequestId: earnings.requestId }, null, 2));
```

Run `node preview.mjs AAPL`. The expected result is a JSON packet with the untouched responses and an annual-filing pointer. The console deliberately reports only identifiers and counts; actual values depend on the issuer and when the job runs.

## Source and provenance

The packet preserves three different SEC-derived inputs: the earnings-preview response, the issuer's returned Form 10-K identity, and Form 4 transaction rows. Retain every response's `requestId`, `traceparent` when present, filing accession, filing URL, and generated timestamp. These fields let a reviewer reopen the source material instead of treating the packet as the source.

## 3. Produce a human review brief

Give an analyst or agent the JSON packet with this instruction:

```text
Summarize the returned earnings-preview fields and recent Form 4 rows for a review
brief. Cite the annual filing accession and URL, preserve each request ID, and
separate reported facts from interpretation. Do not create an earnings forecast,
call a transaction bullish or bearish, or treat absent rows as no activity.
```

Read the linked filing and every material Form 4 before distributing commentary. Form 4 codes are SEC transaction codes, not investment sentiment labels.

## Limits and failure modes

- A `401` or `403` means the runtime did not receive a valid entitled key; do not write a partial packet as if it were complete.
- The three calls have independent availability. If one fails, fail the job or label the packet incomplete rather than silently substituting empty data.
- Follow `nextCursor` when `hasMore` is true if the insider history must be complete for the chosen window.
- The latest 10-K may not be the filing most relevant to an imminent report; preserve the filing date and select any additional source deliberately.
