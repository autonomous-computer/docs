---
title: "Build an earnings preview agent"
description: "Create a source-linked earnings brief from the SEC API earnings preview bundle for one issuer"
---

Use this job to create a compact pre-earnings research brief for one issuer. It saves the returned bundle and renders its disclosed drivers and risks with the source context intact. It does not predict results or recommend a trade.

## Set up a clean environment

You need Python 3.9 or later, `requests`, and an SEC API key. This example uses NVIDIA.

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install requests
export SECAPI_API_KEY="your_api_key"
```

## Request and save the preview bundle

Create `earnings_preview.py`:

```python
import json
import os
from pathlib import Path

import requests

ticker = "NVDA"
response = requests.get(
    "https://api.secapi.ai/v1/intelligence/earnings-preview",
    headers={"x-api-key": os.environ["SECAPI_API_KEY"]},
    params={"ticker": ticker},
    timeout=30,
)
response.raise_for_status()
bundle = response.json()

Path("earnings-preview.json").write_text(json.dumps(bundle, indent=2))
print(json.dumps({
    "symbol": bundle.get("symbol"),
    "asOf": bundle.get("asOf"),
    "drivers": len(bundle.get("keyDrivers", [])),
    "risks": len(bundle.get("keyRisks", [])),
    "requestId": bundle.get("requestId"),
}, indent=2))
```

Run `python earnings_preview.py`. The command writes `earnings-preview.json` and prints the issuer, as-of timestamp, counts of returned drivers and risks, and request ID. The exact content varies by issuer and available supporting material.

## Render a reviewer brief

Append this code after the request above:

```python
def label(item):
    return item.get("label") or item.get("key") or "Untitled item"

lines = [
    f"# {bundle.get('symbol', ticker)} earnings preview",
    f"As of: {bundle.get('asOf', 'not supplied')}",
    f"Request ID: {bundle.get('requestId', 'not supplied')}",
    "",
    "## Summary",
    bundle.get("summaryMd", "No summary returned."),
    "",
    "## Key drivers",
]
lines.extend(
    f"- {label(item)}: {item.get('summary') or item.get('explanation') or 'No explanation returned.'}"
    for item in bundle.get("keyDrivers", [])
)
lines.extend(["", "## Key risks"])
lines.extend(
    f"- {label(item)}: {item.get('summary') or item.get('explanation') or 'No explanation returned.'}"
    for item in bundle.get("keyRisks", [])
)
Path("earnings-preview.md").write_text("\n".join(lines) + "\n")
```

This adds `earnings-preview.md`. Keep the JSON bundle beside the rendered brief so a reviewer can inspect `trace`, provenance, freshness, or other supporting context returned with an item.

## Limits and failure modes

- A non-2xx response is a failed retrieval, not evidence that an issuer has no earnings context. Preserve the response body, `requestId`, and `traceparent` when available.
- Do not fill an absent field with a third-party value and present it as SEC API output. Keep external data separately labeled and sourced.
- Drivers and risks are research inputs, not forecasts or materiality determinations. Open the returned trace or source context before quoting one in an investment, compliance, or public-facing brief.
- Save each run with its `asOf` value. A later run can return a different bundle as filings and supporting material change.
