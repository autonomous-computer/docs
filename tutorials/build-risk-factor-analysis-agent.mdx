---
title: "Build a filing risk-review queue"
description: "Use semantic retrieval to collect cited 10-K passages for review without turning retrieval rank into an automated conclusion"
---

This Python job collects candidate risk-disclosure passages for one issuer and writes a citation queue. It finds evidence by meaning. It does not prove a risk is new, material, complete, or legally significant; a reviewer must read the cited filing section in context.

## Set up a clean environment

You need Python 3.9 or later, `requests`, and an API key.

```bash
python -m pip install requests
export SECAPI_API_KEY="your_api_key"
```

## Test a bounded question

Use an API key and a question specific enough to inspect. This example asks about a potential supply-chain disclosure in NVIDIA's 10-K.

```bash
curl --get --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  --data-urlencode "q=supply chain disruption risk" \
  --data-urlencode "ticker=NVDA" \
  --data-urlencode "form=10-K" \
  --data-urlencode "mode=hybrid" \
  --data-urlencode "limit=10" \
  "https://api.secapi.ai/v1/search/semantic" \
  | jq '{query, mode, sections, requestId, traceparent}'
```

The response contains a ranked `sections.data` list. A row can include `accession`, `source_url`, `section_key`, `char_start`, `char_end`, and `highlighted_snippet`. Citation-span fields may be null when validation is degraded; keep the filing identity even in that case.

## Write the review queue

Create `risk_queue.py`:

```python
import json
import os
import sys
from pathlib import Path
import requests

ticker = (sys.argv[1] if len(sys.argv) > 1 else "NVDA").upper()
query = sys.argv[2] if len(sys.argv) > 2 else "supply chain disruption risk"

response = requests.get(
    "https://api.secapi.ai/v1/search/semantic",
    headers={"x-api-key": os.environ["SECAPI_API_KEY"]},
    params={"q": query, "ticker": ticker, "form": "10-K", "mode": "hybrid", "limit": 10},
    timeout=30,
)
response.raise_for_status()
payload = response.json()

queue = [{
    "ticker": row.get("ticker"), "form": row.get("form"),
    "accession": row.get("accession"), "source_url": row.get("source_url"),
    "section_key": row.get("section_key"), "char_start": row.get("char_start"),
    "char_end": row.get("char_end"), "highlighted_snippet": row.get("highlighted_snippet"),
    "citation_degraded": row.get("_citation_degraded"),
} for row in payload.get("sections", {}).get("data", [])]

artifact = {"query": payload.get("query"), "mode": payload.get("mode"),
            "ticker": ticker, "queue": queue, "requestId": payload.get("requestId"),
            "traceparent": payload.get("traceparent")}
Path("risk-review-queue.json").write_text(json.dumps(artifact, indent=2))
print(json.dumps({"queueItems": len(queue), "requestId": artifact["requestId"]}, indent=2))
```

Run `python risk_queue.py NVDA "supply chain disruption risk"`. The output is `risk-review-queue.json`, a list for a reviewer to inspect, not a change score.

## Compare years without overstating retrieval

When the question is historical, add a distinct `filing_year` value to each scoped query and save each artifact. Compare cited source passages from comparable forms. A different rank, snippet, or section arrangement does not prove management added, removed, or changed a risk.

## Source limits and failures

`q` is required; `keyword`, `semantic`, and `hybrid` are the supported modes; `limit` is 1 through 100. An empty result can mean the scope is too narrow or the language is not indexed as expected. It does not establish that no risk disclosure exists.

For every queue item, retain accession, source URL, section key, available span, query, mode, `requestId`, and `traceparent`. When `_citation_degraded` is set or citation fields are absent, open the full source using the filing identity rather than treating a truncated snippet as proof.
