title: "Build a Risk Factor Analysis Agent"
description: "Retrieve two Item 1A sections and produce a reviewable text comparison."
---

Compare Item 1A from two named 10-K filings. The output is a review artifact,
not a materiality score or legal conclusion.

## 1. Find two accessions

```bash
curl --get --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  --data-urlencode "ticker=AAPL" --data-urlencode "form=10-K" \
  --data-urlencode "limit=5" --data-urlencode "sort=filing_date_desc" \
  "https://api.secapi.ai/v1/filings"
```

Record two accession numbers, filing dates, and URLs from the result. Do not
quietly compare different forms.

## 2. Fetch Item 1A and generate a diff

```python
import difflib, os, requests
from pathlib import Path

def section(accession):
    response = requests.get(
        f"https://api.secapi.ai/v1/filings/{accession}/sections/item_1a",
        headers={"x-api-key": os.environ["SECAPI_API_KEY"]}, timeout=30,
    )
    response.raise_for_status()
    payload = response.json(); data = payload.get("data", payload)
    return data.get("contentMd", data.get("text", "")), payload.get("requestId")

current, current_id = section("CURRENT_ACCESSION_NUMBER")
previous, previous_id = section("PREVIOUS_ACCESSION_NUMBER")
if not current or not previous: raise RuntimeError("Item 1A was unavailable")
diff = difflib.unified_diff(previous.splitlines(), current.splitlines(), lineterm="")
Path("item-1a-comparison.diff").write_text("\n".join(diff) + "\n")
print({"currentRequestId": current_id, "previousRequestId": previous_id})
```

Read the diff beside both filings. Formatting, reordering, or extraction
differences can create large changes without a new disclosure. Persist accessions,
section keys, filing URLs, request IDs, and the comparison version. Route empty
sections and `404` responses to review instead of creating a negative finding.
