This tutorial builds a Python script that requests the latest two parsable 13F reports for a manager and writes a reviewable change report. The output is a record of reported holdings, not a claim about a manager's current portfolio or intent. The comparison route chooses the two reports; keep its filing dates and provenance so the report can be checked later.

## What you need

- An SEC API key in `SECAPI_API_KEY`
- Python 3.9 or later
- The manager CIK you want to track

Install the only dependency:

```bash
python -m pip install requests
```

## Request the comparison

Create `tracker.py`:

```python
import json
import os
from pathlib import Path

import requests

BASE_URL = "https://api.secapi.ai"
HEADERS = {"x-api-key": os.environ["SECAPI_API_KEY"]}
MANAGER_CIK = "0001067983"


def get_latest_comparison(cik: str) -> dict:
    response = requests.post(
        f"{BASE_URL}/v1/owners/13f/compare",
        headers={**HEADERS, "content-type": "application/json"},
        json={"cik": cik, "limit": 50},
        timeout=30,
    )
    response.raise_for_status()
    return response.json()


comparison = get_latest_comparison(MANAGER_CIK)
Path("13f-comparison.json").write_text(json.dumps(comparison, indent=2))

print(comparison.get("managerName"))
print("current filing:", comparison.get("currentFilingDate"))
print("previous filing:", comparison.get("previousFilingDate"))
print("request ID:", comparison.get("requestId"))
```

Run it with:

```bash
SECAPI_API_KEY=secapi_your_key python tracker.py
```

The expected console output contains the returned manager name, two filing dates, and a request ID. `13f-comparison.json` retains the full response; do not replace it with the rendered report when a reviewer needs the original row fields.

## Create a review report

Append this function to group the returned rows without inventing a trading interpretation:

```python
def write_report(comparison: dict) -> str:
    rows = comparison.get("rows", [])
    groups = {"added": [], "removed": [], "changed": [], "unchanged": []}
    for row in rows:
        groups.setdefault(row.get("status", "unknown"), []).append(row)

    lines = [
        f"# {comparison.get('managerName', 'Manager')} 13F comparison",
        "",
        f"Current filing date: {comparison.get('currentFilingDate')}",
        f"Previous filing date: {comparison.get('previousFilingDate')}",
        f"Request ID: {comparison.get('requestId')}",
        "",
    ]
    for status in ("added", "removed", "changed"):
        lines.append(f"## {status.title()} positions")
        for row in groups[status]:
            lines.append(
                f"- {row.get('issuer')} ({row.get('cusip')}): "
                f"{row.get('previousShares')} -> {row.get('currentShares')} shares"
            )
        if not groups[status]:
            lines.append("- None returned")
        lines.append("")

    lines.extend([
        "## Interpretation limit",
        "These are differences between two reported 13F filings. They do not show "
        "the manager's complete or current portfolio, trade dates, or investment intent.",
    ])
    return "\n".join(lines)


Path("13f-comparison.md").write_text(write_report(comparison))
```

The API selects the latest two parsable reports. It does not accept two arbitrary report periods. If your workflow needs a particular quarter, list the manager's filings and request each report using the returned `reportDate` or `filingDate`.

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/owners/13f/filings?cik=0001067983&limit=8"
```

## Source and provenance

The comparison is derived from the reporting manager's two latest parsable Form 13F reports selected by the API. Keep the manager CIK, both report and filing dates, accession numbers or filing URLs returned in `provenance`, and `requestId` in the JSON and markdown outputs. Open the linked SEC report before describing a material change to another person.

## Limits and failure modes

The comparison request requires a CIK and accepts a limit from 1 through 50. `requests.raise_for_status()` surfaces invalid requests and service failures rather than writing an empty report. A row change can result from corporate actions, amendments, or reporting conventions that a 13F row cannot explain; Form 13F is neither a complete exposure map nor a current portfolio. Log the response's request identifier when available, and retry only according to your application's failure policy.
