Retrieve the latest two 13F reports for a manager and save the comparison. This
job does not infer trades, performance, or current ownership.

## Prerequisites

- Python 3.9+ and `SECAPI_API_KEY`.
- A manager CIK and durable storage for production results.

```python
import json, os
from datetime import datetime, timezone
from pathlib import Path
import requests

cik = "0001067983"
response = requests.post(
    "https://api.secapi.ai/v1/owners/13f/compare",
    headers={"x-api-key": os.environ["SECAPI_API_KEY"]},
    json={"cik": cik, "limit": 50}, timeout=30,
)
response.raise_for_status()
payload = response.json()
stamp = datetime.now(timezone.utc).strftime("%Y%m%dT%H%M%SZ")
output = Path(f"13f-compare-{cik}-{stamp}.json")
output.write_text(json.dumps(payload, indent=2) + "\n")
print({"output": str(output), "requestId": payload.get("requestId")})
```

A successful run writes the service's selected-report comparison and prints the
file path plus `requestId`. Inspect the returned shape before building a
dashboard; keep accession numbers, report dates, CUSIPs, filing URLs where
provided, and request metadata with every summary.

Handle `400` as an invalid CIK, `401`/`403` as an access problem, and temporary
`5xx` responses with bounded retries. Do not mark a manager unchanged after an
incomplete or failed retrieval.

## Limitation and next steps

13F data is delayed and limited to reportable securities. A new or closed
position is a change between selected reports, not proof of a real-time trade.
