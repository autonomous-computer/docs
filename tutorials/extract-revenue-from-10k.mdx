---
title: "Extract Revenue from a 10-K"
description: "Retrieve annual SEC revenue facts with the selected concept, period, unit, and filing provenance intact."
---

Use this journey when you need annual reported revenue for a model, diligence memo, or data pipeline. The returned number is only useful when its unit, reporting period, selected XBRL concept, and filing provenance travel with it. The finished output is a reproducible observation record, not a display-formatted headline number.

## Prerequisites

- `SECAPI_API_KEY`, `curl`, and `jq`, or Node.js 18+.
- A stable issuer identifier. This example uses `AAPL`.

```bash
export SECAPI_API_KEY="your_api_key"
```

## 1. Identify the annual filing

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/filings/latest?ticker=AAPL&form=10-K" \
  | jq '{accessionNumber, filingDate, form, companyName, filingUrl, requestId, provenance}'
```

Save the accession number, filing URL, filing date, and request ID. This is the source document you will use to investigate a surprising number.

## 2. Retrieve annual revenue facts

```bash
curl --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/facts?ticker=AAPL&tag=Revenues&form=10-K&limit=4" \
  | jq '{requestedTag, resolvedTag, aliasStrategy, data: [.data[] | {value, unit, periodEnd, fy, fp, form, filedAt, provenance}], requestId, traceparent}'
```

The expected result is up to four annual fact observations. `resolvedTag` records the concept actually selected; values are raw units, not display-formatted dollars.

## 3. Write a provenance-preserving result

```js
const response = await fetch("https://api.secapi.ai/v1/facts?ticker=AAPL&tag=Revenues&form=10-K&limit=4", {
  headers: { "x-api-key": process.env.SECAPI_API_KEY ?? "" },
});
if (!response.ok) throw new Error(`SEC API ${response.status}: ${await response.text()}`);
const payload = await response.json();
console.log(JSON.stringify({ requestedTag: payload.requestedTag, resolvedTag: payload.resolvedTag,
  observations: (payload.data ?? []).map((fact) => ({ value: fact.value, unit: fact.unit,
    periodEnd: fact.periodEnd, fiscalYear: fact.fy, fiscalPeriod: fact.fp, form: fact.form,
    filedAt: fact.filedAt, accessionNumber: fact.provenance?.accessionNumber ?? null,
    filingUrl: fact.provenance?.filingUrl ?? null })), requestId: payload.requestId,
  traceparent: payload.traceparent }, null, 2));
```

## Common errors and false positives

- `401` or `403`: the job did not receive a valid entitled key.
- Empty data: confirm the identifier and inspect `resolvedTag`; issuers can report equivalent concepts under different taxonomy tags.
- Period confusion: do not compare a quarterly fact with an annual fact without retaining `fp`, `periodEnd`, and `filedAt`.
- Restatements: filing accession and date are part of the metric identity. Decide explicitly whether your model uses the latest restated value or originally reported value.

## Production considerations

Store raw values as decimals or integers in the returned unit, cache historical requests, and attach request/trace metadata to support records. Never expose the key in browser code, and label derived analysis as analysis rather than an SEC statement.
