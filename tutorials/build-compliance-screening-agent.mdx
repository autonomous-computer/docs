description: "Route signed SEC filing-monitor events into a review agent that preserves the issuer, filing, evidence, and decision boundary."

This guide builds a small screening service: SEC API sends a signed filing-monitor event, the service extracts the filing facts that matter, and a reviewer receives an evidence-linked queue item. It does not make a legal or investment decision, resolve a name match, or clear a counterparty.

## Prerequisites

- Python 3.9+, Flask, and a public HTTPS receiver URL.
- A signed-in SEC API organization administrator for the [SEC API dashboard](https://secapi.ai/app/webhooks).
- `SECAPI_WEBHOOK_SIGNING_SECRET` from the configured endpoint and `SECAPI_API_KEY` only for the read-only match lookup.

## Configure the monitor first

In the signed-in [SEC API dashboard](https://secapi.ai/app/webhooks), create an organization webhook endpoint subscribed to `monitor.match`, then create the saved monitor for your client, portfolio, or counterparty set. These configuration actions require a WorkOS browser session. An API key cannot create monitors or webhooks.

Use a narrow monitor name, keyword query, form filter, and ticker list. Save the dashboard-provided signing secret as `SECAPI_WEBHOOK_SIGNING_SECRET`.

## Run the receiver

```python
import hashlib
import hmac
import json
import os
import time

from flask import Flask, jsonify, request

app = Flask(__name__)
SECRET = os.environ["SECAPI_WEBHOOK_SIGNING_SECRET"]

def valid(raw: bytes, header: str) -> bool:
    fields = dict(part.split("=", 1) for part in header.split(",") if "=" in part)
    timestamp, signature = fields.get("t", ""), fields.get("v1", "")
    try:
        stale = abs(time.time() - int(timestamp)) > 300
    except ValueError:
        return False
    if not signature or stale:
        return False
    expected = hmac.new(SECRET.encode(), f"{timestamp}.".encode() + raw, hashlib.sha256).hexdigest()
    return hmac.compare_digest(expected, signature)

@app.post("/webhooks/secapi/compliance")
def receive():
    raw = request.get_data()
    if not valid(raw, request.headers.get("x-secapi-signature", "")):
        return jsonify({"error": "invalid_signature"}), 401
    event = json.loads(raw)
    if event.get("type") != "monitor.match":
        return jsonify({"status": "ignored"}), 200
    for match in event.get("data", {}).get("matches", []):
        print({"ticker": match.get("ticker"), "form": match.get("form"), "filedAt": match.get("filingDate"), "sourceUrl": match.get("htmlUrl")})
    return jsonify({"received": True}), 200
```

Run it with `pip install flask` and `SECAPI_WEBHOOK_SIGNING_SECRET=whsec_... flask --app receiver run`. Expose the receiver at the HTTPS destination configured in the dashboard. A valid `monitor.match` produces one printed source-linked record per returned match; a valid `webhook.test` is acknowledged as ignored.

## Add evidence, then review

For a configured monitor, use an API key to inspect current matches before or after delivery:

```bash
curl -H "x-api-key: $SECAPI_API_KEY" \
  "https://api.secapi.ai/v1/monitors/MONITOR_ID/matches?limit=25"
```

Carry the issuer, form, filing date, source URL, monitor name, and SEC API request ID into every downstream record. An LLM can classify or summarize the filing, but the source filing and the human review decision should remain visible.

## Failure modes

- `401 invalid_signature` usually means the HMAC used parsed JSON instead of the raw bytes, the secret is wrong, or the timestamp is stale.
- The example only prints records. In production, write the event id to a durable unique store before enqueueing work so retry delivery cannot create duplicate screenings.
- A `webhook.test` proves the destination and signing path, not that the configured query will find a filing.
- A text match or an LLM classification is a review lead. Preserve the source URL and require the appropriate reviewer to make the decision.
