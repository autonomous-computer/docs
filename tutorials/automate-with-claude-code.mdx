---
title: "Automate a 13F Review with Claude Code"
description: "Use a small, source-preserving 13F comparison script with Claude Code to prepare a human review queue."
---

Use this workflow to turn a manager's two latest parsable Form 13F reports into a review artifact. Claude Code can help inspect and format the artifact, but the SEC API response and linked filings remain the evidence. A 13F is a delayed reported snapshot; it is not a current portfolio, trading blotter, or statement of investment intent.

## Prerequisites

- [Claude Code](https://claude.com/claude-code) installed and authenticated.
- Node.js 18+ and an SEC API key in `SECAPI_API_KEY`.
- The reporting manager's CIK. This example uses `0001067983`.

```bash
export SECAPI_API_KEY="your_api_key"
export MANAGER_CIK="0001067983"
mkdir 13f-review && cd 13f-review
```

## 1. Fetch a bounded comparison

Create `fetch-13f-comparison.mjs`:

```js
import { writeFile } from "node:fs/promises";

const cik = process.env.MANAGER_CIK;
const apiKey = process.env.SECAPI_API_KEY;
if (!cik || !apiKey) throw new Error("MANAGER_CIK and SECAPI_API_KEY are required");

const response = await fetch("https://api.secapi.ai/v1/owners/13f/compare", {
  method: "POST",
  headers: { "x-api-key": apiKey, "content-type": "application/json" },
  body: JSON.stringify({ cik, limit: 50 }),
});
if (!response.ok) throw new Error(`SEC API ${response.status}: ${await response.text()}`);

const comparison = await response.json();
await writeFile("13f-comparison.json", JSON.stringify(comparison, null, 2));
console.log(JSON.stringify({
  managerName: comparison.managerName,
  currentFilingDate: comparison.currentFilingDate,
  previousFilingDate: comparison.previousFilingDate,
  rows: comparison.rows?.length ?? 0,
  requestId: comparison.requestId,
}, null, 2));
```

Run `node fetch-13f-comparison.mjs`. The expected result is `13f-comparison.json` containing the two selected filings, position rows, provenance, and request metadata. The route selects the latest two parsable reports; it does not compare arbitrary historical quarters.

## 2. Ask Claude Code for a review artifact

Open the directory in Claude Code and give it this bounded request:

```text
Read 13f-comparison.json. Create 13f-review.md with sections for added, removed,
and changed rows. For every row retain issuer, CUSIP, previous and current shares,
the two filing dates, and any accession or filing URL returned by the API. Do not
describe a change as a trade, current holding, or investment thesis. Add a short
"Review required" note that points the reader to the source filings.
```

Review the generated markdown before sharing it. A useful artifact groups the returned `status` rows and makes both filing dates visible; it does not manufacture explanations for a change.

## 3. Resolve the underlying reports

Use the filings route when the reviewer needs the report date, accession, or SEC link for a selected comparison:

```bash
curl --get --fail-with-body -H "x-api-key: $SECAPI_API_KEY" \
  --data-urlencode "cik=$MANAGER_CIK" --data-urlencode "limit=8" \
  "https://api.secapi.ai/v1/owners/13f/filings" \
  | jq '{data, requestId, traceparent, provenance}'
```

Keep the manager CIK, report date, filing date, accession, filing URL, and `requestId` alongside the generated review. The source filing is the authoritative record for a material statement.

## Failure modes

- A `400` response commonly means the CIK is missing or malformed; this route needs the reporting manager CIK, not a portfolio-company ticker.
- A comparison needs two parsable reports. An empty or unavailable comparison is not evidence of no exposure or no change.
- Do not let an agent replace null values with zero or infer a trade from an added or removed row; corporate actions and reporting conventions can change a row.
- Keep the JSON artifact out of version control if it contains data your repository should not retain, and never place the API key in a prompt, source file, or commit.
