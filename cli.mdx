---
title: "SEC API CLI"
description: "Install the SEC API CLI, authenticate through your environment, and run source-backed filing workflows from a terminal"
---

Use the SEC API CLI for terminal workflows, shell scripts, and quick checks. It prints structured output so you can keep filing identifiers and request metadata in the next command or a saved artifact.

## Install

```bash
npm install -g @secapi/cli
secapi --version
```

New scripts should use the `secapi` command.

## Authenticate

Set the API key in the calling environment:

```bash
export SECAPI_API_KEY="secapi_..."
secapi doctor
```

`doctor` reports the active API origin, credential source name, and available setup information without printing your key. Do not place credentials in shell history, process arguments, or saved command output.

## Run a first workflow

```bash
secapi entities resolve --ticker AAPL
secapi filings latest --ticker AAPL --form 10-K
secapi sections get --ticker AAPL --form 10-K --section item_1a --view agent
```

Keep the returned `requestId`, `accessionNumber`, `filingDate`, and `filingUrl` with any result you use. They identify the request and the underlying SEC filing.

## Inspect before you run

Use built-in help to check a command's flags and examples before issuing a request:

```bash
secapi --help
secapi help all
secapi filings latest --help
secapi agent-context
```

`agent-context` emits a machine-readable command inventory for an agent or integration that needs to select a CLI command deliberately.

For supported account or delivery commands, `--dry-run` prints the request shape without making a network call:

```bash
secapi api-keys create --label development --dry-run
secapi webhooks create \
  --destination-url https://example.com/hooks/sec \
  --event-types monitor.match \
  --dry-run
```

Use the signed-in dashboard for key, billing, and delivery-resource configuration. These organization changes require a human-authenticated browser session.

## Use profiles for multiple environments

Profiles can store a base URL and the *name* of an environment variable that contains a key. They do not need to store secret values.

```bash
secapi config show
secapi config profiles
secapi --profile local config show
```

Use `--base-url` for one command when you are testing an environment you control. Leave production as the default unless you mean to change it.

## Troubleshoot a command

Run `secapi doctor` first. Then preserve the command, HTTP status, error code, request ID, and trace context from a failed request. Read [Troubleshooting](/troubleshooting) for retry and billing guidance, or [MCP installation](/mcp-install) when the issue is an MCP client configuration.
