## Prerequisites

- Node.js and npm.
- An API key in your shell environment.

```bash
npm install -g @secapi/cli
export SECAPI_API_KEY="secapi_..."
secapi --version
```

The preferred command is `secapi`. `omni-sec` remains a compatibility alias.

## Check local configuration

```bash
secapi config show
secapi doctor
```

`config show` is local-only and reports the configured origin and credential source names without printing secrets. `doctor` can call the API to check setup and account context; use it when authentication is uncertain.

## Run one source-backed command

```bash
secapi filings latest --ticker AAPL --form 10-K
```

Expect JSON containing a current filing identity. Preserve `accessionNumber`, `filingDate`, `filingUrl`, and `requestId` if you use the result in a script or report. The filing identity is live and changes when a newer annual filing becomes available.

To inspect a command before making a request:

```bash
secapi filings latest --help
secapi help all
```

## Run a Special Situations workflow

The published CLI supports these Special Situations discovery and detail commands. Start broad, then retrieve a selected record and preserve its cited source context:

```bash
secapi situations list --types merger --statuses pending --limit 25
secapi situations by-form --form "SC 13D" --limit 25
secapi situations feed --tickers ACME --limit 25
secapi situations calendar --date-types vote,expected_close --days 30
secapi situations stats --window 30d
secapi situations performance --group-by type
secapi situations get --situation-id sit_abc123
secapi situations filings --situation-id sit_abc123 --limit 25
secapi situations summary --situation-id sit_abc123
```

For the complete paid REST surface, including immutable issue reads, source-cited exports, and underwriting packs, use the [Special Situations API reference](/api-reference/situations). Anonymous Embed routes are deliberately separate and capped; the CLI does not turn them into a substitute for authenticated research.

## Handle failures

- A missing or invalid key returns an authentication failure. Check the environment variable rather than passing a secret as a command argument.
- For `429`, wait for the documented retry interval before retrying a read.
- For a persistent failure, rerun with `--request-summary` and retain the request ID, status, endpoint, and timestamp. The command response remains on stdout; the summary is written to stderr.

## Production notes

- In CI, inject keys through the runner's secret mechanism. Never write them into profiles, command history, or command output.
- Use `--dry-run` only on supported mutating commands to inspect a request shape; it does not test account permission or delivery behavior.
- Profiles store environment-variable names, not secret values. Use a separate profile and base URL only for an environment you control.
- Parse JSON as a contract that can gain optional fields; pin any application assumptions to the relevant API reference.
